Les 3.5: Documentatie (XML Comments)

Je hebt nu een veilige, robuuste class gebouwd. Maar als een andere programmeur (of jijzelf over 3 maanden) jouw code gebruikt, hoe weten ze dan wat de regels zijn? Tijd voor de handleiding.

De Metafoor: De Tooltip

Stel je voor dat je een game speelt. Je vindt een zwaard in je inventory.

De Code (De Blauwdruk)

Dit is hoe het zwaard gemaakt is (het metaal, het smeden). Dit is interessant voor de smid (jij), maar niet voor de avonturier.

De Tooltip (De Info)

Als je met je muis over het zwaard gaat, zie je: "Damage: 50. Effect: Fire." Dit vertelt je hoe je het gebruikt, zonder dat je hoeft te weten hoe het gesmeed is.

In Visual Studio heet dit IntelliSense. Door speciale comments te schrijven, zorg je dat Visual Studio jouw uitleg laat zien als pop-up in Program.cs.

Het Probleem: Onzichtbare Comments

Misschien gebruik je al // voor commentaar. Dat is prima voor notities binnen de methode.

// Dit checkt of de naam leeg is
if (string.IsNullOrEmpty(newName)) { ... }

Het probleem: Iemand die in Program.cs jouw code gebruikt, ziet deze tekst niet. Die moet gokken wat de parameters doen.

De Oplossing: XML Comments (///)

Als je boven een methode, constructor of property drie keer een slash typt (`///`), gebeurt er iets magisch. Visual Studio genereert automatisch een template.

De belangrijkste Tags

  • <summary>: De koptekst. Wat doet dit blok code in één zin?
  • <remarks>: De kleine lettertjes. Hier zet je extra details, waarschuwingen of voorbeelden.
  • <param>: Wat wordt er verwacht als input?
  • <returns>: Wat krijg ik terug als antwoord?

Implementatie: DigitalPet Documenteren

We gaan onze class professioneel documenteren. Open Models/DigitalPet.cs.

Stap 1: De Class beschrijven (met Remarks)

Ga helemaal naar boven, vlak boven public class DigitalPet. Typ `///` en vul de tekst in.

Gebruik de remarks tag om uit te leggen wat er speciaal is aan deze class.

/// Let op: De energie van dit dier kan nooit onder de 0 of boven de 100 komen.
/// Dit wordt automatisch geregeld door de properties.

Stap 2: De Constructor Documenteren

Dit is de belangrijkste plek! In Program.cs typ je vaak new DigitalPet(...). Hier wil je precies weten wat de optionele getallen betekenen.

    /// <summary>
    /// Maakt een nieuw digitaal huisdier aan.
    /// </summary>
    /// <param name="name">De naam van het dier (verplicht).</param>
    /// <param name="energy">Start energie (Standaard 50).</param>
    /// <param name="hunger">Start honger (Standaard 0).</param>
    /// <param name="isSleeping">Slaapt het dier bij geboorte? (Standaard false).</param>
    public DigitalPet(string name, int energy = 50, int hunger = 0, bool isSleeping = false)
    {
        this.Name = name;
        // ... rest van de code
    }

Stap 3: Methodes documenteren (Rename)

Ook bij methodes is het handig om te weten wat je terugkrijgt. Vooral de returns tag is hier belangrijk.

    /// <summary>
    /// Probeert de naam van het dier te wijzigen.
    /// </summary>
    /// <remarks>
    /// Als de nieuwe naam leeg of null is, wordt de wijziging genegeerd.
    /// </remarks>
    /// <param name="newName">De gewenste nieuwe naam.</param>
    /// <returns>Een tekstbericht of de wijziging gelukt of mislukt is.</returns>
    public string Rename(string? newName)
    {
        // ... (jouw code)
    }
Let op: Lieg niet in je comments! Als je in de code iets verandert (bijv. de default energie wordt 60), moet je ook je commentaar updaten. Foute documentatie is erger dan geen documentatie.

Het Resultaat zien

Je hoeft je code niet te runnen om dit te testen.

  1. Ga naar Program.cs.
  2. Typ ergens: new DigitalPet(
  3. Zodra je het haakje opent, zie je jouw eigen tekst verschijnen!

Je ziet nu de Summary direct. De Remarks zie je vaak als je op het kleine pijltje "More info" klikt in de pop-up.

Samenvatting Module 3

Gefeliciteerd! Je hebt Module 3 afgerond. Je bent van "losse variabelen" naar "professionele objecten" gegaan:

  • Encapsulation: Data beschermen met private set.
  • Logic: Validatie in de Property set (Backing Fields).
  • Safety: Nullable types (`string?`) en Type Safety.
  • Documentation: XML comments met summary en remarks voor IntelliSense.
