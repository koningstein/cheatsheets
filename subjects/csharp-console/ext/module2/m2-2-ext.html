Namespaces & Using

Als je project groter wordt, krijg je steeds meer bestanden. Je wilt niet dat alles op één grote hoop ligt. Je wilt structuur. In C# gebruiken we hiervoor Namespaces.

Het Probleem: Dubbele Namen

Stel je voor dat jij een game maakt en je maakt een class genaamd Console (voor een spelcomputer in je game).
Maar... C# heeft zelf óók al een class die Console heet (die gebruiken we voor WriteLine).

Als je nu Console typt, raakt C# in de war. Welke bedoel je? Die van jou, of die van Microsoft?

De Vergelijking: Achternamen

Stel dat er twee jongens in de klas zitten die allebei "Jan" heten. Als de leraar roept: "Jan, kom eens hier!", weten ze niet wie hij bedoelt.

Daarom gebruiken we achternamen: "Jan Jansen" en "Jan de Vries".

Een Namespace is eigenlijk de achternaam van je Class.

1. Een Namespace maken

Je geeft je code een 'achternaam' door bovenaan je bestand aan te geven in welke namespace het hoort. Vaak gebruiken we hier de naam van het project of de map voor.

De Moderne Manier (File Scoped)

Sinds C# 10 kunnen we dit heel kort opschrijven. Je zet het bovenaan met een puntkomma. Alles in dit bestand hoort nu bij die familie.

// BESTAND: DigitalPet.cs

// Dit is de 'achternaam' van deze code
namespace MyTamagotchiGame;

class DigitalPet
{
    // ...
}

De volledige naam van je class is nu: MyTamagotchiGame.DigitalPet.

Oude code lezen

In oudere tutorials zie je vaak dat de namespace accolades { } heeft en dat alle code inspringt. Dat werkt nog steeds, maar we gebruiken liever de nieuwe manier (File Scoped) omdat je dan minder witruimte verspilt.

2. Using Statements: Code importeren

Nu hebben we onze classes netjes in hokjes (namespaces) gestopt. Maar wat als we ze willen gebruiken?

Stel je wilt System.Console.WriteLine() gebruiken. Het is heel irritant om steeds die volledige naam (inclusief achternaam) te typen.

Hiervoor is het keyword using. Hiermee zeg je tegen C#: "Als ik een naam typ die je niet kent, zoek dan even in deze doos."

// ZONDER using (Heel veel typwerk):
System.Console.WriteLine("Hallo");
System.Console.ReadLine();

// MET using (Bovenaan je bestand):
using System;

Console.WriteLine("Hallo"); // C# snapt nu dat je 'System.Console' bedoelt.

Toepassing in Digital Pet

Laten we kijken hoe dit werkt als we onze eigen bestanden koppelen.

Stap A: De Class (DigitalPet.cs)

We zetten de class in de namespace van het spel.

using System;

// We definiëren de namespace
namespace DigitalPetGame;

public class DigitalPet
{
    public string Name { get; set; }

    // Constructor (zoals geleerd in Module 1)
    public DigitalPet(string name)
    {
        this.Name = name;
    }
}

Stap B: Het Gebruik (Program.cs)

Stel dat Program.cs per ongeluk in een andere namespace zit, of dat we mappen gebruiken (daarover later meer). Dan moeten we vertellen waar DigitalPet woont.

using System;
// We importeren onze eigen namespace!
using DigitalPetGame;

// Nu kunnen we de korte naam gebruiken
// Let op: We gebruiken de constructor die we in Module 1 leerden
DigitalPet dino = new DigitalPet("Rex");

Wat kan er fout gaan? (Veelgemaakte Fouten)

⚠️ Rode strepen (CS0246)

Krijg je een rode streep onder DigitalPet met de melding "The type or namespace name could not be found"?

Dan ben je waarschijnlijk de using vergeten.
Tip: Klik op de rode tekst en druk op Ctrl + . (punt). Visual Studio stelt dan vaak voor om de using automatisch toe te voegen.

Samenvatting

  • Een Namespace groepeert je code en voorkomt naamconflicten (zoals een achternaam).
  • Je definieert een namespace bovenaan je bestand: namespace MijnGame;.
  • Met using importeer je een namespace, zodat je niet steeds de volledige naam hoeft te typen.
