Mappenstructuur

In de vorige lessen stonden al onze bestanden (Program.cs, DigitalPet.cs) gewoon los naast elkaar. Dat is prima als je maar 3 bestanden hebt. Maar wat als je een grote game maakt met honderden classes?

Dan wordt je project een rommeltje. Je kunt niks meer terugvinden. Daarom gaan we leren hoe je een C# project netjes indeelt.

1. Eén Class = Eén Bestand

De allerbelangrijkste regel in C# is: Stop elke class in zijn eigen bestand.

  • Heb je een class DigitalPet? Maak een bestand DigitalPet.cs.
  • Heb je een class Food? Maak een bestand Food.cs.

Stop ze nooit samen in één bestand, ook al zijn ze klein. Dit houdt je code overzichtelijk.

2. Mappen gebruiken (Models)

Net zoals je op je computer mappen gebruikt voor 'School', 'Foto's' en 'Games', doen we dat ook in onze code.

Een standaard afspraak in C# is dat we "blauwdrukken" van data (zoals een Huisdier, een Speler, of een Product) in een map genaamd Models stoppen.

De Vergelijking: De Kledingkast

Als je al je kleren op één hoop op de grond gooit (hoofdmap), vind je nooit je sokken terug.
Daarom heb je lades (mappen):

  • Lade Sokken (Models)
  • Lade T-shirts (Services)

Het opruimen kost even tijd, maar daarna werkt het veel sneller.

Hoe maak je een map in Visual Studio?

  1. Klik met de rechtermuisknop op je Projectnaam (niet de Solution).
  2. Kies Add -> New Folder.
  3. Noem de map Models.
  4. Sleep nu je bestand DigitalPet.cs in deze map.
MijnGame/
│
├── Program.cs          (Het startpunt blijft in de hoofdmap)
├── MijnGame.csproj
│
└── Models/             (De nieuwe map)
    ├── DigitalPet.cs   (Verplaatst)
    └── Food.cs

3. Namespace volgt de Map

Dit is een belangrijk punt. Als je een bestand verplaatst naar een map, verandert zijn "adres". In C# betekent dit dat de Namespace moet veranderen.

Als DigitalPet in de map Models staat, moet de namespace ook eindigen op .Models.

Stap A: Pas de Namespace aan (DigitalPet.cs)

// BESTAND: Models/DigitalPet.cs

// OUD: namespace MijnGame;
// NIEUW (want hij zit in de map Models):
namespace MijnGame.Models;

public class DigitalPet
{
    public string Name { get; set; }
    public int Energy { get; set; }

    // Onze constructor uit Module 1
    public DigitalPet(string name)
    {
        this.Name = name;
        this.Energy = 50;
    }
}

Tip: Automatisch Aanpassen

Als je een bestand versleept in Visual Studio, vraagt hij soms: "Do you want to adjust the namespaces?". Zeg hier altijd JA op. Dan past hij de code voor je aan.

Stap B: Gebruik de Using (Program.cs)

Nu staat DigitalPet opgesloten in de map (namespace) Models. Maar Program.cs staat nog buiten die map. Hij kan DigitalPet niet meer vinden.

Je moet nu bovenaan Program.cs vertellen dat hij ook in de map Models moet kijken.

// BESTAND: Program.cs
using System;

// BELANGRIJK: Importeer de map waar je class in zit
using MijnGame.Models;

// Nu snapt hij weer wat 'DigitalPet' is
// We gebruiken netjes de constructor die we gemaakt hebben
DigitalPet dino = new DigitalPet("Dino");

Console.WriteLine("Het beestje is aangemaakt!");

Wat kan er fout gaan? (Veelgemaakte Fouten)

⚠️ Namespace vergeten aan te passen

Je hebt het bestand wel verplaatst naar de map Models, maar in de code staat nog steeds namespace MijnGame;.

Dit is verwarrend voor andere programmeurs (en voor jezelf). Zorg dat de namespace altijd klopt met de mappenstructuur.

⚠️ Verkeerde Using

Krijg je een rode streep onder DigitalPet? Dan ben je waarschijnlijk using MijnGame.Models; vergeten in Program.cs. Klik op de rode streep en druk op Ctrl + . om het te fixen.

Samenvatting

  • Houd je project netjes: 1 class per bestand.
  • Gebruik mappen (zoals Models) om je bestanden te groeperen.
  • Als je een map gebruikt, verander dan ook de Namespace (bijv. MijnGame.Models).
  • Gebruik using om bestanden uit die map te kunnen gebruiken in andere bestanden.
