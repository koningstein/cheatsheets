Breakpoints: De Tijd Stilzetten

Je hebt je code geschreven, je drukt op Start... en er gebeurt iets wat je niet verwacht. Misschien crasht het programma, of misschien blijft de energie van je huisdier op 0 staan. Wat nu?

De professionele manier om fouten te vinden heet Debuggen.

Wat is Debuggen?

Debuggen betekent dat je de uitvoering van je programma op een specifiek moment pauzeert. Terwijl het programma bevroren is, kun jij rustig rondkijken wat de computer aan het doen is.

De Vergelijking: Een Film Pauzeren

Stel je kijkt een actiefilm en je ziet een stunt waarvan je niet snapt hoe ze die deden.

  • Zonder Debuggen: Je kijkt de film op normale snelheid en probeert te gokken wat er gebeurde.
  • Met Debuggen: Je zet de film op pauze (Breakpoint). Daarna speel je hem frame voor frame af (Stepping) om elk detail te zien.

1. De Breakpoint (Het Stopbord)

Om te debuggen, moet je eerst aangeven waar de computer moet stoppen. Dit doe je met een Breakpoint.

Klik in Visual Studio helemaal links in de grijze rand naast een regel code (of zet je cursor op de regel en druk op F9). Er verschijnt een Rode Bol.

Laten we dit testen met onze Digital Pet.

Stap A: De Code (DigitalPet.cs)

namespace MyGame.Models;

public class DigitalPet
{
    public int Energy { get; set; }

    public DigitalPet()
    {
        // Zoals geleerd in Module 1: Startwaarden in de constructor!
        this.Energy = 100;
    }

    public void Walk()
    {
        // Zet hier straks een breakpoint! (F9)
        this.Energy = this.Energy - 10;
        Console.WriteLine("We hebben gewandeld.");
    }
}

Stap B: Het Programma (Program.cs)

using MyGame.Models;

DigitalPet dino = new DigitalPet();

Console.WriteLine("Start het spel...");

// Roep de methode aan
dino.Walk();

Console.WriteLine("Einde spel.");

2. Het programma bevriezen

Zet een breakpoint op de regel dino.Walk(); in Program.cs. Druk nu op F5 (Start Debugging).

Het programma start, maar stopt plotseling. De regel met de rode bol wordt Geel.

Dit betekent: "Ik sta op het punt om deze regel uit te voeren, maar ik wacht op jou."

3. Navigeren (Stepping)

Nu heb jij de controle. Bovenin Visual Studio zie je een werkbalk met pijltjes. De belangrijkste toetsen zijn F10 en F11. Het verschil is cruciaal.

Step Over (F10)

Betekenis: "Voer deze regel uit en ga naar de volgende regel in DIT bestand."

  • Als je op dino.Walk() staat en je drukt F10...
  • Dan voert hij de hele wandeling in één keer uit.
  • De gele balk springt direct naar Console.WriteLine("Einde spel.");.

Gebruik dit als je zeker weet dat de functie Walk goed werkt en je er niet in hoeft te kijken.

Step Into (F11)

Betekenis: "Als deze regel een methode aanroept, duik er dan IN."

  • Als je op dino.Walk() staat en je drukt F11...
  • Dan springt de gele balk naar het bestand DigitalPet.cs.
  • Je ziet nu regel voor regel hoe de energie eraf gaat.

Gebruik dit als je wilt controleren of je methode wel goed werkt.

Toets Naam Actie
F5 Continue Haal de pauze eraf en ren door tot het volgende breakpoint (of het einde).
F10 Step Over Doe de regel en ga naar de volgende (blijf in huidig bestand).
F11 Step Into Spring in de methode om details te zien.
Shift+F11 Step Out Ik heb genoeg gezien, maak deze methode snel af en ga terug.

Wat kan er fout gaan? (Veelgemaakte Fouten)

⚠️ De gele regel is nog NIET uitgevoerd

Dit is de grootste verwarring. Als de regel Energy = Energy - 10; geel is, is de energie nog niet verlaagd. De computer staat klaar om het te doen. Pas als je op F10 drukt en de gele balk voorbij die regel is, is het gebeurd.

Samenvatting

  • Breakpoint (F9): Hier moet het programma pauzeren.
  • Gele regel: Dit is de regel die zo meteen uitgevoerd gaat worden.
  • F10 (Over): Volgende regel in dit bestand.
  • F11 (Into): Spring in de methode om details te zien.
