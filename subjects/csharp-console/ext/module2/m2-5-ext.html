Breakpoints: De Tijd Stilzetten

Je hebt nu code die over meerdere bestanden verspreid staat (Program.cs en DigitalPet.cs). Als er iets fout gaat, is het moeilijk te zien waar het misgaat. In welk bestand zit de fout?

De professionele manier om dit te onderzoeken heet Debuggen.

Wat is Debuggen?

Debuggen betekent dat je de uitvoering van je programma op een specifiek moment pauzeert. Terwijl het programma bevroren is, kun jij als een detective rondkijken wat de computer aan het doen is.

De Vergelijking: Een Film Pauzeren

Stel je kijkt een actiefilm en je ziet een stunt waarvan je niet snapt hoe ze die deden.

  • Zonder Debuggen: Je kijkt de film op normale snelheid en probeert te gokken wat er gebeurde.
  • Met Debuggen: Je zet de film op pauze (Breakpoint). Daarna speel je hem frame voor frame af (Stepping) om elk detail te zien.

Stap 1: Voorbereiding

We voegen een methode toe die iets teruggeeft (return). Zo kunnen we zien hoe data heen en weer reist tussen bestanden.

A. Pas DigitalPet.cs aan

Zet deze methode in je class DigitalPet. Let op: het is een string, geen void!

// IN BESTAND: Tamagotchi/Models/DigitalPet.cs

public string Sleep()
{
    this.IsSleeping = true;
    this.Energy += 20;

    // We printen NIET hier, maar sturen de tekst terug
    return "Zzz... Dat deed deugd! (+20 Energie)";
}

B. Pas Program.cs aan

Voeg onderaan Program.cs de code toe die de methode aanroept en het antwoord opvangt.

// IN BESTAND: Program.cs

// ... (Je bestaande code) ...

Console.WriteLine("Het wordt laat...");

// Zet HIER STRAKS je Breakpoint (op de regel hieronder)
Console.WriteLine(dino.Sleep());
Console.ReadLine();

Stap 2: De Breakpoint (Het Stopbord)

Om te debuggen, moet je aangeven waar de computer moet stoppen.

  1. Ga naar Program.cs.
  2. Zoek de regel: string resultaat = dino.Sleep();.
  3. Klik helemaal links in de grijze rand naast deze regel (of druk op F9).
  4. Er verschijnt een Rode Bol.

Stap 3: Step Into (F11) - De Sprong

Start de debugger met F5. De code stopt en de regel wordt geel.

We staan nu in Program.cs, maar de actie gebeurt in DigitalPet.cs.
Als we op F10 drukken, missen we de actie. Wij willen erin duiken!

Druk op F11 (Step Into).

Visual Studio springt nu IN de methode. Je scherm wisselt automatisch naar DigitalPet.cs.

Stap 4: Variabelen Inspecteren

Nu je in de methode staat, kun je kijken wat er gebeurt.

  1. Druk een paar keer op F10 (Step Over) om door de regels te lopen.
  2. Stop vlak voor de return.
  3. Beweeg je muis over this.Energy. Je ziet de waarde live veranderen (bijv. van 50 naar 70).

Stap 5: Terug naar de start

Druk nog een keer op F10 bij de return-regel.

Je springt nu automatisch weer terug naar Program.cs! De variabele resultaat is nu gevuld met de tekst die uit de andere class kwam. Beweeg je muis er maar eens overheen.

Toets Naam Actie
F5 Continue Haal de pauze eraf en ren door tot het volgende breakpoint.
F10 Step Over Volgende regel (blijf in dit bestand).
F11 Step Into Spring in de methode (wissel van bestand).

Samenvatting

  • Gebruik F9 om een stoppunt te maken.
  • Gebruik F11 om te zien hoe een methode van binnen werkt (jump naar andere file).
  • Debuggen laat je zien hoe data heen (parameters) en weer (return) reist.
