Variables Inspecteren: Onder de motorkap

In de vorige les hebben we geleerd hoe we de code kunnen pauzeren met een Breakpoint. De code staat stil en de regel is geel.

Maar... alleen maar stilstaan heeft geen zin. Je wilt weten waarom je code crasht of waarom je huisdier opeens dood is. Je wilt de waardes van je variabelen zien op dat specifieke moment.

De Vergelijking: De Monteur

Stel je voor dat je auto een raar geluid maakt. Je zet de auto stil langs de kant van de weg (Breakpoint).

Nu ga je niet willekeurig onderdelen vervangen. Nee, je doet de motorkap open en kijkt naar de meters: Hoe heet is de motor? Hoeveel olie zit erin? Zit er een kabel los?

Het inspecteren van variabelen is precies dat: kijken naar de meters van je programma.

Methode 1: Hoveren (Snel kijken)

De snelste manier om een waarde te checken is met je muis.

  1. Zorg dat het programma gepauzeerd is (gele regel).
  2. Beweeg je muis over een variabele naam, bijvoorbeeld energy.
  3. Er verschijnt direct een klein pop-up venster met de waarde: 50.

Als je over een object (zoals dino) hovert, zie je een klein pijltje. Als je daarop klikt, klapt het object open en zie je alle properties (Name, Energy, Hunger) in één keer.

Methode 2: Het 'Locals' Venster

Onderaan in Visual Studio zie je tijdens het debuggen het tabblad Locals. Dit is je dashboard.

Hier toont C# automatisch alle variabelen die op dit moment bekend zijn in de methode waar je bent. Je hoeft dus niet te zoeken.

Naam        Waarde           Type
------------------------------------------
dino        {DigitalPet}     DigitalPet
  > Name    "Dino"           string
  > Energy  50               int
i           0                int

Methode 3: Het 'Watch' Venster (De Monitor)

Soms is Locals te druk. Je wilt alleen één specifiek ding in de gaten houden, bijvoorbeeld of de energie onder de 10 komt.

Hiervoor gebruik je de Watch. Hier kun je zelf dingen typen.

  1. Klik met de rechtermuisknop op een variabele in je code.
  2. Kies Add to Watch.
  3. Onderaan verschijnt het Watch venster.

Het krachtige is dat je hier ook sommen of vragen kunt typen:

  • dino.Energy (Toont de waarde: 50)
  • dino.Energy - 10 (Toont wat het zou zijn: 40)
  • dino.Energy < 0 (Toont: false)

Terwijl je met F10 door de code stapt, worden deze waardes live bijgewerkt.

Speciale Truc: Waardes aanpassen (God Mode)

Dit weten veel beginners niet: Je kunt tijdens het debuggen de waardes veranderen!

Stel je wilt testen wat er gebeurt als je huisdier doodgaat. Normaal moet je dan 10 minuten spelen tot de energie 0 is. Dat duurt te lang.

  1. Pauzeer de code.
  2. Ga naar het Locals of Watch venster.
  3. Dubbelklik op de waarde van Energy (bijv. 50).
  4. Typ 0 en druk op Enter.
  5. Druk op F5 (Continue).

Je programma gaat nu verder alsof de energie altijd al 0 was. Zo kun je heel snel testen of je 'Game Over' scherm werkt.

⚠️ Ik zie niks als ik hover

Dit gebeurt als het programma niet gepauzeerd is. Je kunt alleen variabelen inspecteren als de code 'bevroren' is (dus als je een gele regel ziet). Als het programma gewoon draait, gaat het te snel voor Visual Studio om iets te laten zien.

⚠️ Variabele bestaat (nog) niet

Stel je staat met de gele balk op regel 10, maar je probeert een variabele te bekijken die pas op regel 20 wordt aangemaakt. Dat kan niet. De variabele bestaat op dat moment nog niet in het geheugen.

Samenvatting

  • Gebruik Hover (muis) voor een snelle check.
  • Gebruik Locals om alles te zien wat nu in het geheugen zit.
  • Gebruik Watch om specifieke berekeningen in de gaten te houden.
  • Je kunt waardes aanpassen tijdens het debuggen om snel scenario's te testen.
