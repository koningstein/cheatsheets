Variables Inspecteren: Onder de motorkap

In de vorige les heb je geleerd hoe je de code pauzeert met een Breakpoint. Maar... alleen maar stilstaan heeft geen zin. Je wilt weten waarom je huisdier doet wat hij doet.

Je wilt onder de motorkap kijken: Hoeveel honger heeft Dino precies? Is Fluffy al aangemaakt? Wat is de waarde van de appel?

De Vergelijking: De Monteur

Stel je voor dat je auto een raar geluid maakt. Je zet de auto stil langs de kant van de weg (Breakpoint).

Nu ga je niet willekeurig onderdelen vervangen. Nee, je doet de motorkap open en kijkt naar de meters: Hoe heet is de motor? Hoeveel olie zit erin?

Het inspecteren van variabelen is precies dat: kijken naar de meters van je programma.

Voorbereiding

Zorg dat je een breakpoint hebt staan in Program.cs, bijvoorbeeld op de regel:

Console.WriteLine(dino.Sleep());

Start het programma met F5 en wacht tot het geel wordt.

Methode 1: Hoveren (Snel kijken)

De snelste manier om een waarde te checken is met je muis.

  1. Beweeg je muis over het woordje dino in je code.
  2. Er verschijnt een pop-up. Waarschijnlijk zie je alleen {Tamagotchi.Models.DigitalPet}.
  3. Klik op het pijltje (>) in de pop-up.

Nu klapt het object open en zie je live de inhoud:

  • Name: "Rex"
  • Energy: 50 (of 70 als hij al geslapen heeft)
  • IsSleeping: true

Methode 2: Het 'Locals' Venster

Onderaan in Visual Studio zie je het tabblad Locals. Dit is je dashboard. Hier zie je alles wat op dit moment bestaat.

Als je iets verder in je code pauzeert (bijvoorbeeld bij de regel over de Appel), zie je hier al je objecten onder elkaar:

Naam        Waarde           Type
------------------------------------------
dino        {DigitalPet}     DigitalPet
  > Name    "Rex"            string
  > Energy  70               int
fluffy      {DigitalPet}     DigitalPet
  > Name    "Fluffy"         string
apple       {Food}           Food
  > Name    "Appel"          string

Methode 3: Het 'Watch' Venster (De Monitor)

Soms is Locals te druk. Je wilt alleen één specifiek ding in de gaten houden. Bijvoorbeeld: "Heeft Dino genoeg energie om te winnen?"

Hiervoor gebruik je de Watch. Hier kun je zelf vragen stellen aan je code.

  1. Klik met de rechtermuisknop op dino in je code.
  2. Kies Add to Watch.
  3. Onderaan verschijnt het Watch venster.

Klik in de lege regel onderaan de lijst en typ zelf deze formules in:

  • Typ: dino.Energy (Resultaat: 70)
  • Typ: dino.Energy > 90 (Resultaat: false -> Hij zit nog niet vol)
  • Typ: dino.Name + " is lief" (Resultaat: "Rex is lief")

Terwijl je met F10 door de code stapt, worden deze antwoorden live bijgewerkt!

Speciale Truc: Waardes aanpassen (God Mode)

Dit weten veel beginners niet: Je kunt tijdens het debuggen de waardes veranderen! Je kunt dus valsspelen in je eigen programma om dingen te testen.

Oefening: Verander de naam

  1. Zorg dat de code gepauzeerd is op een regel voordat de GetDescription wordt geprint.
  2. Ga naar het Locals venster en klap dino open.
  3. Dubbelklik op de waarde "Rex".
  4. Verander het in "Godzilla" en druk op Enter.
  5. Druk nu op F5 om het programma verder te laten gaan.

Kijk in je Console output. Er staat nu: "Ik ben Godzilla..." terwijl je dat nooit geprogrammeerd hebt! Dit is ideaal om snel te testen wat er gebeurt als een waarde heel raar is (bijv. Energy: -1000).

Wat kan er fout gaan? (Veelgemaakte Fouten)

⚠️ Variabele is 'null'

Zie je bij fluffy de waarde null staan? Dat betekent dat je breakpoint te vroeg staat. De regel new DigitalPet(...) voor Fluffy is nog niet uitgevoerd. Druk een paar keer op F10.

⚠️ Ik zie niks als ik hover

Je kunt alleen variabelen inspecteren als de code geel (gepauzeerd) is. Als het programma gewoon draait, gaat het te snel voor Visual Studio om iets te laten zien.

Samenvatting

  • Gebruik Hover (muis) om snel in een object te kijken.
  • Gebruik Locals om al je variabelen (Dino, Fluffy, Apple) tegelijk te zien.
  • Gebruik Watch om berekeningen te testen (bijv. Energy > 100).
  • Je kunt waardes aanpassen tijdens het debuggen ("God Mode") om scenario's te testen.
