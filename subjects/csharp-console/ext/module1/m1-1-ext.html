Class: De Blauwdruk

Welkom bij Object Oriented Programming (OOP). Voordat we gaan programmeren, moet je de basis snappen. Alles in C# begint met de Class.

Wat is een Class?

Je kunt een class zien als een technische tekening of een recept. Het is een beschrijving van hoe iets eruitziet en wat het kan doen. Maar let op: de class is nog niet het "ding" zelf.

Voorbeeld: De Robot Fabriek

Stel je voor: je ontwerpt een robot. Je maakt een tekening waarop staat:

  • De robot heeft een kleur.
  • De robot heeft batterij-energie.
  • De robot kan lopen en praten.

Deze tekening is jouw Class. Je kunt niet praten met de tekening. Maar, de fabriek kan met die ene tekening wel 1.000 echte robots bouwen. Die echte robots noemen we straks Objecten.

De Class in ons project

In deze module maken we een digitaal huisdier (Digital Pet). We willen straks misschien wel drie dieren hebben: Dino, Fluffy en Blafmans.

Het is onhandig om voor elk dier losse code te schrijven. In plaats daarvan maken we één blauwdruk: de DigitalPet class.

Hierin schrijven we op wat elk huisdier heeft:

  • Elk dier heeft een Naam.
  • Elk dier heeft Energie.

Hoe maak je een Class?

In C# is het heel simpel. Je gebruikt het woord class, gevolgd door de naam. Daarna gebruik je accolades { } voor de inhoud.

class DigitalPet
{
    // Alles tussen deze accolades hoort bij de blauwdruk.
    // Nu is het nog leeg, maar straks komt hier code te staan.
}

De Spelregels (Best Practices)

Om te zorgen dat jouw code netjes is en goed samenwerkt met code van anderen, zijn er strenge regels in C#.

1. De Naamgeving (PascalCase)

Geef je class altijd een Engelse naam. We gebruiken hiervoor PascalCase. Dat betekent:

  • De naam begint altijd met een Hoofdletter.
  • Elk nieuw woord begint ook met een Hoofdletter.
  • Gebruik geen spaties of lage streepjes (_).
Goed (PascalCase) Fout Waarom?
DigitalPet digitalPet Moet met een hoofdletter beginnen.
GameManager game_manager Geen underscores gebruiken in class namen.
Program program Dit is de standaard in C#.

2. De Bestandsnaam

Houd je project overzichtelijk: Eén class per bestand.

De naam van het bestand moet precies hetzelfde zijn als de naam van de class, met .cs erachter.

  • Class: DigitalPet → Bestand: DigitalPet.cs
  • Class: Program → Bestand: Program.cs

Waarom is dit belangrijk?

Als je later in een groot team werkt met honderden bestanden, moet je snel dingen kunnen terugvinden. Als je class DigitalPet heet, maar het bestand heet huisdier.cs of code.cs, kan niemand je werk terugvinden.

3. De Accolades

In C# zetten we de openings-accolade { altijd op een nieuwe regel. Dit leest rustiger.

// ZO HOORT HET NETJES IN C#:
class DigitalPet
{
    // ...
}

// DIT DOEN WE LIEVER NIET:
class DigitalPet {
   // ...
}
