Properties: De Eigenschappen

We hebben nu een Class (de blauwdruk) en een Object (de robot). Maar een kale robot is saai. Een robot heeft eigenschappen: een kleur, een serienummer en een batterijniveau.

In C# noemen we deze eigenschappen Properties. Het zijn de gegevens die bij een specifiek object horen.

Wat zijn Properties precies?

Eigenlijk ken je dit concept al. Een property is niets anders dan een variabele, maar dan eentje die vastzit aan een object.

De Vergelijking: Het Paspoort

Denk aan een paspoort. Elk paspoort ziet er qua indeling (Class) hetzelfde uit. Maar de inhoud verschilt per persoon.

  • Naam: "Jan Jansen"
  • Geboortedatum: "01-01-2000"
  • Lengte: "1.80m"

Deze invulvelden (Naam, Datum, Lengte) zijn de Properties.

Properties in onze Digital Pet

Laten we kijken naar ons huisdier. Wat moeten we onthouden van één dier?

Voor onze DigitalPet class hebben we de volgende eigenschappen nodig:

  • Name (Naam): Hoe heet het beestje? (Tekst)
  • Energy (Energie): Hoeveel energie heeft hij nog? (Getal)

Hoe maak je een Property?

Je maakt properties aan in je Class bestand. Er zijn in C# twee belangrijke verschillen met gewone variabelen:

  1. Je moet aangeven dat het public is (iedereen mag het zien).
  2. Je voegt { get; set; } toe.
class DigitalPet
{
    // Hier definiëren we de lege velden (de properties)
    public string Name { get; set; }
    public int Energy { get; set; }
}

Wat betekent { get; set; }?

Dit maakt C# heel krachtig. Het betekent:

  • get: Iedereen mag de waarde lezen ("Wat is je naam?").
  • set: Iedereen mag de waarde veranderen ("Je heet nu Dino").

In latere modules leren we hoe je de 'set' kunt beveiligen, zodat niemand de energie op -50 kan zetten.

Waarden geven en uitlezen

Nu we de velden hebben gemaakt in de blauwdruk, kunnen we ze gaan invullen bij onze objecten. Hiervoor gebruiken we de punt ..

De Punt-notatie (.)

Je kunt de punt lezen als: "Ga naar binnen bij dit object, en pak...".

// 1. Maak Dino (een nieuw object)
DigitalPet pet1 = new DigitalPet();
pet1.Name = "Dino";  // SET: We schrijven de naam erin
pet1.Energy = 100;

// 2. Maak Fluffy (een ander object)
DigitalPet pet2 = new DigitalPet();
pet2.Name = "Fluffy";
pet2.Energy = 50;

// Laten we ze eens bekijken (GET)
Console.WriteLine("Dier 1 heet: " + pet1.Name);
// Output: Dier 1 heet: Dino

Belangrijk: Onafhankelijkheid

Kijk goed naar de code hierboven. Als ik de Name van pet1 verander, blijft pet2 gewoon Fluffy heten. Elk object heeft zijn eigen set properties.

Wat kan er fout gaan? (Veelgemaakte Fouten)

C# is strenger dan veel andere talen. Let op deze valkuilen:

1. Verkeerde Datatypes

Je hebt gezegd dat Energy een int (getal) is. Je mag er dan geen tekst in stoppen.

pet1.Energy = "Honderd"; // FOUT! C# accepteert dit niet.
pet1.Energy = 100;       // GOED!

2. Hoofdletters (Case Sensitivity)

In C# is Name (met hoofdletter) iets anders dan name (met kleine letter). Omdat we in de class Name met een hoofdletter hebben geschreven, moeten we dat overal doen.

pet1.name = "Dino"; // FOUT! 'DigitalPet' bevat geen 'name'.
pet1.Name = "Dino"; // GOED!

Waarom doen we dit?

Misschien denk je: "Ik kan dit toch ook met een lijstje (array) doen?"

// Slechte manier:
string[] petData = { "Dino", "100" };

Dat kan, maar het is onhandig. Je moet onthouden dat nummer 0 de naam is en nummer 1 de energie. En wat als je energie als getal wilt gebruiken om te rekenen? In de array is het tekst.

Bij een object zeg je gewoon pet.Name en pet.Energy. Het is duidelijk, leesbaar en C# controleert of je geen rekenfouten maakt met tekst.
