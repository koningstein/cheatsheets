Constructors: De Geboorte

Tot nu toe maakten we een object aan in twee stappen:

  1. Eerst het object maken met new DigitalPet().
  2. Daarna pas de naam en energie instellen.

Maar dit is eigenlijk gevaarlijk. Wat als je stap 2 vergeet? Dan heb je een huisdier zonder naam. Dat mag in een game niet gebeuren. We willen afdwingen dat een huisdier altijd een naam krijgt op het moment dat hij gemaakt wordt.

Hiervoor gebruiken we de Constructor.

Wat is een Constructor?

Een constructor is een speciale methode die automatisch wordt uitgevoerd zodra je het woord new gebruikt. Het is de plek om de startwaarden (fabrieksinstellingen) van je object goed te zetten.

De Vergelijking: Een Nieuwe Telefoon

Als je een nieuwe telefoon koopt en hem voor het eerst aanzet (new Phone()), kun je hem niet meteen gebruiken. Je moet eerst een taal kiezen en inloggen.

De constructor is dat installatiescherm: het zorgt ervoor dat het apparaat klaar is voor gebruik voordat je er iets mee kunt doen.

Hoe maak je een Constructor?

Een constructor lijkt op een normale methode, maar heeft twee strikte regels:

  1. De naam is exact hetzelfde als de Class.
  2. Er is geen return type (ook geen void).
class DigitalPet
{
    public string Name { get; set; }
    public int Energy { get; set; }

    // Dit is de Constructor
    // Hij heet 'DigitalPet', net als de class.
    public DigitalPet(string name)
    {
        // We gebruiken de parameter 'name' om de property te vullen
        this.Name = name;

        // We geven elk dier standaard 50 energie bij de start
        this.Energy = 50;
    }
}

Het Gebruik (Program.cs)

Nu we in de class hebben gezegd dat de constructor een string name verwacht, is onze oude manier van aanmaken kapot. C# dwingt je nu om die naam mee te geven.

// FOUT: Dit mag niet meer!
// DigitalPet pet = new DigitalPet();
// Error: "There is no argument given that corresponds to the required formal parameter 'name'"

// GOED: We moeten de naam nu tussen de haakjes zetten
DigitalPet myPet = new DigitalPet("Blafmans");

Console.WriteLine(myPet.Name);   // Output: Blafmans
Console.WriteLine(myPet.Energy); // Output: 50 (Dit heeft de constructor gedaan)

De 'Default' Constructor

Waarom werkten die lege haakjes () eerst wel?
Als je zelf geen constructor schrijft, maakt C# er stiekem eentje voor je (een lege). Maar zodra je er zelf één maakt (met parameters), gooit C# die stiekeme lege constructor weg. Vanaf dat moment gelden jouw regels.

Wat kan er fout gaan? (Veelgemaakte Fouten)

⚠️ Het woordje 'void'

Veel studenten typen uit gewoonte public void DigitalPet().
Pas op: Als je er void voor zet, is het geen constructor meer, maar een gewone methode! Hij wordt dan niet uitgevoerd bij new.

⚠️ Hoofdletters

De constructor moet precies heten als de class. Als je class DigitalPet heet, mag de constructor niet digitalPet (kleine letter) heten.

Samenvatting

  • De Constructor wordt aangeroepen op het moment dat je new gebruikt.
  • Gebruik het om data verplicht te stellen (zoals een naam) of standaardwaarden in te vullen (zoals start-energie).
  • Hij heeft geen return type en heet hetzelfde als de Class.
