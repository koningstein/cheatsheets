Object: Van Plan naar Realiteit

In het vorige hoofdstuk heb je het bestand DigitalPet.cs gemaakt. Dat was de Class (de blauwdruk). Maar met een blauwdruk alleen gebeurt er nog niks.

Nu gaan we die tekening gebruiken om echte huisdieren te maken in je programma. Deze 'levende' kopieÃ«n noemen we Objecten.

Wat is het verschil?

Dit is de kern van Object Oriented Programming. Zorg dat je dit verschil goed snapt.

De Vergelijking: De Fabriek

Denk terug aan de robot-tekening (de Class). Die tekening sturen we naar de fabriek.

  • De fabriek gebruikt die ene tekening om 1.000 robots te bouwen.
  • Elke robot is een Object.
  • Als ik Robot 1 rood verf, blijft Robot 2 gewoon grijs. Ze zijn onafhankelijk van elkaar.

Waar typ je de code?

Hier raken veel mensen in de war. Je hebt nu twee bestanden in je project. Het is belangrijk om ze goed uit elkaar te houden:

Bestand Doel
DigitalPet.cs Dit is de Blauwdruk. Hier beschrijf je alleen hoe een huisdier eruitziet. Je voert hier GEEN acties uit.
Program.cs Dit is de Fabriek (het startpunt). Hier gebruik je het commando new om de huisdieren daadwerkelijk te maken.

Stap voor Stap: Een Object maken

Ga naar het bestand Program.cs. Hier gaan we onze huisdieren tot leven wekken met het keyword new.

Het commando new zegt eigenlijk tegen C#: "Kijk naar de blauwdruk en reserveer geheugen voor een nieuw exemplaar."

// BESTAND: Program.cs

// 1. Maak een variabele 'dino' en stop er een NIEUWE DigitalPet in.
DigitalPet dino = new DigitalPet();

// 2. We kunnen er nog eentje maken!
// 'fluffy' is een compleet ander object in het geheugen.
DigitalPet fluffy = new DigitalPet();

Console.WriteLine("De huisdieren zijn aangemaakt!");

Wat gebeurt hier?

We hebben nu twee variabelen: dino en fluffy. Ze zijn gemaakt van dezelfde blauwdruk, maar ze leven hun eigen leven in het geheugen van de computer.

Hoe bekijk ik mijn Object?

Als je een variabele als int getal = 5; naar de console schrijft, zie je netjes "5". Maar bij objecten werkt dat anders.

// BESTAND: Program.cs

Console.WriteLine(dino);
// Output: "MijnGame.DigitalPet"

Waarom zie ik geen naam?
Een object is complex. C# weet niet wat jij wilt zien (de naam? de energie? de kleur?). Daarom toont hij standaard alleen het Type (de naam van de class).

(Later leren we hoe we dit kunnen aanpassen zodat je wel mooie info ziet.)

Help! Mijn scherm flitst weg

Als je op de groene Start-knop drukt, zie je misschien heel even een zwart schermpje en dan... weg! Dit is normaal, maar wel onhandig.

De computer is supersnel. Hij maakt de huisdieren, print de tekst en denkt dan: "Mijn takenlijst is klaar! Ik sluit af."

De Oplossing

We moeten het programma dwingen om even te wachten voordat het afsluit. Voeg helemaal onderaan je code deze regel toe:

// BESTAND: Program.cs

// ... (je andere code) ...

Console.WriteLine(dino);

// Wacht tot de gebruiker op Enter drukt
Console.ReadLine();

Nu blijft het scherm openstaan met een knipperende cursor. Pas als jij op Enter drukt, gaat hij verder en sluit hij af.

Veelgemaakte Fouten

Bij het maken van objecten gaat het soms mis. Let hier op:

1. De haakjes vergeten

Achter de naam van de class moeten ronde haakjes () staan. Dit is verplicht in C#.

DigitalPet bad = new DigitalPet;    // FOUT!
DigitalPet good = new DigitalPet(); // GOED!

2. Code in het verkeerde bestand

Je mag geen new DigitalPet() typen binnen in het bestand DigitalPet.cs. Een blauwdruk kan zichzelf niet bouwen!

Samenvatting: Waarom doen we dit?

Misschien denk je: "Waarom al die moeite met Classes en Objecten?"

Stel dat je een game maakt met 100 vijanden.

  • Zonder OOP: Je moet honderden losse variabelen maken (`enemy1_hp`, `enemy1_x`, `enemy2_hp`, etc.). Het wordt een chaos.
  • Met OOP: Je schrijft 1x de code voor Enemy. Daarna maak je in Program.cs met een simpele loop razendsnel 100 vijanden aan. De code blijft schoon en overzichtelijk.
