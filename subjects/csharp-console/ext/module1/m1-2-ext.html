Object: Van Plan naar Realiteit

In het vorige hoofdstuk hebben we de Class gemaakt. Dat was onze bouwtekening. Maar met een tekening kun je nog niet spelen. Nu gaan we die tekening gebruiken om echte dingen te maken. Die dingen noemen we Objecten.

Wat is het verschil?

Dit is de kern van Object Oriented Programming. Zorg dat je dit verschil goed snapt.

De Vergelijking: De Fabriek

Denk terug aan de robot-tekening (de Class). Die tekening sturen we naar de fabriek.

  • De fabriek gebruikt die ene tekening om 1.000 robots te bouwen.
  • Elke robot is een Object.
  • Als ik Robot 1 rood verf, blijft Robot 2 gewoon grijs. Ze zijn onafhankelijk van elkaar.

Objecten in onze Game

We hebben de DigitalPet class gemaakt. Dat is het idee van een huisdier. Nu gaan we echte huisdieren maken.

Stel dat we het spel starten. Je maakt twee huisdieren aan:

  1. Dino (een stoere T-Rex).
  2. Fluffy (een schattig konijn).

Dit zijn twee losse Objecten, gemaakt van dezelfde DigitalPet class.

Stap voor Stap: Een Object maken

Om van een class een object te maken, gebruiken we het keyword: new.

Het commando new zegt eigenlijk tegen C#: "Kijk naar de class en reserveer geheugen voor een nieuw exemplaar."

// 1. We hebben de class 'DigitalPet' al gemaakt in een ander bestand.

// 2. Nu maken we een nieuw object (instantiÃ«ren)
DigitalPet dino = new DigitalPet();

// 3. We kunnen er nog eentje maken!
DigitalPet fluffy = new DigitalPet();

Wat gebeurt hier?

We hebben nu twee variabelen: dino en fluffy. Ze lijken op elkaar omdat ze van dezelfde class komen, maar het zijn twee totaal verschillende dingen in het geheugen van de computer.

Veelgemaakte Fouten

Bij het maken van objecten gaat het soms mis. Let hier op:

1. De haakjes vergeten

Achter de naam van de class moeten ronde haakjes () staan. Dit zijn de haakjes van de Constructor (daar leren we later meer over).

DigitalPet bad = new DigitalPet;   // FOUT! C# snapt dit niet.
DigitalPet good = new DigitalPet(); // GOED!

2. Denken dat variabelen gekoppeld zijn

Beginners denken soms dat als je iets verandert aan dino, dat fluffy dan ook verandert. Dat is niet zo! Elk object heeft zijn eigen stukje geheugen.

Hoe bekijk ik mijn Object?

Als je een variabele als int getal = 5; naar de console schrijft, zie je netjes "5". Maar bij objecten werkt dat anders.

Console.WriteLine(dino);
// Output: "MyProject.DigitalPet"

Waarom? Een object is te complex. C# weet niet wat jij wilt zien (de naam? de energie?). Daarom toont hij standaard alleen het Type (de naam van de class).

(Later leren we hoe we dit kunnen aanpassen met de `ToString()` methode, zodat je wel mooie info ziet.)

Samenvatting: Waarom doen we dit?

Misschien denk je: "Waarom al die moeite met Classes en Objecten?"

Stel dat je een game maakt met 100 vijanden.

  • Zonder OOP: Je moet honderden losse variabelen maken (`enemy1_hp`, `enemy1_x`, `enemy2_hp`, etc.). Het wordt een chaos.
  • Met OOP: Je schrijft 1x de code voor Enemy. Daarna maak je met new razendsnel 100 vijanden aan. De code blijft schoon en overzichtelijk.
