Named Arguments: Duidelijke Code

In de vorige lessen heb je geleerd hoe je een constructor maakt. Zolang je maar één parameter hebt, is dat prima te lezen. Maar wat als je object ingewikkelder wordt? Wat als je 5 getallen moet meegeven bij het aanmaken?

Dan wordt je code onleesbaar en onhandig. C# heeft hier een geweldige oplossing voor: Named Arguments.

Het Probleem: De Getallenbrij

Stel je voor dat we onze DigitalPet uitbreiden. We voegen ook standaardwaarden toe (Optional Parameters), zodat je niet alles verplicht hoeft in te vullen.

// BESTAND: DigitalPet.cs
class DigitalPet
{
    public string Name { get; set; }
    public int Energy { get; set; }
    public int Hunger { get; set; }
    public bool IsSleeping { get; set; }

    // Een constructor met Default Values (Optionele parameters)
    // Energy is standaard 50, Hunger is standaard 0, Slapen is false.
    public DigitalPet(string name, int energy = 50, int hunger = 0, bool isSleeping = false)
    {
        this.Name = name;
        this.Energy = energy;
        this.Hunger = hunger;
        this.IsSleeping = isSleeping;
    }
}

Als we nu een dier willen maken dat toevallig al slaapt, maar wél de standaard energie en honger heeft, hebben we op de oude manier een probleem.

// OUDE MANIER:
// We willen alleen de laatste parameter (slapen) op 'true' zetten.
// Maar we MOETEN die middelste getallen ook invullen, anders snapt C# de volgorde niet.
DigitalPet dino = new DigitalPet("Rex", 50, 0, true);

Dit is onhandig. Je bent verplicht om waarden in te typen die je eigenlijk helemaal niet wilde veranderen, alleen maar om bij die laatste parameter te komen.

De Oplossing: Named Arguments

Met Named Arguments mag je de naam van de parameter erbij typen. Je zet de naam, gevolgd door een dubbele punt : en dan de waarde.

De Krachtcombinatie: Overslaan

Omdat je nu expliciet zegt "Dit getal is voor de honger", mag je andere parameters overslaan (mits die een default waarde hebben in de class).

Hoe werkt het in Program.cs?

Je kunt nu parameters "skippen". In het voorbeeld hieronder hoeven we energy en hunger niet in te vullen. C# pakt daar de standaardwaarden (50 en 0), en pakt voor isSleeping jouw waarde.

// BESTAND: Program.cs (Nieuwe manier)

// 1. We skippen de middelste parameters!
// C# snapt: "Naam is Rex, Slapen is true. De rest doe ik standaard."
DigitalPet dino = new DigitalPet("Rex", isSleeping: true);

// 2. Volledig invullen kan ook, en de volgorde maakt niet meer uit:
DigitalPet fluffy = new DigitalPet(
    hunger: 10,
    isSleeping: false,
    name: "Fluffy",
    energy: 80
);

Nu is je code niet alleen korter (je typt minder onnodige defaults), maar ook veel leesbaarder ("Self-Documenting").

Wanneer gebruik je dit?

Situatie Advies Voorbeeld
Weinig parameters Niet nodig Math.Max(5, 10);
Parameters overslaan Gebruiken! new Pet("Rex", isSleeping: true);
Veel parameters (Configuratie) Gebruiken! CreateUser(isAdmin: true, sendEmail: false);

⚠️ Typefouten

De naam die je typt voor de dubbele punt (bijvoorbeeld energy:) moet exact overeenkomen met de naam van de parameter in de constructor (camelCase). Als je in de class int levens hebt staan, werkt energy: 50 niet.

Samenvatting

  • Gebruik Named Arguments (naam: waarde) om parameters over te slaan die al een standaardwaarde hebben.
  • Je hoeft je niet aan de volgorde te houden.
  • Je code wordt leesbaarder omdat er labels bij de waarden staan.
