Methoden: De Acties

In de vorige lessen hebben we geleerd over de Class (de blauwdruk) en de Properties (de eigenschappen). We weten nu wat een object is (bijvoorbeeld: Dino de DigitalPet, met 100 energie).

Maar... een huisdier dat alleen maar stilstaat, is saai. In een spel willen we actie. Het dier moet kunnen lopen, eten, slapen en geluid maken.

Hiervoor gebruiken we Methoden.

Wat is een Methode?

Een methode is eigenlijk heel simpel: het is een functie, maar dan eentje die in een class zit. Het definieert het gedrag van een object.

De Vergelijking: Werkwoorden

Als je programmeren vergelijkt met taal, dan ziet het er zo uit:

  • Class: Het zelfstandig naamwoord (Het Huisdier).
  • Property: Het bijvoeglijk naamwoord (Het Energieke Huisdier).
  • Method: Het werkwoord (Het Huisdier blaft).

Deel 1: De Class (De Blauwdruk)

We gaan een methode toevoegen aan onze DigitalPet class. We willen dat het dier zich kan voorstellen.

We noemen deze functie GetDescription. Let op: we gebruiken hier return en geen Console.WriteLine. De methode geeft de tekst aan ons terug.

Maar hoe weet het dier zijn eigen naam? Hoe weet hij hoeveel energie hij heeft?

Het geheim van 'this'

Stel je voor dat Dino en Fluffy naast elkaar staan.
Als Dino zegt: "Ik heet Dino", dan heeft hij het over zichzelf.
Als Fluffy zegt: "Ik heet Fluffy", dan heeft zij het over haarzelf.

In C# is this het woordje voor "mezelf".
Met this.Name zegt het object: "Pak de naam die in MIJN eigenschappenlijst staat."

Hier is de code voor de Class:

class DigitalPet
{
    // De Eigenschappen (Wat BEN ik?)
    public string Name { get; set; }
    public int Energy { get; set; }

    // De Methode (Wat DOE ik?)
    // 'string' betekent: ik geef tekst terug als antwoord.
    public string GetDescription()
    {
        // We gebruiken 'this' om bij onze eigen data te komen.
        return "Ik ben " + this.Name + " en ik heb " + this.Energy + " energie.";
    }
}

Deel 2: Het Gebruik (Program.cs)

Nu we de blauwdruk hebben aangepast, kunnen alle huisdieren die we maken dit kunstje uitvoeren. We roepen de methode aan met de punt . gevolgd door de naam van de methode én ronde haakjes ().

// 1. Maak Dino (Object aanmaken)
DigitalPet pet1 = new DigitalPet();
pet1.Name = "Dino";
pet1.Energy = 100;

// 2. Maak Fluffy
DigitalPet pet2 = new DigitalPet();
pet2.Name = "Fluffy";
pet2.Energy = 50;

// 3. Roep de methode aan
// pet1 gebruikt ZIJN eigen naam en energie omdat we 'this' gebruikten.
string info1 = pet1.GetDescription();
Console.WriteLine(info1);
// Output: "Ik ben Dino en ik heb 100 energie."

// pet2 doet hetzelfde trucje met HAAR eigen data.
Console.WriteLine(pet2.GetDescription());
// Output: "Ik ben Fluffy en ik heb 50 energie."

Void vs Return

In het voorbeeld hierboven gaf de methode een antwoord terug (string). Soms wil je dat een methode gewoon iets doet en verder niks terugzegt. Dan gebruik je void.

public void Bark()
{
    Console.WriteLine("Woef!");
}
// Gebruik: pet1.Bark(); (Je hoeft niks op te vangen)

Wat kan er fout gaan? (Veelgemaakte Fouten)

⚠️ Vergeet de haakjes niet!

Een methode is een functie. Een functie heeft altijd haakjes nodig om te starten, ook als ze leeg zijn.

  • ❌ pet1.GetDescription; (C# denkt dat dit een property is, maar die bestaat niet)
  • ✅ pet1.GetDescription(); (Voer de functie uit!)

Samenvatting

  • Methoden zijn functies binnen een class. Ze bepalen het gedrag.
  • Gebruik this. binnen een methode om bij de eigenschappen van dat specifieke object te komen.
  • Gebruik een Return Type (zoals string of int) als je een antwoord wilt, of void als je alleen actie wilt.
  • Je roept een methode aan met .Naam().
