Getters: Data Formatteren

In de vorige lessen hebben we properties gemaakt waar je data in kunt stoppen (Set) en uit kunt halen (Get). Vaak is die data "rauw".

Bijvoorbeeld: Energy is gewoon het getal 50. Maar misschien wil je dit op het scherm tonen als een mooie tekst: "ðŸ”‹ Energie: 50 / 100".

Je kunt dit steeds in Program.cs aan elkaar plakken, maar het is veel slimmer om dit in de Class zelf te regelen.

Wat is een Custom Getter?

Normaal gesproken geeft een property direct de waarde terug die opgeslagen is. Bij een Custom Getter (aangepaste getter) voeren we eerst wat code uit voordat we het antwoord geven. We "formatteren" de data.

De Vergelijking: De Barista

Stel je voor dat je koffie bestelt.

  • De Opslag (Variabele): De koffiebonen en het water in de machine.
  • De Getter (De Barista): Jij vraagt om "Koffie". Je krijgt niet de losse bonen. De barista maalt ze, voegt water toe, doet er melk bij en geeft jou een kant-en-klaar product.

De Code: Hoe maak je het?

We maken een nieuwe property in onze class. Deze property slaat zelf niks op, maar kijkt naar de andere data (Naam en Energie) en maakt daar een mooi tekstje van.

Omdat we deze data alleen willen lezen (het is een resultaat), gebruiken we alleen een get. Geen set.

// BESTAND: DigitalPet.cs
class DigitalPet
{
    // De ruwe data (hier slaan we het echt op)
    public string Name { get; set; }
    public int Energy { get; set; }

    // De "Formatter" Property
    // Deze slaat niks op, maar geeft een mooi opgemaakt resultaat.
    public string EnergyDisplay
    {
        get
        {
            // We plakken tekst en data aan elkaar
            return "ðŸ”‹ " + this.Name + " - Level: " + this.Energy + "/100";
        }
    }
}

Het gebruik in Program.cs

In je programma merk je geen verschil tussen een "echte" variabele en onze "verzonnen" variabele.

// BESTAND: Program.cs
DigitalPet dino = new DigitalPet();
dino.Name = "Rex";
dino.Energy = 50;

// We vragen de mooi gemaakte tekst op
Console.WriteLine(dino.EnergyDisplay);

// Output: ðŸ”‹ Rex - Level: 50/100

Verdieping: Waarom een Property en geen Methode?

Je vraagt je misschien af: "Waarom moeilijk doen met een get-blok? Ik kan toch ook gewoon een methode maken?"

Dat klopt! Technisch doen ze hetzelfde (tekst teruggeven). Het verschil zit in hoe je de code gebruikt in je programma. Laten we ze naast elkaar leggen.

Manier A: De Methode (Zoals je gewend bent)

Dit is een actie. Omdat het een actie is, moet je verplicht haakjes () gebruiken.

// IN DE CLASS:
public string GetInfo()
{
    return "Naam: " + Name;
}

// IN PROGRAM.CS (Het gebruik):
Console.WriteLine(dino.GetInfo()); // <-- Haakjes verplicht!

Manier B: De Property (Met de 'get')

Dit vermomt zich als een variabele. Je gebruikt geen haakjes.

// IN DE CLASS:
public string Info
{
    get
    {
        return "Naam: " + Name;
    }
}

// IN PROGRAM.CS (Het gebruik):
Console.WriteLine(dino.Info); // <-- Lijkt net een variabele!

Het geheim van 'Get'

In C# betekent het woordje get eigenlijk: "Wanneer iemand deze variabele probeert te lezen, voer dan stiekem deze code uit."

Het zorgt ervoor dat je slimme code (logica/berekeningen) kunt verstoppen achter iets dat eruitziet als een simpele eigenschap.

Wanneer kies je wat? (De Vuistregel)

  • Is het een Actie? (Blaffen, Lopen, Opslaan, Berekenen)
    -> Gebruik een Methode met ().
    -> Voorbeeld: dino.Bark()

  • Is het Data/Informatie? (Naam, Leeftijd, Omschrijving)
    -> Gebruik een Property (Getter) zonder ().
    -> Voorbeeld: dino.EnergyDisplay

De 'Fat Arrow' (De Afkorting)

Programmeurs houden van korte code. Als een getter heel simpel is (zoals hierboven: alleen maar return ...), dan vinden we al die accolades { } en het woordje return te veel werk.

Hiervoor is de Fat Arrow (Dikke Pijl) => bedacht. Het betekent simpelweg: "Deze property is gelijk aan...".

De Lange Manier (Block Body) De Korte Manier (Expression Body) 
public string Info
{
    get
    {
        return "Naam: " + Name;
    }
}
 
public string Info => "Naam: " + Name;

// Geen 'get', geen 'return', geen accolades.
// De pijl wijst direct naar het resultaat.

Samenvatting

  • Gebruik een Getter om data mooi te maken (formatteren) voordat je het toont.
  • Gebruik get { ... } om code te vermommen als variabele.
  • Gebruik Properties voor informatie en Methoden voor acties.
  • De Fat Arrow => is een handige afkorting.
