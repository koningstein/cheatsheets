Module 4-4: De Grote Verhuizing

Je Program.cs staat bomvol. Dat is niet handig. In professionele software splitsen we taken op. We gaan de code verhuizen naar twee nieuwe Classes: Game (de manager) en Menu (de ober).

Stap 1: De Ober (Menu.cs)

We maken een class die verantwoordelijk is voor alle tekst op het scherm. Maak een nieuw bestand: Menu.cs.

We maken de methodes static. Dat betekent dat we ze direct kunnen gebruiken, zonder new Menu() te hoeven typen. 

using Figgle.Fonts;

namespace Tamagotchi;

public static class Menu
{
    // Titel tonen
    public static void ShowTitle()
    {
        Console.Clear();
        Console.WriteLine(FiggleFonts.Standard.Render("Dino Shelter"));
    }

    // Een vraag stellen en het antwoord teruggeven
    public static string AskString(string question)
    {
        Console.Write(question + " ");
        return Console.ReadLine() ?? "";
    }
}

Stap 2: De Manager (Game.cs)

Nu maken we de baas van de applicatie. Deze class beheert de lijst met dieren en de game-loop. Maak een nieuw bestand: Game.cs.

We beginnen met de lijst (nu private, want alleen de Game mag erbij) en de Start-methode. 

using Tamagotchi.Models;

namespace Tamagotchi;

public class Game
{
    // De lijst met dieren (Private field)
    private List<DigitalPet> _pets = new List<DigitalPet>();

    public void Start()
    {
        Menu.ShowTitle();
        bool appRunning = true;

        while (appRunning)
        {
            // Hier komt straks de logica
            Console.WriteLine("Game draait...");
            string choice = Menu.AskString("Toets Q om te stoppen:");

            if (choice.ToUpper() == "Q")
            {
                appRunning = false;
            }
        }
    }
}

Stap 3: Program.cs Opschonen

Nu we alles verplaatst hebben, mag Program.cs bijna leeg zijn. Dit bestand is alleen nog maar de "deur" van je applicatie. 

using Tamagotchi;

// 1. Maak de game aan
Game myGame = new Game();

// 2. Start de game
myGame.Start();
