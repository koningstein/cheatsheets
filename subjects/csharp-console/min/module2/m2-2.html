Namespaces & Using

Achternamen voor code

Als je meerdere bestanden hebt, wil je structuur. Hiervoor gebruiken we Namespaces. Je kunt dit zien als een achternaam voor je Class. Het voorkomt dat namen dubbel voorkomen en geeft aan bij welke 'familie' het bestand hoort.

1. Namespace instellen

Bovenaan je class-bestand geef je aan in welke namespace hij woont. We gebruiken de moderne, korte schrijfwijze (File Scoped Namespace) met een puntkomma.

// BESTAND: DigitalPet.cs

// Dit bestand hoort nu bij de familie 'Tamagotchi'
namespace Tamagotchi;

public class DigitalPet
{
    // ...
}

2. Het gevolg: Code is 'kwijt'

Omdat DigitalPet nu een 'achternaam' heeft gekregen, kan Program.cs hem niet meer zomaar vinden. Je krijgt rode strepen.

3. Using Statements (De oplossing)

Met het keyword using importeer je de namespace in een ander bestand. Je zegt eigenlijk: "Zoek ook in deze doos naar code."

// BESTAND: Program.cs

// Importeer de namespace
using Tamagotchi;

// Nu werkt het weer!
DigitalPet dino = new DigitalPet("Rex");

Waarom werkt Console.WriteLine wel?

Standaard C# code (zoals System.Console) werkt zonder using omdat Visual Studio stiekem op de achtergrond al Implicit Usings (automatische imports) heeft aangezet voor veelgebruikte Microsoft code. Jouw eigen code moet je wel zelf importeren.
