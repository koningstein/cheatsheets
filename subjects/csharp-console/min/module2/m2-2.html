Namespaces & Using

Orde in de chaos

Stel je voor dat je een class Console wil maken voor je game-console. Maar C# heeft zelf óók al een class Console (waarmee je WriteLine doet). Hoe weet C# welke je bedoelt?

Hiervoor gebruiken we Namespaces. Een namespace is als een achternaam voor je class.

Een Namespace definiëren

Je code staat bijna altijd in een namespace. Vaak is dit de naam van je project.

namespace MyTamagotchiGame
{
    class DigitalPet
    {
        // ...
    }
}

De volledige naam van je class is nu MyTamagotchiGame.DigitalPet.

Using Statements

Je hebt vast al bovenaan je bestanden regels gezien zoals using System;. Dit betekent: "Ik wil alle classes uit de doos 'System' kunnen gebruiken zonder steeds de achternaam te typen."

// ZONDER using:
System.Console.WriteLine("Hallo");

// MET using:
using System;

Console.WriteLine("Hallo"); // C# snapt nu dat je System.Console bedoelt

File Scoped Namespaces (Nieuw in C# 10)

Vroeger moest je hele bestand inspringen (indents) binnen de accolades van de namespace. Tegenwoordig mag het korter:

// Oude stijl:
namespace MyGame
{
    class Player { } // Alles springt in
}

// Nieuwe stijl (File Scoped):
namespace MyGame; // Puntkomma aan het eind

class Player { } // Geen extra indentatie nodig!
