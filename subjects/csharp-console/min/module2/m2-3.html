Mappenstructuur

Alles in één map?

In het begin staan al je bestanden (Program.cs, DigitalPet.cs) los in de hoofdmap van je project. Bij 3 bestanden is dat prima, maar bij 50 bestanden wordt het een rommeltje.

1. Eén class per bestand

De gouden regel in C# is: Stop elke class in zijn eigen bestand.

  • Class: DigitalPet -> Bestand: DigitalPet.cs
  • Class: GameManager -> Bestand: GameManager.cs

Zet nooit meerdere classes in één bestand, tenzij ze heel klein zijn en puur bij elkaar horen.

2. Mappen gebruiken (Models)

Een veelgebruikte conventie is om je data-objecten (de blauwdrukken zoals DigitalPet, Food, Player) in een map genaamd Models te zetten.

MyTamagotchiGame/
│
├── Program.cs          (Het startpunt)
├── MyTamagotchiGame.csproj
│
└── Models/             (Map)
    ├── DigitalPet.cs
    └── FoodItem.cs

Namespaces volgen de map

Als je een bestand in een map zet, verandert vaak ook de namespace. Visual Studio doet dit automatisch voor je.

// Bestand: Models/DigitalPet.cs
namespace MyTamagotchiGame.Models; // De namespace bevat nu ook .Models

public class DigitalPet
{
    // ...
}

Importeren in Program.cs

Omdat DigitalPet nu in een andere 'kamer' (namespace) woont, moet je in Program.cs aangeven dat je die kamer wilt gebruiken.

using MyTamagotchiGame.Models; // Importeer de map

DigitalPet dino = new DigitalPet("Dino");
