Variables Inspecteren

De staat van je programma

Het nut van pauzeren (break) is dat je kunt zien wat de waardes van je variabelen zijn op dat exacte moment. Waarom is Energy opeens 0? Waarom is Name null?

1. Hoveren

De simpelste manier: beweeg je muis over een variabele terwijl de code gepauzeerd is. Er verschijnt een pop-up met de huidige waarde. Je kunt zelfs objecten uitklappen om hun properties te zien.

2. Locals Window

Onderaan in Visual Studio (tijdens debuggen) zie je het tabblad Locals. Hier zie je automatisch alle variabelen die in de huidige methode bekend zijn.

Name        Value           Type
------------------------------------------
myPet       {DigitalPet}    DigitalPet
  > Name    "Dino"          string
  > Energy  50              int
i           0               int

3. Watch Window

Soms wil je een specifieke variabele in de gaten houden die niet lokaal is, of wil je een berekening zien. In het Watch venster kun je zelf expressies typen.

  • Klik in een lege regel in Watch.
  • Typ bijvoorbeeld: myPet.Energy < 10
  • Druk op Enter. Je ziet nu live of dit true of false is terwijl je door de code stapt (F10).

Waardes aanpassen

Pro Tip: Je kunt in het Locals of Watch venster ook waardes aanpassen! Dubbelklik op een waarde (bijv. 50) en maak er 100 van. Je programma gaat nu verder met die nieuwe waarde. Ideaal om edge-cases te testen zonder je code te herschrijven.
