De Call Stack

Wie riep wie aan?

Programma's zijn een kettingreactie. Methode A roept B aan, en B roept C aan. Als je in methode C zit, wil je soms weten: "Hoe ben ik hier gekomen?"

Dit zie je in de Call Stack (De Stapel).

1. De Code (Ketting maken)

Om een "stapel" te zien, laten we de ene methode de andere aanroepen. Pas je `Sleep` methode aan in `DigitalPet.cs`:

// IN BESTAND: DigitalPet.cs

public string Sleep()
{
    this.IsSleeping = true;
    this.Energy += 20;

    // KETTING: Sleep roept nu zelf ook weer een methode aan
    UpdateStatus();

    return "Zzz... (+20 Energie)";
}

// De helper methode
public void UpdateStatus()
{
    // Zet HIER je Breakpoint (F9) op de regel hieronder:
    this.Hunger += 5;
}

2. De Stack bekijken

  1. Zet een Breakpoint (F9) in de nieuwe methode UpdateStatus.
  2. Start het programma (F5).
  3. Als het programma stopt, kijk je rechtsonder naar het tabblad Call Stack.

Je ziet nu de geschiedenis van je programma (van boven naar beneden):

1. UpdateStatus   (Hier ben je nu)
2. Sleep          (Wacht op UpdateStatus)
3. Main           (Wacht op Sleep)

3. Tijdreizen (Groene Pijl)

Dubbelklik in de lijst op Main (of Sleep). Visual Studio springt terug naar die methode en laat met een Groene Pijl zien waar de code aan het wachten is. Zo kun je zien welke variabelen er op dat moment waren.
