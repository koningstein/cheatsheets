Debugging & Breakpoints

Code voorbereiden

Om te oefenen met debuggen, gebruiken we een methode die data teruggeeft. Voeg deze toe aan je project:

// 1. In Models/DigitalPet.cs
public string Sleep()
{
    this.Energy += 20;
    return "Zzz... Dat deed deugd! (+20 Energie)";
}
// 2. In Program.cs
Console.WriteLine(dino.Sleep()); // <-- Zet HIER je breakpoint (F9)

Debuggen in 3 stappen

1. Breakpoint zetten (F9)

Klik in de grijze rand naast de regel dino.Sleep();. De rode bol betekent dat de code daar stopt.

2. Starten (F5)

Het programma start en pauzeert bij de rode bol. De regel wordt Geel.

3. Step Into (F11)

Dit is de belangrijkste toets voor vandaag. We staan op een regel die naar een ander bestand verwijst.

  • Druk op F11.
  • Je ziet dat Visual Studio direct naar DigitalPet.cs springt.
  • Beweeg je muis over Energy om de waarde te zien veranderen.

Cheat Sheet

Toets Actie
F5 Continue: Stop met pauzeren en ren door.
F10 Step Over: Volgende regel (blijf in dit bestand).
F11 Step Into: Spring IN de methode (wissel van bestand).
