Breakpoints & Stepping

Wat is Debuggen?

Als je code niet doet wat je verwacht, kun je willekeurig dingen gaan veranderen en hopen dat het werkt. Of je kunt de code Debuggen. Dit betekent dat je de uitvoering van je programma pauzeert en "onder de motorkap" kijkt terwijl het draait.

1. Breakpoints (F9)

Een breakpoint is een stopbord voor je code. Klik in de grijze marge links naast een regel code (of druk op F9). Er verschijnt een rode bol.

Start je programma nu in Debug mode (F5). Zodra de computer bij die regel komt, bevriest het programma en licht de regel geel op.

2. Stepping (Stap voor stap)

Nu de code gepauzeerd is, heb jij de controle. Je kunt regel voor regel door je programma lopen.

De Knoppen

  • Step Over (F10): Voer de huidige regel uit en ga naar de volgende regel. (Als de regel een methode-aanroep is, wordt die methode in één keer uitgevoerd).
  • Step Into (F11): Als de huidige regel een methode-aanroep is, spring je in die methode om daar regel voor regel verder te gaan.
  • Step Out (Shift+F11): Maak de huidige methode snel af en pauzeer zodra je weer terug bent bij de aanroeper.

Continue (F5)

Heb je genoeg gezien? Druk weer op F5 om het programma gewoon verder te laten draaien tot het volgende breakpoint.
