Computed Properties

Soms heb je informatie nodig die je niet apart hoeft op te slaan, omdat je het kunt berekenen aan de hand van bestaande data. Dit noemen we Computed Properties of "Read-only Properties".

Stel we willen weten of ons huisdier moe is. We hoeven geen variabele bool IsTired bij te houden en steeds te updaten. We kunnen gewoon naar de Energy kijken.

class DigitalPet
{
    public int Energy { get; set; }

    // Deze property heeft ALLEEN een get
    public string Status
    {
        get
        {
            if (this.Energy > 50)
            {
                return "Happy";
            }
            return "Tired";
        }
    }
}

Geen Set?

Omdat deze property geen set heeft, kun je hem niet instellen. Proberen om pet.Status = "Happy" te doen, zal een foutmelding geven. De status is puur het gevolg van de energie.

Expression Body Members (=>)

In C# komen getters die maar één regel code bevatten heel vaak voor. Hiervoor is een kortere schrijfwijze: de Expression Body syntax (ook wel "fat arrow" genoemd).

// Korte syntax
public bool IsAlive => this.Energy > 0;

// Dit is precies hetzelfde als:
/*
public bool IsAlive
{
    get { return this.Energy > 0; }
}
*/

Wanneer gebruik je dit?

Gebruik getters voor data die direct afgeleid kan worden. Bijvoorbeeld:
- FullName (combineert Firstname en Lastname)
- Area (berekent width * height)
- CanAttack (checkt of cooldown 0 is)
