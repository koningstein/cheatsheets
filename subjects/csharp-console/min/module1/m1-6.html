Getters: Data Formatteren

Mooie data tonen

Soms wil je data niet "rauw" tonen (bijv. 50), maar mooi opgemaakt (bijv. "Level: 50/100"). Hiervoor gebruiken we een Custom Getter in de class.

De Code

Je maakt een property die alleen een get heeft. Deze slaat zelf niks op, maar berekent het resultaat.

// BESTAND: DigitalPet.cs
public string EnergyDisplay
{
    get
    {
        // Plak tekst en variabelen aan elkaar
        return "ðŸ”‹ " + this.Name + " - Level: " + this.Energy;
    }
}

Het Gebruik

In Program.cs gebruik je het alsof het een gewone variabele is (zonder haakjes).

// BESTAND: Program.cs
Console.WriteLine(dino.EnergyDisplay);
// Output: ðŸ”‹ Rex - Level: 50

Verschil: Property of Methode?

Je kunt dit ook met een methode doen. Wanneer kies je wat?

Type Gebruik Wanneer gebruiken?
Methode dino.GetInfo()
(Met haakjes)		 Bij een Actie (Werkwoord).
Bijv: Berekenen, Opslaan, Lopen.
Property dino.Info
(Zonder haakjes)		 Bij Informatie (Zelfstandig naamwoord).
Bijv: Naam, Status, Omschrijving.

De Fat Arrow (=>)

Als je getter heel kort is, mag je de Fat Arrow syntax gebruiken. Dit scheelt typwerk.

// Lange manier
public string Status { get { return "Energy: " + Energy; } }

// Korte manier (Doet precies hetzelfde)
public string Status => "Energy: " + Energy;
