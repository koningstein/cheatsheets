Named Arguments

Het probleem met parameters

Naarmate je applicatie groeit, krijg je vaak constructors of methodes met meerdere parameters. Stel je voor dat onze DigitalPet niet alleen een naam nodig heeft, maar ook een start-energie en een start-honger.

public DigitalPet(string name, int energy, int hunger)
{ ... }

Als je dit object aanmaakt, ziet de code er zo uit:

DigitalPet pet = new DigitalPet("Dino", 50, 10);

Maar wat betekenen die getallen? Is 50 de honger of de energie? Als je deze code over een maand terugleest, weet je dat waarschijnlijk niet meer.

De Oplossing: Named Arguments

C# biedt een geweldige feature om dit op te lossen: Named Arguments. Je mag bij het aanroepen van een functie of constructor de naam van de parameter erbij typen, gevolgd door een dubbele punt.

// Veel duidelijker:
DigitalPet pet = new DigitalPet(name: "Dino", energy: 50, hunger: 10);

Voordelen

Het gebruik van Named Arguments heeft twee grote voordelen:

  1. Leesbaarheid: Je ziet direct wat elk getal betekent.
  2. Volgorde: Omdat je de namen gebruikt, maakt de volgorde niet meer uit (hoewel het netjes is om de logische volgorde aan te houden).
// Dit werkt ook, omdat C# door de namen weet wat waar hoort:
DigitalPet pet = new DigitalPet(hunger: 10, name: "Dino", energy: 50);

Best Practice

Bij methodes met veel parameters (vooral getallen of bools) is het een goede gewoonte om Named Arguments te gebruiken. Het maakt je code "self-documenting".
