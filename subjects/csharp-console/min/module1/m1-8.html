Named Arguments

Flexibel aanroepen

Als een constructor veel parameters heeft, of parameters met standaardwaarden (defaults), is het onhandig om alles op volgorde in te vullen. Met Named Arguments geef je expliciet aan welke waarde bij welke parameter hoort.

De Class met Defaults

We geven de parameters in de constructor een standaardwaarde met het = teken. Deze zijn nu optioneel.

// BESTAND: DigitalPet.cs
class DigitalPet
{
    // ... properties ...

    // Constructor met DEFAULT waardes (= 50, = 0, = false)
    public DigitalPet(string name, int energy = 50, int hunger = 0, bool isSleeping = false)
    {
        this.Name = name;
        this.Energy = energy;
        this.Hunger = hunger;
        this.IsSleeping = isSleeping;
    }
}

Parameters overslaan

Stel, je wilt een dier maken dat slaapt (`isSleeping`), maar je vindt de standaard energie en honger prima. Zonder Named Arguments moet je alles invullen. Met Named Arguments kun je skippen.

// BESTAND: Program.cs

// Oude manier (Onhandig):
// Je MOET 50 en 0 invullen om bij 'true' te komen.
DigitalPet pet1 = new DigitalPet("Rex", 50, 0, true);

// Nieuwe manier (Named Arguments):
// Je skipt energy en hunger! C# pakt daar de defaults voor.
DigitalPet pet2 = new DigitalPet("Rex", isSleeping: true);

Leesbaarheid

Bij veel getallen of `true/false` waarden, zie je niet wat wat is. Named Arguments lossen dit op.

// Duidelijk (Volgorde maakt niet uit):
DigitalPet pet3 = new DigitalPet(
    hunger: 10,
    isSleeping: false,
    name: "Fluffy",
    energy: 80
);

Let op

De naam voor de dubbele punt (energy:) moet exact overeenkomen met de parameter in je constructor.
