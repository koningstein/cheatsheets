Van Class naar Object

In de vorige les heb je een class gemaakt (de blauwdruk). Maar met alleen een blauwdruk kun je nog niet spelen. Je moet de class tot leven wekken. Dit proces noemen we instantiëren. Het resultaat is een Object.

Analogie

De Class is de stempel.
Het Object is de afdruk op papier.

Met één stempel (DigitalPet class) kun je duizenden verschillende afdrukken (DigitalPet objecten) maken.

Het 'new' keyword

Om een nieuw object te maken, gebruiken we in C# het woord new. Je maakt eerst een variabele aan van het type van je class, en stopt daar vervolgens een nieuw object in.

// We maken een variabele 'myPet' van het type 'DigitalPet'
// En we stoppen er een NIEUWE instantie in.
DigitalPet myPet = new DigitalPet();

Meerdere objecten

Het, kracht van Object Oriented Programming is dat je meerdere, onafhankelijke objecten kunt maken van dezelfde class. Ze lijken op elkaar, maar zijn niet hetzelfde.

DigitalPet pet1 = new DigitalPet(); // Dit wordt Dino
DigitalPet pet2 = new DigitalPet(); // Dit wordt Fluffy

// pet1 en pet2 zijn compleet gescheiden objecten in het geheugen.

Objecten gebruiken

Zodra je een object hebt, kun je via de variabele (bijvoorbeeld pet1) toegang krijgen tot de inhoud. Daar gebruiken we de punt . voor.

Console.WriteLine(pet1); // Geeft standaard het type terug: "ProjectNaam.DigitalPet"
