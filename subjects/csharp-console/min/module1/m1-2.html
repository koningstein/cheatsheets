Van Class naar Object

In de vorige les heb je de class gemaakt (het bestand DigitalPet.cs). Dit is je blauwdruk. Om er mee te werken, moeten we de class tot leven wekken. Dit proces heet instantiëren en doe je in Program.cs.

Vergelijking

De Class is de stempel.
Het Object is de afdruk op papier.

Met één stempel (DigitalPet class) kun je duizenden verschillende afdrukken (DigitalPet objecten) maken.

Bestanden scheiden

Het is belangrijk om te weten waar je typt:

  • DigitalPet.cs: Hier beschrijf je hoe het dier eruitziet (De Class).
  • Program.cs: Hier maak je de dieren daadwerkelijk aan (Het Object).

Het 'new' keyword

Om een nieuw object te maken, gebruiken we het woord new. Je maakt eerst een variabele aan, en stopt daar een nieuw object in.

// IN BESTAND: Program.cs

// 1. Type (DigitalPet)
// 2. Naam (myPet)
// 3. Keyword (new)
DigitalPet myPet = new DigitalPet();

Meerdere objecten

De kracht van OOP is dat je meerdere, onafhankelijke objecten kunt maken van dezelfde class.

// IN BESTAND: Program.cs

DigitalPet dino = new DigitalPet(); // Dit wordt Dino
DigitalPet fluffy = new DigitalPet();// Dit wordt Fluffy

// pet1 en pet2 zijn compleet gescheiden in het geheugen.

Objecten gebruiken

Zodra je een object in een variabele hebt, kun je die variabele gebruiken.

Console.WriteLine(dino);
// Geeft standaard alleen het type terug: "MijnGame.DigitalPet"

Tip: Scherm open houden

Sluit het zwarte schermpje direct na het starten? Voeg deze regel toe aan het einde van je code om te wachten:

Console.ReadLine(); // Wacht tot je op Enter drukt
