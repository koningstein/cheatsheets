Les 3.2: Validatie & Logic

We verplaatsen validatielogica (zoals "maximaal 100 energie") naar de Property zelf met behulp van Backing Fields.

Full Properties

Om logica toe te voegen aan een property, splitsen we deze in twee delen:

  1. Backing Field (`private`): De variabele die de data echt opslaat.
  2. Property (`public`): De toegangspoort die controleert.

Stap 1: Backing Field toevoegen

Voeg in DigitalPet.cs een privaat veld toe boven de properties.

public class DigitalPet
{
    private int _energy; // Opslag voor energie

    // ...

Stap 2: Property ombouwen

Vervang de oude `Energy` property door deze volledige versie met validatie logic.

    public int Energy
    {
        get { return _energy; }
        private set
        {
            // 'value' is de nieuwe waarde die wordt aangeboden
            if (value < 0)
            {
                _energy = 0;
            }
            else if (value > 100)
            {
                _energy = 100;
            }
            else
            {
                _energy = value;
            }
        }
    }
Let op: Gebruik in de `set` altijd het veld `_energy`. Als je `Energy = ...` gebruikt, ontstaat er een oneindige lus (StackOverflow).

Stap 3: Refactoring

De methode SetEnergy is nu overbodig, omdat de property zelf de controle uitvoert.

  • Verwijder de methode public void SetEnergy(...) uit `DigitalPet.cs`.
  • In `Program.cs`: Verwijder aanroepen naar `SetEnergy`.

Stap 4: Testen

Test in Program.cs of de validatie werkt via de Constructor.

// Probeer 9999 energie te geven
DigitalPet cheater = new DigitalPet("Cheater", 9999);

Console.WriteLine(cheater.EnergyDisplay());
// Verwacht: "Level: 100/100" (automatisch gecorrigeerd)
