Les 3.4: Type Safety & Nullable Types

We voorkomen crashes door veilig om te gaan met ontbrekende data (`null`) en user input.

Het Concept

  • Type Safety: Variabelen hebben een vast type.
  • Null: De afwezigheid van waarde. Dit veroorzaakt crashes als je het niet checkt.
  • Nullable Type (`?`): Geeft expliciet aan dat een variabele leeg mag zijn (bijv. `string?`).

Stap 1: DigitalPet update

Pas de Rename methode aan zodat deze string? accepteert en checkt op null.

    // string? betekent: input mag tekst zijn OF null
    public string Rename(string? newName)
    {
        // IsNullOrWhiteSpace checkt op: null, lege tekst "", of spaties " "
        if (!string.IsNullOrWhiteSpace(newName))
        {
            this.Name = newName;
            return $"Naam gewijzigd naar: {this.Name}";
        }
        else
        {
            return "Ongeldige naam, oude naam behouden.";
        }
    }

Stap 2: Program.cs Interactie

We gebruiken Console.ReadLine() om input te vragen. Omdat de gebruiker niets kan invullen, is dit een risico.

Vervang de code in Program.cs:

DigitalPet dino = new DigitalPet("Rex");
Console.WriteLine($"Huidige naam: {dino.Name}");

Console.Write("Typ een nieuwe naam: ");

// Input lezen (kan null zijn)
string? input = Console.ReadLine();

// Veilige methode aanroepen
Console.WriteLine(dino.Rename(input));

Console.WriteLine(dino.GetDescription());
Console.ReadLine();
