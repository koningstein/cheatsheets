Les 3.1: Encapsulation & Properties

We beveiligen data door schrijftoegang te beperken. We gebruiken private set om directe wijzigingen van buitenaf te blokkeren.

Het Doel

Op dit moment kan Program.cs elke waarde direct aanpassen (`dino.Energy = -100`). Dit willen we voorkomen. Alleen de class zelf mag zijn interne data wijzigen.

Stap 1: Private Setters toevoegen

Pas in Models/DigitalPet.cs de properties aan. Voeg private toe voor de set.

public class DigitalPet
{
    // De buitenwereld mag lezen (get), maar niet schrijven (private set).
    public string Name { get; private set; }
    public int Energy { get; private set; }
    public int Hunger { get; private set; }
    public bool IsSleeping { get; private set; }

    // ...

Stap 2: Constructor Controle

De Constructor heeft toegang tot private setters omdat deze onderdeel is van de class. De onderstaande code in DigitalPet.cs blijft dus correct werken:

    public DigitalPet(string name, int energy = 50, int hunger = 0, bool isSleeping = false)
    {
        this.Name = name; // Dit is toegestaan binnen de class
        this.Energy = energy;
        this.Hunger = hunger;
        this.IsSleeping = isSleeping;
    }

Stap 3: Wijzigingsmethodes maken

Om de naam later toch te kunnen wijzigen, moeten we een publieke methode aanbieden.

    // Voeg deze methode toe aan DigitalPet.cs
    public void Rename(string newName)
    {
        this.Name = newName;
    }

Stap 4: Program.cs aanpassen

In Program.cs mag je properties niet meer direct toewijzen. Pas je code aan:

DigitalPet dino = new DigitalPet("Rex");

// ✅ LEZEN mag:
Console.WriteLine(dino.Name);

// ❌ SCHRIJVEN mag niet meer (verwijder dit soort regels):
// dino.Name = "Bob";

// ✅ GEBRUIK METHODES:
dino.Rename("Super Rex");
dino.Sleep();
