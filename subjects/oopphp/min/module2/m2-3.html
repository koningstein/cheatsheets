Wat is Composer?

Composer is een tool om externe code (packages) te beheren in PHP. Het is vergelijkbaar met npm voor JavaScript of pip voor Python.

Wat kan Composer voor je doen?

  • Packages installeren: je kunt externe libraries installeren met één commando
  • Autoloading: classes worden automatisch geladen volgens de PSR-4 standaard
  • Versies beheren: je beheert welke versies van packages je gebruikt
  • Updates: je kunt alle packages updaten met één commando

Composer installeren

Controleer eerst of Composer al geïnstalleerd is:

# Terminal / Command Prompt
composer --version

# Als dit een versie toont, is Composer geïnstalleerd!
# Anders: download Composer van https://getcomposer.org/

Installatie

  • Voor Windows: Download de installer van getcomposer.org en volg de wizard
  • Voor Mac/Linux: Volg de instructies op de Composer website

Composer initialiseren in je project

Ga naar je project map en start Composer:

# Terminal
cd /pad/naar/jouw/blackjack-project
composer init

Beantwoord de volgende vragen:
Package name: blackjack/game
Description: Een Blackjack-spel met objectgeoriënteerde PHP
Author: [Jouw naam en email]
Minimum Stability: [leeg laten - enter]
Package Type: project
License: [leeg laten - enter]
Dependencies: no
Dev dependencies: no
PSR-4 autoload: [enter]
Confirm: yes

composer.json bestand

Na het commando composer init maakt Composer een composer.json bestand aan:

{
    "name": "blackjack/game",
    "description": "Een Blackjack-spel met objectgeoriënteerde PHP",
    "type": "project",
    "autoload": {
        "psr-4": {
            "Blackjack\\": "src/"
        }
    },
    "require": {}
}

Belangrijk

Als Composer automatisch "Blackjack\\Rpggame\\" heeft toegevoegd, pas dit dan aan naar "Blackjack\\".

Autoloader genereren

Maak de autoloader bestanden aan met:

# Terminal
composer dump-autoload

# Dit maakt een vendor/ directory aan met de autoloader

Wat krijg je na composer init?

  • composer.json: dit bestand bevat de configuratie van je project en welke packages je gebruikt
  • composer.lock: dit bestand bevat de exacte versies van alle geïnstalleerde packages (komt na de eerste install)
  • vendor/: deze map bevat alle geïnstalleerde packages en de autoloader
  • vendor/autoload.php: dit is het autoloader bestand dat je moet includen in je code

.gitignore aanpassen

Voeg de vendor directory toe aan je .gitignore:

# .gitignore
vendor/

# De composer.lock blijft wel in git
# (deze houdt exact bij welke versies geïnstalleerd zijn)

Waarom vendor/ niet committen?

De vendor/ map kan groot worden en bevat code van externe packages. Andere developers kunnen deze map opnieuw aanmaken met het commando composer install.

Composer voordelen

  • PSR-4 standaard: dit is de standaard manier voor autoloading die veel PHP projecten gebruiken
  • Geen eigen autoloader nodig: Composer regelt het automatisch laden van je classes
  • Package management: je kunt libraries installeren zoals Symfony en Laravel components
  • Updates: je kunt alle packages updaten met één commando
