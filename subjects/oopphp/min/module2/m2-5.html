Samenvatting: Autoloading en Projectstructuur

In Module 2 heb je geleerd hoe je classes automatisch laadt en hoe je je PHP project professioneel organiseert met namespaces en Composer.

Autoloading Concepten

Concept Doel Syntaxis (Voorbeeld)
Simpele Autoload Classes automatisch laden zonder require_once. spl_autoload_register(function($class) {...})
Namespaces Je code organiseren in logische groepen en naamconflicten voorkomen. namespace Blackjack;
Use Statement Classes uit een namespace importeren. use Blackjack\Card;
Composer Tool om packages te beheren en professionele autoloading (PSR-4) te gebruiken. composer init
PSR-4 Standaard die bepaalt hoe namespaces en mappen met elkaar overeenkomen. "Blackjack\\": "src/"

Van Simpel naar Professioneel

De evolutie van class loading in PHP projecten:

Methode Voordelen Nadelen
Handmatig require_once Simpel, directe controle Veel herhalende code, onoverzichtelijk bij veel classes
Simpele autoloader Classes worden automatisch geladen, je schrijft minder code Geen ondersteuning voor namespaces, beperkte structuur
Autoloader met namespaces Betere organisatie van je code, naamconflicten worden voorkomen Je moet je eigen autoloader onderhouden
Composer met PSR-4 Dit is de standaard in de PHP wereld, je kunt packages beheren, het is professioneel Extra dependency (maar altijd de voorkeur!)

Best Practice

Gebruik altijd Composer met PSR-4 autoloading in je projecten. Plaats je classes in de src/ map, voeg vendor/ toe aan .gitignore, en laad de autoloader met require_once 'vendor/autoload.php';

Projectstructuur met Composer

blackjack-project/
├── src/
│   ├── Card.php         (namespace Blackjack;)
│   ├── Deck.php         (namespace Blackjack;)
│   └── Player.php       (namespace Blackjack;)
├── vendor/
│   └── autoload.php     (Composer autoloader)
├── index.php            (require_once 'vendor/autoload.php';)
├── composer.json        ("Blackjack\\": "src/")
└── .gitignore           (vendor/)

Bulk Setters vs Constructor

Een constructor kan geen waarden teruggeven en kan dus geen berichten terugsturen. Bulk setters wel. Dit is handig voor debugging en validatie: echo $card->setCard("Hearts", "Ace", "ace.png");
