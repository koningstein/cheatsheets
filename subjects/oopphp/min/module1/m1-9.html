Named Arguments

Wat zijn Named Arguments?

Named Arguments is een functie in PHP 8.0 waarmee je bij het aanroepen van een methode kunt aangeven welke parameter welke waarde krijgt. Je doet dit door de naam van de parameter erbij te zetten.

Voorbeeld: Kaartspel

Bij het aanmaken van een kaart heb je soms optionele eigenschappen:

  • faceUp: Is de kaart zichtbaar? (standaard true)
  • isDealt: Is de kaart uitgedeeld? (standaard false)

Met Named Arguments kun je alleen de parameters invullen die je wilt, zonder alle parameters in de juiste volgorde te zetten.

Waarom Named Arguments gebruiken?

  • Bij functies met veel parameters
  • Bij functies met optionele parameters
  • Wanneer je sommige parameters wilt overslaan terwijl je andere wilt instellen

Voorbeeld met standaardwaarden

Hier is een Card class met optionele parameters:

class Card
{
    public function __construct(
        public $suit,
        public $value,
        public $faceUp = true,
        public $isDealt = false
    ) {
    }
}

Zonder Named Arguments

Je moet alle argumenten in de juiste volgorde meegeven:

$card1 = new Card("Hearts", "10", true, true);
$card2 = new Card("Spades", "Ace", true, false);

Met Named Arguments

Je kunt parameters bij naam aangeven:

$card1 = new Card(
    suit: "Hearts",
    value: "10",
    faceUp: true,
    isDealt: true
);

// Je kunt ook alleen specifieke parameters wijzigen
$card2 = new Card(
    suit: "Spades",
    value: "Ace",
    isDealt: true  // faceUp behoudt standaardwaarde
);

Combinatie van positional en named arguments

Je kunt de eerste argumenten normaal meegeven en de laatste met naam:

// Eerste argumenten normaal, laatste met naam
$card3 = new Card("Diamonds", "King", isDealt: true);

Regels

  • Positional arguments moeten altijd vóór named arguments komen
  • Je kunt niet "tussenin overslaan" met positional arguments

Voordelen

  • Duidelijk: Je ziet meteen welke waarde bij welke parameter hoort
  • Flexibel: Je kunt parameters overslaan die een standaardwaarde hebben
  • Makkelijker: De volgorde van de argumenten is minder belangrijk
