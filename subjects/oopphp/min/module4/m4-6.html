Objecten in $_SESSION

Normaal gesproken wordt een PHP-object vernietigd zodra de pagina is geladen. Om objecten te onthouden (zoals een lijst met spelers), slaan we ze op in $_SESSION.

1. De Gouden Regel (Volgorde)

Omdat je objecten opslaat, moet PHP eerst weten hoe de Class eruitziet voordat hij de sessie opent. Doe je dit verkeerd, dan krijg je een foutmelding (__PHP_Incomplete_Class).

// 1. EERST de classes laden
require 'vendor/autoload.php';
use Blackjack\Game;

// 2. DAARNA pas de sessie starten
session_start();

2. Een Container Class maken

Sla liever niet allemaal losse objecten op in de sessie. Maak één "Hoofd Class" (bijv. Game of PlayerList) die een array van objecten bevat.

class PlayerList
{
    // Hierin bewaren we alle spelers
    private array $players = [];

    public function addPlayer(Player $player)
    {
        $this->players[] = $player;
    }

    public function getPlayers(): array
    {
        return $this->players;
    }
}

3. De Workflow (index.php)

In je hoofdbestand gebruik je altijd dit stappenplan:

// STAP A: Ophalen (of nieuw maken)
// Bestaat de lijst al in de sessie? Haal hem op.
// Zo niet (??), maak een nieuwe, lege lijst.
$myList = $_SESSION['playerList'] ?? new PlayerList();

// STAP B: Aanpassen
// Bijvoorbeeld: nieuwe speler toevoegen uit formulier
if ($_SERVER['REQUEST_METHOD'] === 'POST') {
    $p = new Player($_POST['name']);
    $myList->addPlayer($p);
}

// STAP C: Opslaan
// Zet de (gewijzigde) lijst terug in de sessie voor de volgende keer
$_SESSION['playerList'] = $myList;

Samenvatting

  • Volgorde: Autoload → session_start().
  • Opslaan: Stop objecten in een lijst/array in een class.
  • Cyclus: Ophalen uit sessie → Wijzigen → Terugzetten in sessie.
