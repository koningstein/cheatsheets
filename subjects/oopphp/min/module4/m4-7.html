Meerdere pagina's met $_GET

Routing met $_GET

Met één index.php bestand kun je meerdere pagina's maken door gebruik te maken van $_GET parameters. De gebruiker navigeert tussen pagina's via links met verschillende GET parameters.

Voorbeeld URLs:

  • index.php - Home pagina
  • index.php?page=createPlayer - Speler aanmaken
  • index.php?page=playerList - Lijst met spelers
  • index.php?page=viewPlayer&id=5 - Speler bekijken

Switch statement voor routing

Een switch statement is overzichtelijker dan veel if-elseif statements als je veel verschillende pagina's hebt. Je checkt de waarde van $page en voert voor elke case andere code uit.

$page = $_GET['page'] ?? 'home';

switch ($page) {
    case 'createPlayer':
        // Toon formulier
        $template->display('createPlayerForm.tpl');
        break;

    case 'playerList':
        // Toon lijst
        $players = $playerList->getPlayers();
        $template->assign('players', $players);
        $template->display('playerList.tpl');
        break;

    case 'viewPlayer':
        // Toon detail pagina
        // Code hier
        break;

    default:
        // Home pagina
        $template->display('home.tpl');
        break;
}

Belangrijke onderdelen:

  • switch ($page) - Check de waarde van $page
  • case 'createPlayer': - Als $page deze waarde heeft
  • break; - Stop en ga niet verder naar volgende cases
  • default: - Als geen enkele case matcht (fallback)

Vergeet break niet!

Zonder break; blijft PHP doorgaan met de volgende case. Dit heet "fall-through" en is bijna altijd een bug.

Meerdere GET parameters

Je kunt meerdere GET parameters combineren om specifieke data op te halen. Bijvoorbeeld een speler bekijken op basis van naam.

// URL: index.php?page=viewPlayer&name=John
$page = $_GET['page'] ?? 'home';

switch ($page) {
    case 'viewPlayer':
        // Haal naam parameter op
        $name = $_GET['name'] ?? null;

        if ($name !== null) {
            // Haal speler op met deze naam
            $player = $playerList->getPlayerByName($name);

            if ($player !== null) {
                $template->assign('player', $player);
                $template->display('playerDetail.tpl');
            } else {
                // Speler niet gevonden
                $template->assign('error', 'Player not found');
                $template->display('error.tpl');
            }
        }
        break;
}

Validatie is belangrijk

  • Check of de parameter bestaat (?? null)
  • Check of het object bestaat voordat je het gebruikt
  • Toon een foutmelding als de parameter ontbreekt of ongeldig is

Links maken naar andere pagina's

In je templates maak je links met GET parameters om naar andere pagina's te navigeren:

<!-- Link naar create pagina -->
<a href="index.php?page=createPlayer" class="btn btn-success">
    New Player
</a>

<!-- Link naar detail pagina met naam -->
<a href="index.php?page=viewPlayer&name={$player->getName()}" class="btn btn-primary">
    View Details
</a>

<!-- Link naar home (geen parameters) -->
<a href="index.php" class="btn btn-secondary">
    Home
</a>

Meerdere parameters combineren

  • Eerste parameter: ?page=viewPlayer
  • Tweede parameter: &name=John
  • Volledige URL: index.php?page=viewPlayer&name=John

Belangrijke punten

  • Gebruik switch statement voor routing met meerdere pagina's
  • Vergeet break; niet na elke case
  • Haal GET parameters op binnen de case waar je ze nodig hebt
  • Valideer GET parameters voordat je ze gebruikt
  • Gebruik default: voor fallback (meestal home pagina)
  • Meerdere parameters combineren met &
  • Check of object bestaat voordat je het gebruikt
