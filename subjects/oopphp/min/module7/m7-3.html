De Transactie: Van Static naar Object

De Brug Slaan

We hebben nu twee werelden:

  1. De Statische Wereld: De ChipShop (de etalage met fiches).
  2. De Object Wereld: De HumanPlayer (met een portemonnee en eigen fiches).

Een transactie is het moment waarop data van de ene wereld naar de andere verhuist. Dit vereist zorgvuldige validatie.

Stap 1: Het Verzoek (De Controller)

Wanneer de speler op "Koop Fiche" klikt, stuurt het formulier een ID (bijv. chip_id=1 voor een Rood Fiche) naar index.php.

// index.php
if ($action === 'buy_chip') {
    // 1. Haal de input op
    $chipId = $_POST['chip_id'];
    $player = $_SESSION['human_player'];

    // 2. Haal het product uit de STATIC shop
    // We hebben geen winkel-object nodig, de shop is er altijd!
    $chipToBuy = ChipShop::getChipById($chipId);

    // 3. Valideer en Voer uit
    if ($chipToBuy && $player->canAfford($chipToBuy->getPrice())) {

        // Transactie: Geld weg, Fiche erbij
        $player->pay($chipToBuy->getPrice());
        $player->addChip($chipToBuy);

        $message = "Aankoop geslaagd: " . $chipToBuy->getColor();
    } else {
        $error = "Niet genoeg saldo of ongeldig fiche!";
    }
}

Stap 2: Validatie in het Object

Het is de verantwoordelijkheid van de Speler (het object) om te weten of hij iets kan betalen. De Shop weet immers niet hoeveel geld de speler heeft.

class HumanPlayer
{
    private float $balance = 100.0;
    private array $chips = [];

    public function canAfford(float $amount): bool
    {
        return $this->balance >= $amount;
    }

    public function pay(float $amount): void
    {
        if ($this->canAfford($amount)) {
            $this->balance -= $amount;
        }
    }

    public function addChip(Chip $chip): void
    {
        // Voeg het fiche toe aan de inventaris van de speler
        $this->chips[] = $chip;
    }
}

Feedback naar de Gebruiker

Na de transactie is het belangrijk om de gebruiker te laten weten of het gelukt is.

  • Succes: Toon een groene melding ("Gekocht!") en update direct het saldo op het scherm.
  • Fout: Toon een rode melding ("Te weinig geld") zodat de speler snapt waarom er niks gebeurde.

Samenvattend

Bij een transactie komen alle OOP concepten samen:

  1. Je gebruikt een Static Method om data op te halen (ChipShop::getById).
  2. Je roept een Instance Method aan om de staat van een object te wijzigen ($player->pay).
  3. Je gebruikt de Controller om deze twee aan elkaar te knopen.
