Samenvatting: Static & Mock Databases

In Module 7 heb je geleerd om buiten de grenzen van individuele objecten te werken. Met het static keyword kun je data en functionaliteit delen over de hele applicatie, zonder dat je specifieke objecten hoeft aan te maken.

Static vs. Non-Static

Het belangrijkste concept is het verschil tussen de instantie (het object) en de blauwdruk (de class).

Kenmerk Object (Non-Static) Class (Static)
Keyword (geen) static
Verwijzing intern $this-> self::
Verwijzing extern $object->methode() ClassNaam::methode()
Doel Data van één specifiek ding (mijn hand, mijn score). Gedeelde data (de spelregels, de winkelvoorraad).

1. Static Methods (Hulpfuncties)

Gebruik static methods voor functionaliteit die altijd waar is, ongeacht de situatie. Vaak wordt dit gebruikt voor validatie of berekeningen.

class Card {
    // Statische lijst met regels
    private static array $suits = ['Hearts', 'Diamonds', 'Spades', 'Clubs'];

    // Statische validatie: Je hebt geen kaart-object nodig om dit te vragen
    public static function isValidSuit(string $suit): bool {
        return in_array($suit, self::$suits);
    }
}

// Aanroep:
if (Card::isValidSuit('Hearts')) { ... }

2. Static Properties (Mock Database)

Gebruik static properties om data centraal op te slaan. Dit is ideaal voor een "winkel" of configuratie, omdat je wilt dat elke speler dezelfde voorraad ziet.

class ChipShop {
    // Deze array bestaat maar 1 keer in het geheugen
    private static array $stock = [];

    public static function getStock(): array {
        // Vul de array als hij leeg is (Lazy Loading)
        if (empty(self::$stock)) {
            self::$stock = [new Chip('Red', 10), new Chip('Blue', 50)];
        }
        return self::$stock;
    }
}

3. De Transactie

De kracht van OOP komt naar voren wanneer je statische data combineert met objecten. Een transactie verplaatst data van de statische bron naar een specifiek object.

// Controller Logic (index.php)
$chip = ChipShop::getChipById(1); // Haal uit Static
$player = $_SESSION['player'];    // Haal Object op

if ($player->canAfford($chip->getPrice())) {
    $player->pay($chip->getPrice()); // Pas Object aan
    $player->addChip($chip);         // Verplaats data
}
