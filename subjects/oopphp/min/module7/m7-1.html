Static Methods: Validatie

Methoden zonder Objecten?

Tot nu toe riepen we methoden altijd aan op een object: $player->getScore(). Dit is logisch, want de score hoort bij die specifieke speler.

Maar sommige dingen zijn universeel. Neem de regels van Blackjack. Is "Harten" een geldige kaartkleur? Dat antwoord is altijd "Ja", ongeacht welke kaarten er op tafel liggen. Het slaat nergens op om eerst een new Card() aan te moeten maken, puur om te vragen of een kleur bestaat.

Het Static Keyword

Met het keyword static maak je eigenschappen en methoden die bij de Class horen, in plaats van bij een object. Je kunt ze direct gebruiken zonder new.

Voorbeeld: Kaart Validatie

We maken een statische lijst met geldige kleuren en een statische methode om te checken of een input geldig is.

class Card
{
    // Static property: Hoort bij de Blauwdruk, niet bij één kaart
    private static array $validSuits = ['Hearts', 'Diamonds', 'Spades', 'Clubs'];

    // Static method: Kan aangeroepen worden zonder kaart-object
    public static function isValidSuit(string $suit): bool
    {
        // Let op: we gebruiken self:: in plaats van $this->
        return in_array($suit, self::$validSuits);
    }
}

Aanroepen met Dubbele Punt (::)

Om een statische methode te gebruiken, gebruik je de naam van de class, gevolgd door :: (de Scope Resolution Operator).

// FOUT:
// $card = new Card();
// $card->isValidSuit('Hearts'); // Dit is onnodig en verwarrend

// GOED: Direct via de Class naam
if (Card::isValidSuit('Hearts')) {
    echo "Dit is een geldige kleur!";
} else {
    echo "Foutieve invoer!";
}

$this vs self::

Context Verwijzing Betekenis
Object (niet-static) $this-> Verwijst naar dit specifieke object (bijv. de kaart die ik nu vastheb).
Class (static) self:: Verwijst naar de code van de class zelf (de algemene regels).

Wanneer gebruik je Static?

Gebruik static methoden voor hulpfuncties die geen toegang nodig hebben tot de data van een specifiek object. Denk aan:

  • Validatie (Card::isValidSuit)
  • Berekeningen (Calculator::add)
  • Formattering (Formatter::formatMoney)
