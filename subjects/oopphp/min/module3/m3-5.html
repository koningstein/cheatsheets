Private Properties

Wat is Visibility?

Visibility (zichtbaarheid) is een concept in OOP dat bepaalt waar in de code de properties en methods van een class toegankelijk zijn. De twee belangrijkste niveaus zijn:

  • public: Overal toegankelijk.
  • private: Alleen toegankelijk binnen de class zelf waar het gedefinieerd is.

De noodzaak van `private`

Het gebruik van het trefwoord private voor properties is belangrijk voor Encapsulatie. Encapsulatie betekent dat de data van een object beschermd wordt tegen wijzigingen van buitenaf.

Bescherming van data

Zonder `private` kan de `score` direct en onlogisch worden gewijzigd:

$player->score = -10; // Mogelijk bij public property

Door `private` te gebruiken, is directe toegang verboden:

class Player
{
    private int $score = 0;
    // Fout als je probeert aan te roepen: $player->score = 21;
}

Getters en Setters

Getters (Lezen)

Omdat je een private property niet direct kunt lezen van buitenaf, heb je een Getter methode nodig. Getters beginnen meestal met get.

public function getScore(): int
{
    return $this->score;
}

Setters (Schrijven/Wijzigen)

Om een private property gecontroleerd te wijzigen, gebruik je een Setter methode. Setters beginnen meestal met set en kunnen controleren of de ingevoerde waarde klopt.

public function setScore(int $newScore): void
{
    // Validatie en controle
    if ($newScore < 0) {
        $newScore = 0;
    }
    if ($newScore > 21) {
        $newScore = 21; // Maximale score in Blackjack
    }
    $this->score = $newScore;
}

Wanneer gebruik je Getters/Setters?

Gebruik ze altijd voor private properties. Zelfs als een setter nu nog geen controle bevat, kun je deze later makkelijk toevoegen zonder de rest van de code aan te passen. Dit is het principe van Encapsulatie.
