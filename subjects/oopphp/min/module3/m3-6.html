Private Methoden

Waarom `private` bij methoden?

Net als bij properties, beschermt het trefwoord private methoden tegen aanroepen van buiten de class. Dit heeft twee belangrijke voordelen:

  • Verbergen van details: Ingewikkelde stappen die alleen intern nodig zijn, worden verborgen. Andere programmeurs hoeven alleen de publieke methoden te kennen.
  • Overzichtelijkheid: Je kunt lange, ingewikkelde publieke methoden opdelen in kleinere, leesbare private methoden.
class Game
{
    public function playRound(Player $player, Player $dealer): string
    {
        $this->dealInitialCards($player, $dealer);  // Private methode
        $this->playerTurn($player);                  // Private methode
        $this->dealerTurn($dealer);                  // Private methode

        return $this->determineWinner($player, $dealer);
    }

    // Deze methode kan alleen door playRound() worden aangeroepen
    private function dealInitialCards(Player $player, Player $dealer): void
    {
        // ... logica voor het uitdelen van kaarten ...
    }

    // Deze methode is ook niet extern aan te roepen
    private function playerTurn(Player $player): void
    {
        // ... logica voor de beurt van de speler ...
    }
}

Voordelen van Private Methoden

Makkelijker onderhoud

Stel je wilt de logica voor het uitdelen van kaarten wijzigen (dealInitialCards). Omdat deze private is, weet je zeker dat alleen de Game class hierdoor wordt be√Ønvloed. Je hoeft je geen zorgen te maken dat andere delen van de applicatie deze methode direct gebruiken.

Fout bij Externe Aanroep

Als je van buiten de class probeert een private methode aan te roepen, geeft PHP een fatale fout (Error):

$game = new Game();
// Dit veroorzaakt een fatale fout!
$game->dealInitialCards($player, $dealer);
