Samenvatting: Types en Contracten

Module 3 ging over het schrijven van betrouwbare en veilige code door types strikt te controleren en encapsulatie toe te passen.

Types en Contracten

Concept Doel Syntaxis (Voorbeeld)
Docblocks Uitleg van code, hulp voor je programma, extra type informatie (@param). /** @param string $name */
Parameter Type Hint Zorgt ervoor dat alleen het juiste datatype bij de methode komt. function (string $name)
Return Type Hint Zorgt ervoor dat de methode altijd het juiste datatype teruggeeft. function (...): int
Property Datatype Zorgt ervoor dat de property altijd het juiste datatype heeft. private ?int $score = null;
Objecten als Type Maakt duidelijk hoe classes met elkaar verbonden zijn. function (Deck $deck)

Visibility en Encapsulatie

Visibility bepaalt waar je properties en methods kunt gebruiken. Encapsulatie betekent dat je de data beschermt met private en toegang geeft via getters en setters.

Visibility Toegang Gebruik (Voorbeeld)
public Overal Methoden voor buitenaf: public function getName()
private Alleen binnen deze class Interne data: private int $score

Vuistregel

Maak properties standaard private. Geef toegang via Getters (get...) en Setters (set...). Gebruik private methoden om ingewikkelde interne code op te splitsen en te verbergen.
