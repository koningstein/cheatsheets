Type hints & Return types

Wat zijn type hints?

Type hints geven aan welk soort gegevens een parameter of return value moet hebben (bijvoorbeeld een getal of een tekst). Dit voorkomt fouten en maakt je code veiliger en duidelijker.

Waarom type hints gebruiken?

  • Veiligheid: PHP controleert automatisch of het juiste type wordt meegegeven
  • Duidelijkheid: Het is direct zichtbaar welk type verwacht wordt
  • Minder bugs: Fouten worden eerder ontdekt
  • Betere hulp van je programma: Autocomplete en het vinden van fouten werkt beter

Parameter type hints

Plaats het verwachte datatype vóór de parameter in de methode. Dit zorgt ervoor dat alleen de juiste invoer wordt toegestaan:

public function setName(string $name): void
{
    $this->name = $name;
}

public function addPoints(int $points): void
{
    $this->score += $points;
}

Return type hints

Plaats het verwachte datatype na de parameters met een dubbele punt (:) ervoor. Dit zorgt ervoor dat de methode altijd het juiste type teruggeeft:

public function getName(): string
{
    return $this->name;
}

public function getScore(): int
{
    return $this->score;
}

Nullable types

Gebruik een vraagteken (?) voor het datatype om aan te geven dat de parameter of return value het opgegeven type óf null mag zijn:

public function setAvatar(?string $avatar): void
{
    $this->avatar = $avatar;
}

public function getWinner(): ?Player
{
    return $this->winner; // kan null zijn
}

Void return type

Gebruik void als return type als een methode geen waarde teruggeeft:

public function resetHand(): void
{
    $this->hand = [];
    $this->score = 0;
}

Veiligheid van types

Als je een verkeerd type meegeeft, krijg je een TypeError (foutmelding):

$card = new Card();
$card->setCard("Hearts", 5, "hearts_5.png"); // ❌ Fout!
// Verwacht: setCard(string, int, string)
// Gegeven: setCard(string, int, string) - maar parameter 2 moet string zijn!

Best practice

Gebruik altijd type hints voor:

  • Alle methode parameters
  • Alle return types (behalve constructors)
  • Properties (zie volgende sectie)

Dit maakt je code robuuster en voorkomt veel bugs!
