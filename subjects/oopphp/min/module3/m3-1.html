Docblocks

Wat zijn docblocks?

Docblocks zijn speciale opmerkingen in je code die uitleggen wat een class of methode doet. Ze helpen andere programmeurs (en jouzelf later!) om de code te begrijpen. Ook krijg je betere hulp van je programma (zoals autocomplete).

Waarom docblocks gebruiken?

  • Betere uitleg van je code: Andere programmeurs (en jijzelf over een paar maanden) snappen meteen wat de code doet
  • Hulp van je programma: Je programma kan betere tips geven en fouten vinden
  • Professioneler: Het is een standaard in de PHP community

Class docblock

Gebruik een class docblock om uit te leggen wat de class doet. Hier zie je een voorbeeld bij de Card class:

/**
 * Represents a playing card in a Blackjack game
 */
class Card
{
    // class implementation
}

Method docblock

Een method docblock legt uit wat de methode doet en wat er terugkomt. Hier zie je een voorbeeld van een methode zonder parameters:

/**
 * Returns the full name of the card
 * @return string The full card name
 */
public function getCardName(): string
{
    return $this->value . " of " . $this->suit;
}

Docblock met parameters

Gebruik de @param tag om het type en de uitleg van een parameter op te schrijven. Hier zie je een voorbeeld bij de setSuit methode:

/**
 * Sets the card's suit
 * @param string $suit The suit to set
 * @return void
 */
public function setSuit(string $suit): void
{
    $this->suit = $suit;
}

Tip

Voeg altijd docblocks toe aan:

  • Elke class
  • Alle public en protected methods
  • Complexe private methods

Je programma kan vaak automatisch een docblock maken - typ /** en druk op Enter boven een method!
