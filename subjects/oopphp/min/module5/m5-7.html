Method Overriding & Polymorfisme

Gedrag Aanpassen

Soms voldoet de methode van de Parent class niet helemaal voor de Child class. De Dealer moet bijvoorbeeld anders spelen dan een HumanPlayer.

Voorbeeld: playTurn()

De Player class heeft een basisimplementatie, maar de Dealer doet iets totaal anders.

    }
}

Functionaliteit Uitbreiden met parent::

Soms wil je de originele code niet vervangen, maar uitbreiden. Je wilt doen wat de ouder doet, plus een beetje extra.

Hiervoor gebruik je parent::methodeNaam().

class Player {
    public function getSummary(): string {
        return "Score: " . $this->getScore();
    }
}

class Cheater extends Player {
    public function getSummary(): string {
        // Roep eerst de ouder aan
        $original = parent::getSummary();

        // Plak er iets achteraan
        return $original . " (I am cheating!)";
    }
}

Polymorfisme

Dit is een duur woord voor een simpel concept: Verschillende objecten op dezelfde manier behandelen.

Waarom is dit handig?

Stel je hebt een array met spelers. Sommige zijn mens, sommige zijn dealer, sommige zijn valsspelers. Omdat ze allemaal Player zijn en allemaal de methode playTurn() hebben, hoeft de Game class niet te weten wie wat is.

$players = [
    new HumanPlayer("John"),
    new Dealer(),
    new HumanPlayer("Jane")
];

foreach ($players as $player) {
    // Polymorfisme in actie:
    // We roepen playTurn() aan, maar de UITVOERING verschilt per object!
    $player->playTurn();
}

Samenvattend

  • Overriding: Een methode in de child class maken met dezelfde naam als in de parent.
  • parent::: De originele methode van de parent aanroepen binnen de override.
  • Polymorfisme: Code schrijven die werkt met de Parent class, maar die automatisch het juiste gedrag van de Child classes gebruikt.
