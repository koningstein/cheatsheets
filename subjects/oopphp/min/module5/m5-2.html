Splitsen van class

De Naïeve Oplossing: Kopiëren en Plakken

Om van de "God Class" af te komen, is de eerste gedachte vaak: "Ik maak gewoon twee losse bestanden: HumanPlayer.php en Dealer.php."

Je kopieert de oude Player code naar beide bestanden en verwijdert wat je niet nodig hebt. De Dealer heeft geen geld, dus dat haal je weg uit Dealer.php.

Het Resultaat: Twee bijna identieke classes

Kijk wat er gebeurt met de code. Beide classes hebben nog steeds kaarten nodig, moeten scores berekenen en kaarten ontvangen.

HumanPlayer.php

class HumanPlayer
{
    private array $hand = [];
    private int $balance = 1000;

    public function addCard(Card $card) {
        $this->hand[] = $card;
    }

    public function getScore() {
        // ... complexe berekening ...
    }
}

Dealer.php

class Dealer
{
    private array $hand = [];
    // Geen balance (dat is het verschil)

    public function addCard(Card $card) {
        $this->hand[] = $card; // DUPLICAAT!
    }

    public function getScore() {
        // ... complexe berekening ... // DUPLICAAT!
    }
}

Het probleem met duplicatie

Op het eerste gezicht lijkt het probleem opgelost: je hebt geen if-statements meer nodig voor de rollen. Maar je hebt een nieuw, groter probleem gecreëerd.

Onderhoudsnachtmerrie

Stel je ontdekt een bug in de getScore() methode (bijvoorbeeld: Azen worden verkeerd geteld). Je moet deze bug nu op twee plekken oplossen: in HumanPlayer.php én in Dealer.php. Vergeet je er één? Dan is je spel stuk.

DRY: Don't Repeat Yourself

Een van de belangrijkste principes in software development is DRY. Elke keer dat je code kopieert en plakt, moet er een alarmbel afgaan.

De oplossing: Inheritance

We hebben een manier nodig om de gemeenschappelijke code (kaarten, score berekenen) op één plek te definiëren, en alleen de verschillen (geld, spelregels) in de specifieke classes te zetten.
