Samenvatting: Inheritance

Module 5 stond in het teken van Inheritance (Overerving). Je hebt geleerd hoe je code duplicatie voorkomt en een logische hiërarchie van classes opzet. In plaats van alles in één "God Class" te stoppen, splits je functionaliteit op in Parents en Children.

Belangrijke Keywords

Keyword Doel Voorbeeld
extends Geeft aan dat een class erft van een andere class. class Dealer extends Player
protected Zichtbaarheid: Toegankelijk voor class én children. protected array $hand;
parent:: Verwijst naar de parent class (voor methodes/constructors). parent::__construct();

Visibility Levels

Bij inheritance is het kiezen van de juiste visibility essentieel voor veiligheid en flexibiliteit:

Visibility Eigen Class Child Class Buitenwereld Gebruik
private ✅ Ja ❌ Nee ❌ Nee Unieke data (bijv. $balance van HumanPlayer).
protected ✅ Ja ✅ Ja ❌ Nee Gedeelde data (bijv. $hand van Player).
public ✅ Ja ✅ Ja ✅ Ja Interactie (bijv. addCard()).

Constructors & Initialisatie

Wanneer een Child class een eigen constructor heeft, wordt de Parent constructor niet automatisch uitgevoerd. Je moet deze expliciet aanroepen.

class HumanPlayer extends Player
{
    private int $balance;

    public function __construct(string $name, int $balance)
    {
        // 1. Eerst de basis initialiseren (hand leegmaken)
        parent::__construct($name);

        // 2. Daarna specifieke eigenschappen instellen
        $this->balance = $balance;
    }
}

Method Overriding

Je kunt het gedrag van een parent-methode aanpassen (overriding) in een child class. Dit doe je door de methode opnieuw te definiëren met dezelfde naam.

class Dealer extends Player
{
    // Override: Zelfde naam als in Player, maar ander gedrag
    public function playTurn(): void
    {
        // Dealer volgt vaste regels: pakken tot 17
        while ($this->getScore() < 17) {
            $this->addCard(new Card());
        }
    }
}

Ontwerp Principes (DRY)

  • DRY (Don't Repeat Yourself): Voorkom dat je dezelfde code (zoals score berekenen) kopieert naar HumanPlayer én Dealer. Zet dit in de Parent class.
  • Encapsulatie: Gebruik protected zodat de Dealer bij zijn kaarten kan, maar de buitenwereld niet.
  • Polymorfisme: Omdat zowel Dealer als HumanPlayer een Player zijn, kun je ze in een lijst door elkaar gebruiken (bijv. foreach ($players as $p) { $p->getScore(); }).
