Method Overriding & Polymorfisme

In de vorige lessen hebben we geleerd hoe een Child class (zoals Dealer) alles erft van de Parent class (Player). Meestal is dat fijn, maar soms is het gedrag van de ouder niet precies wat we willen.

Soms is het kind eigenwijs en wil het dingen anders doen.

Deel 1: Method Overriding (Overschrijven)

Stel je voor dat de Player class een methode playTurn() heeft. Voor een menselijke speler betekent dit: wachten op knoppen indrukken.

Maar... de Dealer is geen mens. De Dealer is een computer die strikte regels volgt. Hij mag niet wachten of kiezen. Hij moet kaarten pakken tot hij 17 punten heeft.

De Regel

Als een Child class een methode maakt met precies dezelfde naam als de Parent, dan vervangt hij de oude methode. Dit noemen we Overriding.

Voorbeeld: Dealer vs Speler

class Player {
    public function playTurn() {
        return "Ik wacht op invoer van de speler...";
    }
}

class Dealer extends Player {
    // OVERRIDE: We gebruiken dezelfde naam, maar veranderen de inhoud!
    public function playTurn() {
        // De Dealer volgt zijn eigen regels (loop tot 17)
        while ($this->getScore() < 17) {
            $this->addCard($deck->draw());
        }
        return "Dealer is klaar met spelen.";
    }
}

Als je nu $dealer->playTurn() aanroept, wordt de nieuwe code uitgevoerd. De oude code van Player wordt genegeerd.

Deel 2: Functionaliteit Uitbreiden (parent::)

Soms wil je de oude code niet weggooien, maar er iets aan toevoegen. Je wilt doen wat de ouder doet, plus een beetje extra.

De Vergelijking: Het Broodje Kaas

Stel je vader (Parent) maakt een standaard broodje kaas. Jij (Child) wilt dat ook, maar jij doet er ook nog komkommer op.

  • Foute manier: Je pakt zelf brood, boter en kaas (dubbel werk) en doet er komkommer op.
  • Goede manier: Je vraagt je vader: "Maak een broodje kaas voor me" (parent::maakBroodje()) en daarna leg je er zelf komkommer op.

Met parent::methodeNaam() roep je de originele code van de ouder aan.

class Cheater extends Player {
    public function getSummary(): string {
        // Stap 1: Vraag de ouder om de standaard tekst
        $originalText = parent::getSummary();

        // Stap 2: Plak er je eigen tekst achteraan
        return $originalText . " (LET OP: Ik speel vals!)";
    }
}

Deel 3: Polymorfisme (Veelvormigheid)

Dit is een duur woord dat je veel punten oplevert bij Scrabble, maar het concept is eigenlijk heel simpel.

  • Poly = Veel
  • Morf = Vorm

Het betekent: Verschillende objecten op dezelfde manier behandelen.

De Vergelijking: De Universele Afstandsbediening

Stel je hebt een universele afstandsbediening met een "POWER" knop.

  • Richt je hem op de TV -> TV gaat aan.
  • Richt je hem op de Radio -> Radio gaat aan.
  • Richt je hem op de Lamp -> Lamp gaat aan.

Jij doet telkens hetzelfde (op de knop drukken), maar het apparaat reageert op zijn eigen manier. Dat is Polymorfisme!

Waarom is dit geweldig voor Blackjack?

In ons spel hebben we een lijst met spelers. Sommige zijn mensen, sommige zijn dealers, sommige zijn valsspelers. Zonder polymorfisme zou je dit moeten doen:

// FOUTE MANIER: Onhandige checks
foreach ($players as $p) {
    if ($p instanceof Dealer) {
        $p->playDealerTurn(); // Aparte naam verzonnen? Onhandig!
    } elseif ($p instanceof Human) {
        $p->playHumanTurn();
    }
}

Dankzij Polymorfisme (en Overriding) heten alle methodes gewoon playTurn(). Het maakt ons niet uit wie het is, zolang ze maar een beurt kunnen spelen.

// GOEDE MANIER: Polymorfisme
$players = [
    new HumanPlayer("Sjaak"),
    new Dealer(),
    new Cheater("Boefje")
];

foreach ($players as $player) {
    // We roepen op iedereen dezelfde methode aan.
    // PHP kijkt ZELF welk type object het is en pakt de juiste methode.
    // De Dealer gaat hitten tot 17, de Human wacht, de Cheater speelt vals.
    $player->playTurn();
}

Samenvatting

  • Overriding: Een methode in de child class dezelfde naam geven als in de parent om het gedrag te vervangen.
  • parent::: Gebruik dit binnen een override om de oude code óók uit te voeren (uitbreiden i.p.v. vervangen).
  • Polymorfisme: Hierdoor kun je een lijst met verschillende objecten (Dealer, Human) allemaal in één loop behandelen, omdat ze dezelfde methodenamen hebben.
