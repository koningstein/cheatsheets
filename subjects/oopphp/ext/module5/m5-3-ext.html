Inheritance: Parent en Child

In de vorige les zagen we dat we code duplicatie (kopiëren-plakken) absoluut willen voorkomen. We willen de code voor kaarten en scores maar op één plek schrijven.

De oplossing heet Inheritance (Overerving). Dit betekent dat een nieuwe class code mag "lenen" of "erven" van een bestaande class.

De Vergelijking: Een Familie

In PHP gebruiken we termen uit de familie om dit uit te leggen. Dit is niet toevallig, want het werkt precies zoals in de natuur.

  • De Parent (Ouder): De basis. Deze class heeft bepaalde eigenschappen (in code: methods en properties).
  • Het Child (Kind): De nieuwe class. Deze "erft" automatisch alle eigenschappen van de ouder bij het aanmaken.

Stel je voor dat je ouder (Parent) heel goed kan rekenen. Jij (Child) erft dat talent. Je hoeft niet zelf opnieuw te leren rekenen; je kunt het gewoon omdat je het hebt geërfd.

Andere namen, zelfde concept

Als je online zoekt of in andere programmeertalen (zoals Java of C#) werkt, zie je soms andere namen voor precies hetzelfde concept:

  • Parent Class wordt ook wel Super Class of Base Class genoemd.
  • Child Class wordt ook wel Sub Class of Derived Class genoemd.

In PHP gebruiken we meestal Parent en Child, omdat PHP letterlijk een keyword parent:: heeft.

Stap 1: De Parent maken (Player.php)

We beginnen met de basis. Hierin zetten we alleen de dingen die iedereen in het spel nodig heeft. Zowel de mens als de dealer heeft kaarten nodig, dus dat zetten we in de Parent.

// Player.php (De Parent)
class Player
{
    // 'protected' betekent: privé voor de buitenwereld,
    // maar mijn Children (Dealer/Human) mogen dit wel gebruiken.
    protected array $hand = [];

    public function addCard(Card $card)
    {
        $this->hand[] = $card;
    }

    public function getScore(): int
    {
        // ... code om punten te tellen ...
        return 21;
    }
}

Stap 2: Het Child maken (Dealer.php)

Nu gaan we de Dealer maken. In plaats van alle code opnieuw te typen, zeggen we tegen PHP: "De Dealer is een kind van Player".

Dit doen we met het keyword extends.

// Dealer.php (Het Child)
require_once 'Player.php';

// Met 'extends' erft de Dealer alles van Player
class Dealer extends Player
{
    // Kijk! Deze class is helemaal leeg!
    // Toch kan hij kaarten pakken en scores tellen,
    // want dat heeft hij van zijn Parent gekregen.
}

Stap 3: Het bewijs

Laten we testen of het werkt. We maken een nieuwe Dealer aan en kijken of hij kaarten kan aannemen.

$myDealer = new Dealer();

// De methode addCard staat niet in Dealer.php...
// Maar omdat hij 'extends Player' heeft, werkt het toch!
$myDealer->addCard(new Card('Harten', 'Aas'));

echo $myDealer->getScore(); // Werkt ook!

De "Is een..." test

Twijfel je of je Inheritance moet gebruiken? Doe de test. Inheritance mag je alleen gebruiken als je de zin "Child is een Parent" kloppend kunt maken.

  • Een Dealer is een Player? -> JA (Goed gebruik van extends)
  • Een Auto is een Wiel? -> NEE (Fout! Een auto heeft een wiel, maar is er geen. Gebruik hier geen extends.)

Stap 4: Het Child uitbreiden (HumanPlayer)

Kinderen zijn vaak meer dan alleen een kopie van hun ouders. Ze leren nieuwe dingen erbij. Dat geldt ook voor classes.

De HumanPlayer erft alles van Player (kaarten, score), maar voegt daar iets aan toe: Geld.

// HumanPlayer.php (Een ander Child)
class HumanPlayer extends Player
{
    // 1. Hij erft $hand, addCard() en getScore() automatisch van de Parent

    // 2. Hij voegt zijn eigen unieke eigenschappen toe
    private int $balance = 1000;

    public function placeBet(int $amount)
    {
        $this->balance -= $amount;
    }
}

Nu heb je het beste van twee werelden:

  • Dealer heeft geen geld (want dat staat niet in de Parent).
  • HumanPlayer heeft wel geld (want dat heeft hij zelf toegevoegd).
  • Beide kunnen kaarten pakken (want dat hebben ze geërfd).

Samenvatting

Term Betekenis
extends Het PHP keyword waarmee je aangeeft wie de Parent is.
Parent Class De basis (ook wel Super Class). Hier zet je de gedeelde code.
Child Class De erfgenaam (ook wel Sub Class). Deze neemt alles over en voegt specifieke dingen toe.
