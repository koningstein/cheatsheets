1. De Basis: Ouders en Kinderen

De kern van deze module was het sleutelwoord extends. Hiermee voorkom je dat je code dubbel schrijft (DRY - Don't Repeat Yourself).

Begrip Wat doet het? In Blackjack taal
Parent Class De basis. Bevat alles wat gedeeld wordt. Player (heeft kaarten en score).
Child Class De specialisatie. Erft alles van de Parent. Dealer (is een Player, maar met regels).
extends Legt de link tussen Ouder en Kind. class Dealer extends Player

2. Veiligheid: Visibility Levels

We hebben geleerd dat private soms té streng is (zelfs je kinderen mogen er niet bij). Daarom hebben we een nieuw niveau geïntroduceerd: protected.

Hieronder zie je precies wie waar bij mag. Dit schema is essentieel voor je toets en je projecten:

Visibility Eigen Class Child Class Buitenwereld Wanneer gebruiken?
private ✅ Ja ❌ Nee ❌ Nee Unieke data
Bijv. $balance (alleen HumanPlayer heeft geld).
protected ✅ Ja ✅ Ja ❌ Nee Gedeelde data
Bijv. $hand (Dealer moet zijn eigen kaarten kunnen lezen).
public ✅ Ja ✅ Ja ✅ Ja Interactie
Bijv. addCard() (het spel moet kaarten kunnen uitdelen).

3. De Startmotor: Constructors

Dit was een belangrijk struikelblok. Als een Child class een eigen __construct heeft, wordt de Parent "vergeten".

De Regel

Als je zelf een constructor schrijft in een Child class, ben je verplicht om parent::__construct() aan te roepen. Doe je dit niet? Dan wordt de basis (zoals de naam van de speler) nooit ingesteld en crasht je spel.

class HumanPlayer extends Player
{
    private int $balance;

    public function __construct(string $name, int $balance)
    {
        // STAP 1: Bel eerst de aannemer voor de fundering (Parent)
        parent::__construct($name);

        // STAP 2: Doe daarna de inrichting (Child)
        $this->balance = $balance;
    }
}

4. Eigenwijs Gedrag: Overriding

Soms wil een kind iets anders doen dan zijn ouders. In code heet dit Overriding. Je maakt een functie met dezelfde naam als in de parent, maar met andere code.

In Blackjack is de playTurn() methode hier het perfecte voorbeeld van:

  • Player (Basis): Weet niet wat hij moet doen (of wacht op input).
  • Dealer (Override): Heeft strikte regels (moet kaarten pakken tot 17).
class Dealer extends Player
{
    // Override: Zelfde naam, ander plan!
    public function playTurn(): void
    {
        while ($this->getScore() < 17) {
            $this->addCard(new Card());
        }
    }
}

5. De Grote Finale: Polymorfisme

Dit dure woord betekent eigenlijk: "Verschillende dingen op dezelfde manier behandelen".

Omdat zowel de Dealer als de HumanPlayer afstammen van Player, garandeer je dat ze allebei de functies getScore() en playTurn() hebben. Hierdoor kun je ze allemaal in één lijst stoppen en er doorheen lopen zonder te checken wie wat is.

$spelers = [
    new HumanPlayer("Sjaak", 1000),
    new Dealer("De Bank"),
    new HumanPlayer("Henk", 500)
];

foreach ($spelers as $p) {
    // Het maakt niet uit of $p een mens of dealer is.
    // We weten zeker dat hij playTurn() kan!
    $p->playTurn();
}
