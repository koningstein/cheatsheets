Class: De Blauwdruk

Welkom in de wereld van Object Oriented Programming (OOP). Voordat we gaan programmeren, moeten we eerst het allerbelangrijkste onderdeel begrijpen: de Class.

Wat is een Class precies?

Je kunt een class zien als een blauwdruk of een bouwtekening. Het is nog niet het "ding" zelf, maar het is de beschrijving van hoe het ding eruit moet zien en wat het moet kunnen.

De Vergelijking: De Architect

Stel, je bent een architect. Je tekent een bouwplan voor een huis. Op dat plan staat:

  • Het huis heeft 4 ramen.
  • Het huis heeft een rode voordeur.
  • In het huis kun je de lichten aan- en uitzetten.

Dit bouwplan is jouw Class. Je kunt in dat plan niet wonen. Maar, met dat ene bouwplan kun je wel 100 echte huizen bouwen in een straat. Die echte huizen noemen we later Objecten.

De Class in Blackjack

Laten we dit vertalen naar ons kaartspel. In Blackjack hebben we heel veel kaarten nodig (Harten Aas, Schoppen Boer, Ruiten Tien, etc.).

Het zou heel veel werk zijn om voor elke kaart losse code te schrijven. In plaats daarvan maken we één blauwdruk: de Card class.

In deze class schrijven we op wat elke kaart heeft:

  • Elke kaart heeft een Kleur (Suit), zoals Harten of Schoppen.
  • Elke kaart heeft een Waarde (Value), zoals Aas of 10.

Dankzij deze ene class weet ons programma straks precies wat een kaart is.

Hoe schrijf je een Class?

In PHP is het maken van een class heel simpel. Je gebruikt het woord class, gevolgd door de naam. Daarna gebruik je accolades { } om aan te geven waar de class begint en eindigt.

Hieronder zie je hoe we de blauwdruk voor een kaart maken:

<?php

class Card
{
    // Alles tussen deze accolades hoort bij de blauwdruk van de kaart.
    // Hier is het nu nog leeg, maar later zetten we hier eigenschappen neer.
}

Belangrijke regels (Best Practices)

Programmeurs hebben afspraken gemaakt zodat iedereen elkaars code kan lezen. Als je je hier niet aan houdt, kan je programma vastlopen of krijg je foutmeldingen.

1. De Naamgeving (PascalCase)

Geef je class altijd een duidelijke, Engelse naam. We gebruiken hiervoor PascalCase. Dat betekent:

  • De naam begint altijd met een Hoofdletter.
  • Als de naam uit meerdere woorden bestaat, begint elk nieuw woord ook met een Hoofdletter.
  • Gebruik geen spaties of underscores.
Goed (PascalCase) Fout Waarom?
Card card Moet met een hoofdletter beginnen.
BlackjackPlayer blackjack_player Geen underscores gebruiken in class namen.
GameManager gamemanager Elk nieuw woord moet een hoofdletter hebben.

2. De Bestandsnaam

Dit is een regel waar veel beginners de fout in gaan. De naam van het bestand moet precies hetzelfde zijn als de naam van de class.

Heb je een class die BlackjackPlayer heet? Dan moet je bestand BlackjackPlayer.php heten. Let hierbij heel goed op hoofdletters en kleine letters!

Waarom is dit zo streng?

PHP heeft slimme technieken (zoals 'Autoloading') om bestanden automatisch te vinden. Als jij in je code vraagt om de Card class, gaat PHP zoeken naar Card.php. Als je bestand card.php (kleine letter) heet, kan PHP het soms niet vinden en crasht je spel.

3. De Accolades (Structuur)

Om je code netjes te houden, zetten we de openings-accolade { altijd op een nieuwe regel, direct onder de naam van de class.

// ZO HOORT HET NETJES:
class Card
{
    // ...
}

// DIT DOEN WE LIEVER NIET (in PHP):
class Card {
   // ...
}

Stap voor Stap: Je eerste Class

Laten we samenvatten wat je moet doen om een class te maken voor onze Blackjack speler:

  1. Maak een nieuw bestand aan en noem het Player.php (Let op de hoofdletter P!).
  2. Open het bestand en begin met <?php.
  3. Typ het woord class gevolgd door de naam Player.
  4. Ga naar de volgende regel en typ {.
  5. Ga een paar regels naar beneden en typ }.
<?php

class Player
{
    // Nu heb je een geldige Player class!
}
