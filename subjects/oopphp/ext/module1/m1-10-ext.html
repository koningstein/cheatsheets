Samenvatting: Je Eerste Stappen

Gefeliciteerd! Je hebt Module 1 afgerond. Dit was misschien wel de belangrijkste module van de hele cursus. Je hebt namelijk een compleet nieuwe manier van denken geleerd: Object Oriented Programming (OOP).

In plaats van een lange lijst met losse commando's (zoals in beginners-PHP), denk je nu in "dingen" (objecten). Voordat we verder gaan, zetten we alles nog één keer rustig op een rijtje.

1. Het Fundament: Class vs. Object

Dit is het allerbelangrijkste concept. Als je dit snapt, snap je de helft van OOP.

De Kern

  • Class (De Blauwdruk): Dit is het bestand op je computer (bijv. Card.php). Hierin beschrijf je wat een kaart is en wat hij kan. Je hebt hier maar één bestand van.
  • Object (Het Ding): Dit is het resultaat als je de code draait. Met new Card() maak je een levende kopie in het geheugen. Je kunt duizenden objecten maken van één class.

2. Eigenschappen en Acties

Een object bestaat uit twee onderdelen:

  • Properties (Eigenschappen): De data. Voor een kaart zijn dit de kleur (`suit`) en de waarde (`value`). Dit zijn eigenlijk variabelen in een class.
  • Methoden (Acties): De functies. Wat kan de kaart doen? Zichzelf tonen (`show()`) of omdraaien (`flip()`).

3. De Constructor: Een vliegende start

Je hebt geleerd dat het onhandig is om een object "leeg" aan te maken en daarna regel voor regel te vullen. De Constructor (`__construct`) lost dit op. Het zorgt ervoor dat een object direct compleet is zodra het "geboren" wordt.

4. Veiligheid: Getters en Setters

Je hebt geleerd om niet zomaar iedereen toegang te geven tot je data (Encapsulation).

  • Setters (De Poortwachter): Controleren de input. "Is dit wel een geldige waarde?" voordat je het opslaat.
  • Getters (De Chef-kok): Maken de output netjes. "Ik formatteer het saldo even met een euroteken voordat ik het teruggeef."

Het Grote Plaatje: Alles samen

Hieronder zie je een compleet voorbeeld van een `Player` class voor Blackjack. Hierin komen alle onderwerpen uit deze module samen.

De Class (Player.php)

<?php

class Player
{
    // PROPERTIES: De eigenschappen van de speler
    // We gebruiken 'Constructor Promoted Properties' (de korte manier)
    public function __construct(
        public $name,           // De naam (bijv. "Sjaak")
        public $balance = 100   // Het saldo (standaard 100 euro)
    ) {}

    // METHODE: Een actie die de speler kan doen
    // In dit geval: geld inzetten
    public function bet($amount)
    {
        // $this verwijst naar het saldo van DEZE specifieke speler
        if ($amount > $this->balance) {
            // We gebruiken RETURN, geen echo!
            return "Fout: Je hebt niet genoeg geld!";
        }

        // Pas het saldo aan
        $this->balance = $this->balance - $amount;

        return "Inzet van €$amount geplaatst. Nieuw saldo: €{$this->balance}";
    }

    // GETTER: Een veilige manier om het saldo te bekijken
    public function getBalanceFormatted()
    {
        // We geven het netjes terug met een euroteken
        return "€ " . number_format($this->balance, 2);
    }
}

Het Gebruik (index.php)

// 1. OBJECT AANMAKEN (De blauwdruk wordt een huis)
// We gebruiken Named Arguments voor duidelijkheid
$player1 = new Player(name: "Sjaak", balance: 500);

// 2. METHODE GEBRUIKEN
// De class geeft tekst terug (return), wij tonen het (echo)
echo $player1->bet(50);
// Output: Inzet van €50 geplaatst...

echo "<br>";

// 3. GETTER GEBRUIKEN
echo "Huidig saldo: " . $player1->getBalanceFormatted();

De valkuilen (Let hier op!)

Tijdens het leren van OOP struikelen de meeste studenten over dezelfde dingen. Check deze lijst als je code niet werkt:

De Fout De Oplossing
$this->$naam Fout! Te veel dollartekens.
Gebruik $this->naam (zonder $ bij de property).
new Card Gebruik altijd haakjes: new Card().
Bestand heet card.php De bestandsnaam moet beginnen met een Hoofdletter: Card.php.
public function _construct Vergeet het tweede liggende streepje niet: __construct (twee keer underscore).
echo in een methode Gebruik return in een methode, zodat je de data nog kunt gebruiken in de rest van je code.

Wat nu?

Je hebt nu een stevige basis. Je weet hoe je objecten maakt en gebruikt. In de volgende modules gaan we dieper. We gaan leren hoe classes met elkaar kunnen samenwerken (Inheritance) en hoe we de toegang nog strenger kunnen beveiligen (Private/Protected).

Maar voor nu: wees trots! Je programmeert nu Object Georiënteerd.
