Object: Van Plan naar Realiteit

In het vorige hoofdstuk hebben we de Class gemaakt. Dat was onze bouwtekening. Maar aan een tekening alleen heb je niet zoveel: je kunt er niet mee kaarten. Nu gaan we die tekening gebruiken om echte dingen te maken. Die dingen noemen we Objecten.

Wat is het verschil?

Dit is het belangrijkste concept van OOP (Object Oriented Programming). Als je dit snapt, snap je de basis.

De Vergelijking: De Fabriek

Denk terug aan de architect. Hij heeft één tekening (de Class). Die tekening gaat naar de fabriek.

  • De fabriek gebruikt die ene tekening om 1.000 huizen te bouwen.
  • Elk huis is een Object.
  • Als ik mijn huis verf, verandert de kleur van jouw huis niet. Ze zijn onafhankelijk.

Objecten in Blackjack

In ons kaartspel hebben we de Card class gemaakt. Dat is het idee van een kaart. Nu gaan we echte kaarten maken.

Stel dat we een spel starten. De dealer deelt twee kaarten uit aan jou:

  1. Een Harten Aas.
  2. Een Schoppen Koning.

Dit zijn twee losse Objecten, gemaakt van dezelfde Card class.

Stap voor Stap: Een Object maken

Om van een class een object te maken, gebruiken we een speciaal woord in PHP: new.

Het commando new zegt eigenlijk tegen PHP: "Kijk naar de bouwtekening en bouw een nieuw exemplaar voor mij in het geheugen."

<?php

// 1. Eerst moet PHP de bouwtekening kennen
require 'Card.php';

// 2. Nu maken we een nieuw object (een kaart)
$myCard = new Card();

// 3. We kunnen er nog eentje maken!
$dealerCard = new Card();

Wat gebeurt hier?

We hebben nu twee variabelen: $myCard en $dealerCard. Ze lijken op elkaar omdat ze van dezelfde class komen, maar het zijn twee totaal verschillende dingen in het geheugen van de computer.

Wat kan er fout gaan? (Veelgemaakte Fouten)

Bij het maken van objecten zien we vaak dezelfde vergissingen:

1. De haakjes vergeten

Achter de naam van de class horen ronde haakjes (). Soms werkt het zonder, maar het is slechte code.

$fout = new Card;   // Niet netjes!
$goed = new Card(); // Zo hoort het.

2. Denken dat alles hetzelfde is

Beginners denken soms dat als je iets verandert aan $myCard, dat $dealerCard dan ook verandert. Dat is niet zo! Elk object staat op zichzelf.

Hoe bekijk ik mijn Object?

Vroeger gebruikte je misschien echo om een variabele op het scherm te tonen. Bij objecten werkt dat niet.

// DIT GEEFT EEN FOUTMELDING:
echo $myCard;
// Fout: "Object of class Card could not be converted to string"

Waarom? Een object is te complex. Het is geen simpele tekst, maar een verzameling van gegevens. Om in een object te kijken, gebruiken we var_dump().

Om het leesbaar te maken, zetten we er HTML <pre> tags omheen. Dit zorgt ervoor dat de tekst netjes wordt opgemaakt.

echo "<pre>";
var_dump($myCard);
echo "</pre>";

Je ziet dan op je scherm iets als:

object(Card)#1 (0) {
}

Dit betekent: "Dit is een object van het type Card, het is object nummer 1, en het is nog leeg."

Samenvatting: Waarom doen we dit?

Misschien denk je: "Waarom al die moeite met Classes en Objecten?"

Stel dat je een Blackjack spel maakt met 52 kaarten en 4 spelers.

  • Zonder OOP: Je moet honderden losse variabelen maken (`$kaart1_kleur`, `$kaart1_waarde`, `$speler1_naam`, etc.). Het wordt een chaos.
  • Met OOP: Je schrijft 1x de code voor Card en 1x de code voor Player. Daarna maak je met new razendsnel 52 kaarten en 4 spelers aan. Het is netter, sneller en minder foutgevoelig.
