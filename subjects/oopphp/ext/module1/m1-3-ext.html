Properties: De Eigenschappen

We hebben nu een Class (de blauwdruk) en een Object (het huis). Maar een kaal huis is saai. Een huis heeft eigenschappen: een kleur, een adres, en een aantal kamers.

In OOP noemen we deze eigenschappen Properties. Het zijn de gegevens die bij een object horen.

Wat zijn Properties precies?

Eigenlijk ken je dit concept al. Een property is niets anders dan een variabele, maar dan eentje die in een object zit.

De Vergelijking: Het Paspoort

Denk aan een paspoort. Elk paspoort ziet er qua indeling (Class) hetzelfde uit. Maar de inhoud verschilt per persoon.

  • Naam: "Jan Jansen"
  • Geboortedatum: "01-01-2000"
  • Lengte: "1.80m"

Deze invulvelden (Naam, Datum, Lengte) zijn de Properties.

Properties in Blackjack

Laten we kijken naar onze speelkaart. Wat maakt een kaart uniek? Wat moeten we onthouden van één kaart?

Voor onze Card class hebben we de volgende eigenschappen nodig:

  • suit (Kleur): Is het Harten, Schoppen, Ruiten of Klaver?
  • value (Waarde): Is het een Aas, een Koning, of een 7?
  • points (Punten): Hoeveel is deze kaart waard in het spel? (Bijv. 10 punten).

Hoe maak je een Property?

Je maakt properties aan in je Class bestand. Je zet ze bovenaan, direct na de opening {.

We gebruiken het woord public. Dit betekent: "Iedereen mag deze eigenschap zien en aanpassen".

class Card
{
    // Hier definiëren we de lege velden (de properties)
    public $suit;
    public $value;
    public $points;
}

Waarden geven en uitlezen

Nu we de velden hebben gemaakt in de blauwdruk, kunnen we ze gaan invullen bij onze objecten. Hiervoor gebruiken we een speciaal teken: het pijltje ->.

Het pijltje (->)

Je kunt dit pijltje lezen als: "Ga naar binnen bij dit object, en pak...".

Hieronder maken we twee verschillende kaarten aan:

<?php

// 1. Maak een Harten Aas
$card1 = new Card();
$card1->suit = "Hearts"; // Vul 'Hearts' in bij suit
$card1->value = "Ace";
$card1->points = 11;

// 2. Maak een Schoppen Tien
$card2 = new Card();
$card2->suit = "Spades";
$card2->value = "10";
$card2->points = 10;

// Laten we ze eens bekijken
echo "Kaart 1 is een " . $card1->suit . " " . $card1->value;
// Output: Kaart 1 is een Hearts Ace

Belangrijk: Onafhankelijkheid

Kijk goed naar de code hierboven. We hebben $card1 en $card2. Als ik de suit van kaart 1 verander, blijft kaart 2 gewoon zoals hij was. Elk object heeft zijn eigen set properties.

Wat kan er fout gaan? (Veelgemaakte Fouten)

Bij properties zien we vaak verwarring over het dollarteken $.

1. De dubbele dollarteken fout

Wanneer je een property aanmaakt in de class, gebruik je een $.
Wanneer je een property gebruikt met een pijltje, gebruik je GEEN $.

Actie Code Correct?
Aanmaken (in Class) public $suit; ✅ Ja
Gebruiken (bij Object) $card->suit = "Hearts"; ✅ Ja
Gebruiken (Fout!) $card->$suit = "Hearts"; ❌ Nee (teveel dollars)

Waarom doen we dit?

Misschien denk je: "Ik kan dit toch ook met een array doen?"

$kaartArray = ['Hearts', 'Ace', 11];

Dat kan, maar het is onhandig. Je moet onthouden dat nummer 0 de kleur is en nummer 1 de waarde. Bij een object zeg je gewoon $card->suit. Dat is veel duidelijker te lezen, zeker als je spel groter en complexer wordt.
