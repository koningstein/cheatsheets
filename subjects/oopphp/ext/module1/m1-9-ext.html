Named Arguments: Duidelijke Labels

We hebben nu gezien hoe je objecten maakt met de Constructor. Als je class maar twee eigenschappen heeft (Kleur en Waarde), is dat simpel. Maar wat als je class ingewikkelder wordt? Wat als je 5, 6 of 10 instellingen hebt?

Dan wordt het aanmaken van een object een grote puzzel. PHP 8.0 heeft hiervoor een oplossing: Named Arguments.

Het probleem: De "Blinddoek" code

Laten we kijken naar een voorbeeld zonder Named Arguments. Stel, we hebben een kaart die niet alleen een kleur en waarde heeft, maar ook:

  • Ligt hij open of dicht? ($faceUp)
  • Is hij al uitgedeeld? ($isDealt)
  • Is het een joker? ($isJoker)

Als je zo'n kaart maakt, ziet dat er zo uit:

$card = new Card("Harten", "Aas", true, false, false);

Kijk naar die laatste drie waarden: true, false, false. Als jij deze code over drie maanden terugleest, weet jij dan nog wat die eerste true betekent? Waarschijnlijk niet. Je moet dan in de class gaan zoeken om het te snappen.

De Oplossing: Labels plakken

Met Named Arguments mag je de naam van de parameter erbij schrijven. Je plakt als het ware een labeltje op de waarde.

De Vergelijking: Het Formulier

Stel je moet een formulier invullen.

  • Oude manier (Positional): Je krijgt een wit papier met vijf lege vakjes. Je moet zelf onthouden: "Eerste vakje is naam, tweede is adres, derde is leeftijd...". Eén foutje en alles schuift op.
  • Nieuwe manier (Named): Bij elk vakje staat duidelijk: "Naam: ...", "Adres: ...". Je kunt ze niet meer per ongeluk verwisselen.

Hoe werkt het?

Je gebruikt de naam van de variabele uit je constructor, gevolgd door een dubbele punt :.

// De Constructor in de Class (met standaardwaardes)
class Card
{
    public function __construct(
        public $suit,
        public $value,
        public $faceUp = false,  // Standaard ligt een kaart dicht
        public $isDealt = false  // Standaard zit hij in het deck
    ) {}
}

// Het aanmaken met Named Arguments
$myCard = new Card(
    suit: "Harten",
    value: "Aas",
    faceUp: true,   // Duidelijk! Hij ligt open.
    isDealt: true   // Duidelijk! Hij is gedeeld.
);

Waarom is dit zo handig?

1. Volgorde maakt niet meer uit

Omdat je er labels bij zet, hoef je niet meer de exacte volgorde te onthouden. PHP zoekt zelf de juiste plek.

// Dit mag gewoon!
$myCard = new Card(
    value: "Aas",
    suit: "Harten"
);

2. Dingen overslaan (Standaardwaardes)

Dit is de grootste kracht. Stel dat je constructor 10 opties heeft, maar de meeste hebben een prima standaardwaarde (default value). Je wilt alleen de allerlaatste optie aanpassen.

Zonder Named Arguments moet je alle tussenliggende waardes ook invullen, anders komt je laatste waarde op de verkeerde plek:

// Pff... ik moet die 'false' invullen, alleen maar om bij de laatste 'true' te komen
$card = new Card("Harten", "Aas", false, true);

Met Named Arguments pak je alleen wat je nodig hebt:

// Ik sla 'faceUp' over, want de standaardwaarde (false) is prima.
// Ik stel alleen 'isDealt' in.
$card = new Card(
    suit: "Harten",
    value: "Aas",
    isDealt: true
);

Wat kan er fout gaan? (Veelgemaakte Fouten)

1. Verkeerde naam gebruiken

De naam vóór de dubbele punt moet precies overeenkomen met de variabele in de `__construct` functie.

// In Class: public $suit
new Card(kleur: "Harten"); // FOUT! PHP kent 'kleur' niet, alleen 'suit'.

2. Mixen (Positional en Named)

Je mag de oude en nieuwe manier mixen, maar er is één strenge regel: De oude manier (zonder naam) moet altijd eerst.

// GOED: Eerst normaal, dan met naam
new Card("Harten", "Aas", isDealt: true);

// FOUT: Eerst met naam, dan normaal
new Card(suit: "Harten", "Aas"); // PHP raakt in de war.

Stap voor Stap: Wanneer gebruik je dit?

  1. Kijk naar je constructor. Heeft hij meer dan 3 parameters? -> Gebruik Named Arguments.
  2. Zitten er true of false waarden tussen? -> Gebruik Named Arguments (anders weet je later niet wat true betekent).
  3. Wil je optionele instellingen overslaan? -> Gebruik Named Arguments.
