Getters: De Presentatoren

In de vorige les hebben we de Setter besproken. Dat was de poortwachter die controleerde wat er in het object kwam. Nu gaan we kijken naar de uitgang: hoe halen we informatie er weer uit?

Natuurlijk kun je direct echo $card->suit; doen. Maar vaak wil je de data nog even netjes maken voordat je het aan de gebruiker laat zien. Daarvoor gebruiken we een Getter.

Wat is een Getter?

Een getter is een methode (functie) die als doel heeft: informatie uit het object ophalen en teruggeven aan jou. De naam begint bijna altijd met get.

De Vergelijking: De Chef-kok

Stel je voor dat je in een restaurant zit.

  • Zonder Getter (Direct): Je loopt de keuken in, pakt een rauwe aardappel en een stuk vlees uit de koelkast. Je hebt wel eten, maar het ziet er niet uit en het smaakt nergens naar.
  • Met Getter (Via de Chef): Je vraagt aan de chef: "Mag ik het hoofdgerecht?". De chef pakt de ingrediënten, kookt ze, legt ze netjes op een bord met wat garnering en geeft het aan jou.

De Getter is de chef. Hij pakt de ruwe data (de ingrediënten) en maakt er iets moois van voor de gebruiker.

Waarom doen we dit?

Waarom zou je moeilijk doen met een functie als je ook gewoon de variabele kunt pakken? Hier zijn drie goede redenen:

1. Opmaak (Formatting)

Stel dat we in onze database de kaartwaarde opslaan als kleine letters: "ace", "king", "queen". Maar op het scherm willen we een hoofdletter zien. De getter kan dit fixen zonder dat we de originele data hoeven aan te passen.

class Card
{
    public $value; // Hierin zit "ace"

    public function getValue()
    {
        // ucfirst() is een PHP functie die de eerste letter een hoofdletter maakt
        return ucfirst($this->value);
    }
}

2. Data combineren (Derived Properties)

Soms wil je informatie combineren. Bijvoorbeeld de volledige naam van de kaart ("Ace of Spades"). We hoeven hier geen aparte variabele voor te maken; de getter kan dit 'live' voor ons berekenen.

public function getFullName()
{
    // Plak de waardes aan elkaar met een stukje tekst ertussen
    return $this->value . " of " . $this->suit;
}

3. Toekomstige Bescherming

Net als bij Setters, bereiden we ons voor op de toekomst. Als we straks properties private maken (verborgen), dan moet je wel een getter gebruiken om de waarde te kunnen zien. Het is goed om dit nu alvast te leren.

Stap voor Stap: Getter en Setter samen

Laten we nu het complete plaatje bouwen voor ons Blackjack spel. We willen een kaart waarbij we de kleur (suit) instellen via een setter, en ophalen via een getter.

class Card
{
    public $suit; // De eigenschap (bijv. "Hearts")

    // DE SETTER (Input)
    // Controleert of de invoer geldig is
    public function setSuit($newSuit)
    {
        $validSuits = ["Hearts", "Diamonds", "Clubs", "Spades"];

        // Als de suit NIET in het lijstje staat...
        if (!in_array($newSuit, $validSuits)) {
            return "Fout: Dit is geen geldige kaartkleur!";
        }

        // Als het wel goed is, slaan we het op
        $this->suit = $newSuit;
        return "Gelukt: Kleur is opgeslagen.";
    }

    // DE GETTER (Output)
    // Geeft de waarde netjes terug, in HOOFDLETTERS (als voorbeeld)
    public function getSuit()
    {
        // strtoupper maakt alles hoofdletters: HEARTS
        return strtoupper($this->suit);
    }
}

De Code in Actie

Laten we kijken wat er gebeurt als we dit gebruiken:

$myCard = new Card();

// 1. We gebruiken de SETTER om data erin te stoppen
echo $myCard->setSuit("Hearts");
// Output: Gelukt: Kleur is opgeslagen.

echo "<br>";

// 2. We gebruiken de GETTER om data eruit te halen
echo "De kaart is: " . $myCard->getSuit();
// Output: De kaart is: HEARTS

Wat kan er fout gaan? (Veelgemaakte Fouten)

1. Echo in plaats van Return

Een veelgemaakte fout is om echo te gebruiken in een getter.

// FOUT:
public function getValue() {
    echo $this->value;
}

// GOED:
public function getValue() {
    return $this->value;
}

Waarom? Als je echo gebruikt, wordt de tekst direct op het scherm geknald. Je kunt er dan niks meer mee doen (zoals opslaan in een database of in een HTML-titel zetten). Gebruik altijd return, dan bepaal je buiten de functie zelf wat je met de tekst doet.

Samenvatting

  • Getters halen data uit een object.
  • Ze geven je de kans om data te formatteren (mooier maken) of te combineren.
  • Gebruik altijd return in een getter, geen echo.
