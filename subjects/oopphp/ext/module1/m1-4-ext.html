Methoden: De Acties

In de vorige lessen hebben we geleerd over de Class (de blauwdruk) en de Properties (de eigenschappen). We weten nu wat een object is (bijvoorbeeld: een Harten Aas). Maar een object dat alleen maar bestaat en niks doet, is saai.

We willen dat onze kaarten iets kunnen doen. Ze moeten zichzelf kunnen laten zien, omdraaien, of hun waarde doorgeven. Hiervoor gebruiken we Methoden.

Wat is een Methode?

Een methode is eigenlijk heel simpel: het is een functie, maar dan eentje die in een class zit.

De Vergelijking: Werkwoorden

Als je OOP vergelijkt met taal, dan ziet het er zo uit:

  • Class: Het zelfstandig naamwoord (De Kaart).
  • Property: Het bijvoeglijk naamwoord (De Rode Kaart).
  • Method: Het werkwoord (De Kaart toont zichzelf).

Methoden in Blackjack

Wat moet een kaart in ons spel kunnen doen? Laten we kijken naar de acties:

  • show(): Print de naam van de kaart netjes op het scherm (bijv. "Harten Aas").
  • getValue(): Vertelt de dealer hoeveel punten de kaart waard is (bijv. 11 punten).

Hoe maak je een Methode?

Je maakt een methode in je Class bestand. Dit doe je met het woord function. Omdat we willen dat iedereen deze actie kan uitvoeren, zetten we er public voor.

class Card
{
    // Eerst de eigenschappen (Properties)
    public $suit;
    public $value;

    // Dan de acties (Methoden)
    public function show()
    {
        // Hier komt de code die iets doet
        echo "Ik ben een kaart!";
    }
}

Het grote geheim: $this

Nu komt het lastigste, maar belangrijkste onderdeel van deze module. Stel je voor dat de methode show() moet zeggen welke kaart het is.

Je hebt misschien de neiging om dit te doen:

echo "Ik ben " . $suit; // FOUT! PHP kent $suit hier niet.

Waarom werkt dit niet? Omdat PHP niet weet welke suit je bedoelt. Bedoel je de suit van deze kaart? Of van die andere kaart op tafel?

Om duidelijk te maken dat we de eigenschap van dit specifieke object bedoelen, gebruiken we het woord $this.

Uitleg: $this

$this betekent letterlijk: "Ikzelf".

Als je in de code schrijft: $this->suit, dan zegt het object: "Pak de suit die in MIJN eigenschappenlijst staat."

Stap voor Stap: Een werkende Card Class

Laten we een complete class maken die zijn eigen naam kan uitprinten.

<?php

class Card
{
    public $suit;
    public $value;

    // De methode om de kaart te tonen
    public function show()
    {
        // Gebruik $this om bij je eigen data te komen
        // Let op de accolades {} in de string, dat is een handig trucje
        echo "Ik ben de {$this->value} van {$this->suit}.<br>";
    }
}

De Methode gebruiken (Aanroepen)

Nu we de methode hebben gemaakt in de blauwdruk, kunnen onze objecten deze actie uitvoeren. Dit doen we weer met het pijltje ->, maar nu met ronde haakjes () erachter.

// 1. Maak twee kaarten aan (Objecten)
$card1 = new Card();
$card1->suit = "Harten";
$card1->value = "Aas";

$card2 = new Card();
$card2->suit = "Schoppen";
$card2->value = "Boer";

// 2. Laat de kaarten hun actie uitvoeren
$card1->show();
// Output: "Ik ben de Aas van Harten."
// ($this wees hier naar card1)

$card2->show();
// Output: "Ik ben de Boer van Schoppen."
// ($this wees hier naar card2)

Wat kan er fout gaan? (Veelgemaakte Fouten)

1. De haakjes vergeten

Een methode is een functie. Een functie heeft altijd haakjes nodig om te starten.

$card1->show;   // FOUT: PHP denkt dat je een property zoekt die 'show' heet.
$card1->show(); // GOED: Voer de functie uit.

2. $this vergeten

Als je binnen een methode een variabele gebruikt zonder $this->, denkt PHP dat het een tijdelijke variabele is die alleen in die functie bestaat.

3. De dubbele dollar bij $this

Dit is de meest gemaakte fout:

$this->$suit; // FOUT!
$this->suit;  // GOED!

Je zegt: Het object ($this) -> wijs naar de eigenschap (suit). De naam van de eigenschap heeft geen dollarteken nodig.

Samenvatting

  • Methoden zijn de acties van een object (werkwoorden).
  • Je maakt ze met public function naam() { }.
  • Binnen de methode gebruik je $this-> om bij de eigenschappen van het object te komen.
  • Je roept ze aan met ->naam() (vergeet de haakjes niet!).
