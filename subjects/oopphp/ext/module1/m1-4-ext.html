Methoden: De Acties

In de vorige lessen hebben we geleerd over de Class (de blauwdruk) en de Properties (de eigenschappen). We weten nu wat een object is (bijvoorbeeld: een Harten Aas, rood van kleur).

Maar... een object dat alleen maar "bestaat" en niks doet, is saai. In een spel willen we actie. Kaarten moeten geschud worden, spelers moeten kaarten pakken en de dealer moet punten tellen.

Hiervoor gebruiken we Methoden.

Wat is een Methode?

Een methode is eigenlijk heel simpel: het is een functie, maar dan eentje die in een class zit. Het definieert het gedrag van een object.

De Vergelijking: Werkwoorden

Als je programmeren vergelijkt met taal, dan ziet het er zo uit:

  • Class: Het zelfstandig naamwoord (De Kaart).
  • Property: Het bijvoeglijk naamwoord (De Rode Kaart).
  • Method: Het werkwoord (De Kaart toont zichzelf).

Deel 1: De Class (De Blauwdruk)

We gaan een methode toevoegen aan onze Card class. We willen dat de kaart ons kan vertellen hoe hij heet.

We noemen deze functie getDescription (krijg omschrijving). Let op: we gebruiken hier return en geen echo. De kaart geeft de tekst aan ons, hij schreeuwt het niet zelf naar het scherm.

Maar hoe weet de kaart zijn eigen naam? Hoe weet hij of hij een Aas of een Boer is?

Het geheim van $this

Stel je voor dat jij en ik allebei een kaart vasthouden.
Als ik zeg: "Ik ben de Harten Aas", dan heb ik het over mezelf.
Als jij zegt: "Ik ben de Schoppen Boer", dan heb jij het over jezelf.

In PHP is $this het woordje voor "mezelf".
Met $this->suit zegt het object: "Pak de suit die in MIJN eigenschappenlijst staat."

Hier is de code voor de Class:

class Card
{
    // De Eigenschappen (Wat BEN ik?)
    public $suit;
    public $value;

    // De Methode (Wat DOE ik?)
    public function getDescription()
    {
        // We gebruiken $this om bij onze eigen data te komen.
        // We plakken de tekst aan elkaar en RETURNEN het resultaat.

        return "Ik ben de " . $this->value . " van " . $this->suit;
    }
}

Deel 2: Het Gebruik (index.php)

Nu we de blauwdruk hebben aangepast, kunnen alle kaarten die we maken dit kunstje uitvoeren. We roepen de methode aan met het pijltje -> gevolgd door de naam van de methode én ronde haakjes ().

Omdat de methode een tekst teruggeeft (return), moeten we die tekst zelf nog even op het scherm zetten met echo.

// 1. Maak de eerste kaart (Object aanmaken)
$card1 = new Card();
$card1->suit = "Harten";
$card1->value = "Aas";

// 2. Maak de tweede kaart
$card2 = new Card();
$card2->suit = "Schoppen";
$card2->value = "Boer";

// 3. Roep de methode aan
// $card1 gebruikt ZIJN eigen suit en value omdat we $this gebruikten.
echo $card1->getDescription();
// Output: "Ik ben de Aas van Harten"

echo "<br>";

// $card2 doet hetzelfde trucje met ZIJN eigen data.
echo $card2->getDescription();
// Output: "Ik ben de Boer van Schoppen"

Waarom Return en geen Echo?

Waarom zetten we de echo niet gewoon in de functie? Dat lijkt makkelijker.

Stel je voor dat je deze kaart later in een app wilt gebruiken, of in een e-mail wilt versturen. Dan wil je de tekst hebben (return) om hem op te slaan, niet direct op het scherm knallen (echo). Door return te gebruiken is je code veel flexibeler.

Wat kan er fout gaan? (Veelgemaakte Fouten)

⚠️ Vergeet de haakjes niet!

Een methode is een functie. Een functie heeft altijd haakjes nodig om te starten, ook als ze leeg zijn.

  • ❌ echo $card1->getDescription; (PHP zoekt naar een variabele, die bestaat niet)
  • ✅ echo $card1->getDescription(); (Voer de functie uit)

⚠️ De dubbele dollar

Bij $this gebruik je geen dollarteken voor de eigenschap.

  • ❌ $this->$suit; (Fout)
  • ✅ $this->suit; (Goed)

Samenvatting

  • Methoden zijn functies binnen een class. Ze bepalen het gedrag.
  • Gebruik $this-> binnen een methode om bij de eigenschappen van dat specifieke object te komen.
  • Gebruik return om een waarde terug te geven, zodat je er buiten de class mee kunt doen wat je wilt (zoals echoën).
  • Je roept een methode aan met ->naam().
