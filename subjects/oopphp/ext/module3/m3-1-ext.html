Docblocks: De Handleiding van je Code

Je hebt vast wel eens code geschreven die je een maand later terugzag en dacht: "Wat doet dit eigenlijk?". Of je werkt samen met een klasgenoot en snapt niets van zijn functies.

Om dit op te lossen, schrijven we Docblocks. Dit is meer dan gewoon commentaar; het is de officiële handleiding van je code.

Wat is een Docblock?

Een Docblock is een speciaal stukje tekst boven je class, functie of variabele. Het begint altijd met /** (twee sterretjes) en eindigt met */.

De Vergelijking: IKEA Handleiding

Stel je voor dat je een IKEA-kast koopt.

  • De code: Dat zijn de planken, schroeven en de hamer. Hiermee bouw je de kast.
  • Het Docblock: Dat is het boekje met instructies. Er staat in: "Dit is kast BILLY", "Je hebt 12 schroeven nodig" (Input) en "Het resultaat is een boekenkast" (Output).

Zonder handleiding kun je de kast misschien wel bouwen, maar het is veel moeilijker en je maakt sneller fouten.

Waarom doen we dit? (De 'Magic')

Misschien denk je: "Ik heb geen zin om commentaar te typen." Maar Docblocks doen iets magisch voor jou.

Programma's zoals VS Code of PhpStorm lezen deze blokken. Als jij daarna je functie wilt gebruiken, geeft het programma automatisch tips (autocomplete).

Hoe ziet het eruit?

Een Docblock bestaat uit drie onderdelen:

  1. De Beschrijving: Wat doet deze functie?
  2. @param (Input): Wat moet erin?
  3. @return (Output): Wat komt eruit?

Stap voor Stap: Een voorbeeld in Blackjack

Laten we kijken naar een functie in onze Card class die de naam van de kaart teruggeeft.

Stap 1: De Class beschrijven

Boven de class vertellen we kort wat dit bestand voorstelt.

/**
 * Represents a playing card in a Blackjack game.
 * This class holds the suit and value of a single card.
 */
class Card
{
    // ...
}

Stap 2: De Methode beschrijven

Nu kijken we naar een methode. Stel we hebben een functie setSuit. Zonder Docblock weet een andere programmeur misschien niet wat voor data hij moet meegeven. Is het een getal? Een tekst?

/**
 * Sets the card's suit (Hearts, Spades, etc.)
 *
 * @param string $suit  The name of the suit (e.g. "Hearts")
 * @return void         This method does not return anything
 */
public function setSuit(string $suit): void
{
    $this->suit = $suit;
}

Uitleg van de termen

  • @param: Dit is kort voor Parameter. Hier zeg je: "Ik verwacht een stukje string (tekst) en die variabele heet $suit."
  • @return: Hier zeg je wat de functie teruggeeft.
    • Geeft hij tekst terug? Gebruik @return string.
    • Geeft hij een getal terug? Gebruik @return int.
    • Doet hij alleen iets (zoals opslaan) en geeft hij niets terug? Gebruik @return void (leeg).

Pro Tip: De snelle manier

Je hoeft dit bijna nooit helemaal zelf te typen!

Ga in je editor (VS Code) op de regel boven je functie staan. Typ /** en druk op Enter. De editor maakt het hele blok automatisch voor je aan! Jij hoeft alleen de tekst nog in te vullen.

Wat kan er fout gaan?

Het gevaarlijkste aan commentaar is dat het verouderd raakt.

Stel, je verandert je code zodat de functie nu een getal (int) teruggeeft in plaats van tekst (string), maar je vergeet je Docblock aan te passen. Dan klopt de handleiding niet meer met de kast! Andere programmeurs raken dan in de war.

Regel: Als je de code verandert, verander je ook direct het Docblock.

Nog een voorbeeld: Kaartnaam ophalen

Hier zie je een functie die wél iets teruggeeft.

/**
 * Returns the full display name of the card.
 * Example: "Ace of Spades"
 *
 * @return string The formatted name of the card
 */
public function getCardName(): string
{
    return $this->value . " of " . $this->suit;
}

Samenvatting

  • Gebruik /** om een Docblock te starten.
  • Het helpt jou omdat je editor dan betere tips kan geven.
  • Het helpt anderen om je code te begrijpen zonder alles te hoeven lezen.
  • Houd je commentaar altijd up-to-date!
