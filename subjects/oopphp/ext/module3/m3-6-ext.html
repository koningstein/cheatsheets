Private Methoden: De Hulpjes

In de vorige les leerde je over private properties (de kluis). Je kunt data beschermen tegen de buitenwereld.

Maar wist je dat je ook methoden (functies) privé kunt maken? Dit klinkt misschien gek. Waarom zou je een functie maken die niemand mag gebruiken? Het antwoord is simpel: om je code op te ruimen en te organiseren.

Wat is een Private Methode?

Een private methode is een functie die alleen binnen zijn eigen class gebruikt mag worden. Je kunt hem niet aanroepen met $game->doeIets(), maar wel met $this->doeIets().

De Vergelijking: Het Restaurant

Stel je voor dat je in een restaurant zit (de buitenwereld).

  • Public Methode (Het Menu): Jij roept tegen de ober: bestelHamburger(). Dit mag jij doen.
  • Private Methoden (De Keuken): Om die burger te maken, moet de chef:
    • snijUien()
    • bakVlees()
    • roosterBroodje()

Jij als klant hoeft niet te roepen: "Snij uien!". Dat doet de chef zelf, als onderdeel van jouw bestelling. De actie "Snij uien" is intern (private).

Waarom doen we dit? (Opruimen)

Stel dat we een Blackjack spelronde programmeren. Dat is een flinke klus. Zonder private methoden krijg je één enorme functie waar niemand meer iets van snapt.

Zonder Private Methoden (Chaos)

class Game {
    public function playRound() {
        // ... 50 regels code om kaarten te schudden ...
        // ... 20 regels code om kaarten uit te delen ...
        // ... 30 regels code om punten te tellen ...
        // ... 40 regels code om de winnaar te checken ...
    }
}

Met Private Methoden (Netjes)

We knippen de grote klus op in kleine, logische stapjes. De publieke functie is nu eigenlijk een soort takenlijstje geworden.

class Game
{
    // Dit is de enige knop die de buitenwereld mag indrukken
    public function playRound(Player $player, Player $dealer): string
    {
        // We roepen onze eigen hulpjes aan met $this
        $this->dealInitialCards($player, $dealer);
        $this->playerTurn($player);
        $this->dealerTurn($dealer);

        return $this->determineWinner($player, $dealer);
    }

    // --- HIERONDER STAAN DE INTERNE HULPJES ---

    // Private: Mag alleen door Game zelf worden gebruikt
    private function dealInitialCards(Player $player, Player $dealer): void
    {
        // Hier staat de ingewikkelde code voor het delen
    }

    private function playerTurn(Player $player): void
    {
        // De logica voor de speler
    }

    private function dealerTurn(Player $dealer): void
    {
        // De logica voor de dealer
    }

    private function determineWinner($player, $dealer): string
    {
        // Wie heeft er gewonnen?
        return "De winnaar is...";
    }
}

Wat kan er fout gaan?

Als je probeert om een hulp-functie van buitenaf aan te roepen, grijpt PHP in. Je mag de keuken niet in rennen!

$game = new Game();

// ✅ Dit mag: De officiële startknop
$game->playRound($player, $dealer);

// ❌ Dit mag NIET: Je probeert direct de interne logica te starten
// Foutmelding: Fatal Error: Call to private method Game::playerTurn()
$game->playerTurn($player);

Waarom is dit handig voor onderhoud?

Stel dat je de manier waarop kaarten worden gedeeld wilt veranderen. Je hoeft dan alleen de code in dealInitialCards aan te passen.

Omdat deze functie private is, weet je 100% zeker dat niemand anders in jouw programma deze functie gebruikt. Je kunt hem veilig aanpassen zonder dat de rest van je website stuk gaat.

Samenvatting

  • Gebruik private methoden om grote taken in stukjes te hakken.
  • Het maakt je code leesbaar: de publieke functie vertelt wat er gebeurt, de private functies vertellen hoe het gebeurt.
  • Je roept ze aan met $this->naamVanMethode().
