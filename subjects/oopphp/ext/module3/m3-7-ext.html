Samenvatting: Veilig Programmeren

Gefeliciteerd met het afronden van Module 3! Waar Module 1 en 2 gingen over "dingen werkend krijgen", ging deze module over kwaliteit en veiligheid.

Je hebt geleerd om streng te zijn voor je code. Je laat niet zomaar alles toe, maar je werkt met duidelijke afspraken en beveiliging.

1. Types: De Strenge Poortwachters

PHP is van zichzelf heel flexibel (misschien wel té flexibel). In deze module hebben we geleerd om die flexibiliteit te beperken met Types. Dit voorkomt fouten.

Onderdeel De Regel Voorbeeld
Parameter Type
(De Ingang)		 Ik accepteer alleen data van dit type. function addScore(int $points)
"Alleen getallen mogen naar binnen!"
Return Type
(De Uitgang)		 Ik beloof dat ik dit type teruggeef. function getName(): string
"Ik geef altijd tekst terug."
Property Type
(De Opslag)		 In deze variabele past alleen dit type. private int $score;
"Hier passen alleen getallen in."
Object Type
(De Relatie)		 Ik wil geen tekst, maar een écht object. function addCard(Card $card)
"Geef mij een echt Kaart-object."

2. Docblocks: De Handleiding

Naast de harde code, hebben we ook geleerd om documentatie te schrijven. Docblocks (/** ... */) zijn berichtjes aan je collega's én aan je programma (zoals VS Code).

  • Ze leggen uit wat de code doet.
  • Ze helpen bij arrays (Card[]) omdat PHP dat zelf nog niet zo goed kan aangeven.

3. Encapsulatie: De Kluis

Het belangrijkste concept van deze module was Encapsulatie (Inkapseling). We stoppen de data in een kluis en doen de deur op slot.

Het Grote Verschil

  • Zonder Encapsulatie (Public): Iedereen kan de score van de speler veranderen naar -500 of "Banaan". Je hebt geen controle.
  • Met Encapsulatie (Private):
    • De data (property) is private (De kluis).
    • Je wijzigt data via een Setter (De bewaker die checkt of het mag).
    • Je leest data via een Getter (De etalage).

Het Ultieme Voorbeeld

Hieronder zie je een `Player` class waarin alles uit deze module samenkomt. Als je deze code snapt, ben je klaar voor de volgende module.

<?php

namespace Blackjack;

/**
 * Represents a player in the Blackjack game.
 */
class Player
{
    // 1. PROPERTY TYPES & VISIBILITY
    // Private: Niemand mag hier direct bij.
    // Type: Er mag alleen een getal (int) in.
    private int $score = 0;

    // Type: Er mag alleen een tekst (string) in.
    private string $name;

    // 2. CONSTRUCTOR MET TYPES
    public function __construct(string $name)
    {
        $this->name = $name;
    }

    // 3. OBJECT ALS PARAMETER
    // We willen een echte 'Card', geen tekst.
    public function addPointsFromCard(Card $card): void
    {
        // We kunnen veilig methodes van Card aanroepen
        $this->score += $card->getValue();
    }

    // 4. RETURN TYPES & GETTERS
    // We beloven een integer terug te geven.
    public function getScore(): int
    {
        return $this->score;
    }

    // 5. NULLABLE TYPES
    // Soms is er nog geen winnaar, dus '?Player' (Player of null).
    public function checkWinner(?Player $opponent): string
    {
        if ($opponent === null) {
            return "Er is geen tegenstander.";
        }
        return "We spelen tegen " . $opponent->name; // (Eigenlijk via getter!)
    }
}

Wat nu?

Je schrijft nu veilige, robuuste code. Fouten worden direct ontdekt door PHP omdat je streng bent op je types. Je data is veilig omdat je private gebruikt.
