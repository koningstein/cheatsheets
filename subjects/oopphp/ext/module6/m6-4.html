Abstracte Classes

Met het keyword abstract voorkom je dat er objecten van een class worden gemaakt. Het dient puur als basis voor inheritance.

Abstracte Methoden (Het Contract)

Een abstracte methode heeft geen body (geen {}) en eindigt met een puntkomma. Het dwingt alle child classes om deze methode te implementeren.

abstract class Player
{
    // Het contract: Iedereen MOET kunnen beslissen
    abstract public function makeDecision(): string;
}

class Dealer extends Player
{
    // De implementatie: Dealer volgt regels
    public function makeDecision(): string {
        return ($this->getScore() < 17) ? 'hit' : 'stand';
    }
}

Polymorfisme

Doordat we zeker weten dat elke Player de methode makeDecision heeft (dankzij de abstracte definitie), kunnen we polymorfisme toepassen in de Game loop.

Polymorfisme betekent dat verschillende objecten (Human, Dealer) reageren op dezelfde methode-aanroep (makeDecision), maar elk hun eigen unieke gedrag vertonen.

User Interface Workflow

In deze module heb je ook geleerd hoe je input van de gebruiker (via formulieren) koppelt aan je objecten.

  1. View (HTML): Gebruiker kiest een actie in een <form>.
  2. Controller (PHP): Vangt $_POST op.
  3. Setter: Controller geeft de keuze door aan het object: $human->setNextAction(...).
  4. Polymorfe Call: Controller roept $game->playRound() aan, die intern makeDecision() gebruikt.
