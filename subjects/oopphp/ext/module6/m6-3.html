Interactie & Polymorfisme

Van Formulier naar Object

We hebben nu abstracte classes en methodes, maar hoe verbinden we dit met de gebruiker? In een webapplicatie gebeurt interactie via HTML-formulieren. De uitdaging is om de keuze van de gebruiker ("Hit" of "Stand") op de juiste plek in je OOP-structuur te krijgen.

Stap 1: De User Interface

In plaats van hardcoded acties, laten we de gebruiker kiezen. We maken een formulier waarin de speler zijn volgende zet doorgeeft.

<!-- templates/game.tpl -->
<form method="POST" action="index.php?page=playTurn">
    <select name="action" class="form-select">
        <option value="hit">Hit</option>
        <option value="stand">Stand</option>
        <option value="double">Double Down</option>
    </select>
    <button type="submit">Voer actie uit</button>
</form>

Stap 2: De Input Verwerken (Setter)

In je PHP-code (de Controller) vang je deze keuze op. Je moet deze keuze doorgeven aan het specifieke object dat aan de beurt is (de HumanPlayer).

// index.php
if ($_SERVER['REQUEST_METHOD'] === 'POST') {
    $action = $_POST['action']; // "hit" of "stand"

    // Haal de menselijke speler op uit het spel
    $human = $game->getHumanPlayer();

    // Geef de keuze door aan het object
    $human->setNextAction($action);

    // Start de ronde
    $game->playRound();
}

Stap 3: Polymorfe Uitvoering

Nu komt de kracht van OOP en abstractie naar voren. De Game class roept simpelweg makeDecision() aan op elke speler. Hij hoeft niet te weten dat de één wacht op input en de ander automatisch speelt.

// HumanPlayer implementatie
class HumanPlayer extends Player {
    private string $nextAction;

    public function setNextAction(string $action) {
        $this->nextAction = $action;
    }

    public function makeDecision(): string {
        // Gebruikt de keuze die via het formulier is binnengekomen
        return $this->nextAction;
    }
}

// Dealer implementatie
class Dealer extends Player {
    public function makeDecision(): string {
        // Negeert input, volgt altijd de regels
        return ($this->getScore() < 17) ? 'hit' : 'stand';
    }
}

foreach ($this->players as $player) {

Samenvatting

  • UI: Gebruiker maakt een keuze in een formulier (`Select` of `Button`).
  • Controller: Vangt `$_POST` op en stelt de state in op het juiste object (`setNextAction`).
  • Model: De `makeDecision` methode gebruikt die state (Human) of negeert hem (Dealer).
  • Polymorfisme: De hoofdlogica (`Game`) behandelt alle spelers gelijk.
