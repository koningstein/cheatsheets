Resource Controllers

Een Resource Controller bevat standaard methodes voor alle CRUD-operaties. Dit bespaart je het handmatig schrijven van 7 losse routes.

Leeruitkomsten

  • Ik kan een Resource Controller aanmaken met de --model flag voor automatic Route Model Binding.
  • Ik kan aliassen gebruiken voor controllers in submappen (Admin/Open).
  • Ik kan gedeeltelijke resource routes maken met only() en except().
  • Ik kan API Resource Controllers maken met --api en apiResource().

1. Aanmaken met Model (Best Practice)

Gebruik altijd de --model vlag. Laravel koppelt het model dan direct aan de methodes (Route Model Binding), zodat je niet zelf met find($id) hoeft te werken.


# Maak controller met alle 7 methodes, gekoppeld aan Product model
php artisan make:controller Admin/ProductController --model=Product

De gegenereerde controller krijgt automatisch de juiste methodes:

Bestand: app/Http/Controllers/Admin/ProductController.php

// Laravel injecteert direct het Model
public function show(Product $product)
{
    return view('admin.products.show', compact('product'));
}

2. Routing met Aliassen (Admin/Open)

Als je controllers in submappen hebt (zoals Admin en Open), wordt je route-bestand snel onleesbaar door lange paden. Gebruik aliassen om dit netjes te houden.

Bestand: routes/web.php

use Illuminate\Support\Facades\Route;
// Importeer de mappen als Alias
use App\Http\Controllers\Admin as Admin;
use App\Http\Controllers\Open as Open;

// Nu kun je kort verwijzen naar Admin\ControllerNaam
Route::resource('admin/products', Admin\ProductController::class);

// En naar Open\ControllerNaam
Route::resource('cart', Open\CartController::class);

3. Gedeeltelijke Resource Routes

Soms heb je niet alle 7 routes nodig. Gebruik only of except om routes uit te sluiten. Dit houdt je applicatie veiliger en schoner.


// Alleen index en show (bijv. voor publieke pagina's)
Route::resource('news', Open\NewsController::class)->only(['index', 'show']);

// Alles behalve destroy (bijv. als je niet mag verwijderen)
Route::resource('users', Admin\UserController::class)->except(['destroy']);

4. API Resource Controllers

Als je een API bouwt (bijv. voor een mobiele app of Vue/React frontend), heb je de formulieren (create en edit) niet nodig. Laravel heeft hier een speciale variant voor.

Genereren:

php artisan make:controller Api/ProductController --api

Routing:


// Maakt alleen index, store, show, update en destroy aan
Route::apiResource('products', Api\ProductController::class);
