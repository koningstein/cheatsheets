Basic Routing & Parameters

Leeruitkomsten

  • Ik begrijp hoe routes werken en waar ze worden gedefinieerd.
  • Ik kan routes maken voor verschillende HTTP verbs (GET, POST, PUT, DELETE).
  • Ik kan dynamische parameters uit de URL halen met {parameter}.
  • Ik kan optionele parameters maken met {parameter?} en standaardwaarden.

1. Basis HTTP Verbs

Je definieert routes in routes/web.php. Laravel ondersteunt alle HTTP werkwoorden.

Bestand: routes/web.php

use Illuminate\Support\Facades\Route;

Route::get('/', function () {
    return view('welcome');
});

Route::post('/contact', [ContactController::class, 'send']);
Route::put('/users/{id}', [UserController::class, 'update']);
Route::delete('/users/{id}', [UserController::class, 'destroy']);

2. Parameters

Soms wil je dynamische waarden uit de URL halen, zoals een ID. Dit doe je met accolades { }.


// URL: /products/5
Route::get('/products/{id}', function (string $id) {
    return 'Product ID: ' . $id;
});

3. Optionele Parameters

Wil je dat een parameter niet verplicht is? Zet er een vraagteken achter {name?} en geef de variabele een standaardwaarde.


// URL: /user  OF  /user/Jan
Route::get('/user/{name?}', function (?string $name = 'Gast') {
    return 'Hallo ' . $name;
});
