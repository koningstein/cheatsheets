Route Model Binding

Laravel kan automatisch Database-modellen ophalen op basis van ID's in de URL. Dit scheelt je het schrijven van User::find($id) en 404-checks.

Leeruitkomsten

  • Ik begrijp hoe Route Model Binding werkt en waarom het nuttig is.
  • Ik kan implicit binding gebruiken om modellen automatisch op te halen op basis van ID.
  • Ik kan custom keys gebruiken om te zoeken op andere kolommen zoals slug of username.
  • Ik kan soft deleted items ophalen met withTrashed().

1. Implicit Binding (Op ID)

Zorg dat de naam van de parameter in de route ({user}) overeenkomt met de naam van de variabele in de controller.

Bestand: routes/web.php

// De parameter heet {user}
Route::get('/users/{user}', [UserController::class, 'show']);
Bestand: app/Http/Controllers/UserController.php

// Type-hint de Model class. Laravel zoekt automatisch User met dat ID.
public function show(User $user)
{
    // Als ID niet bestaat, geeft Laravel automatisch een 404.
    return view('users.show', ['user' => $user]);
}

2. Custom Keys (Niet op ID)

Soms wil je zoeken op een andere kolom, bijvoorbeeld een slug of username voor mooiere URL's.


// URL: /posts/mijn-eerste-blog
Route::get('/posts/{post:slug}', [PostController::class, 'show']);

Laravel zoekt nu in de posts tabel waar de kolom slug gelijk is aan de parameter.

3. Soft Deletes (Trashed Items)

Standaard vindt Model Binding geen items die 'soft deleted' zijn. Wil je dit wel (bijv. in een admin prullenbak)? Gebruik withTrashed().


Route::get('/users/{user}/restore', [UserController::class, 'restore'])
    ->withTrashed();
