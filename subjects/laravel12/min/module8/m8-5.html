Single Action Controllers

Soms is een controller zo complex of specifiek dat hij maar één taak heeft. In plaats van een methode-naam te verzinnen (zoals process), gebruiken we de magische __invoke methode.

Leeruitkomsten

  • Ik begrijp wanneer een Single Action Controller nuttig is.
  • Ik kan een Single Action Controller aanmaken met de --invokable flag.
  • Ik kan de __invoke methode gebruiken voor de controller logica.
  • Ik kan een route registreren die alleen de controller class nodig heeft zonder methode naam.

1. Aanmaken

Gebruik de --invokable vlag.

php artisan make:controller ProvisionServer --invokable

2. De Controller

De controller heeft maar één methode: __invoke.

Bestand: app/Http/Controllers/ProvisionServer.php

namespace App\Http\Controllers;

use Illuminate\Http\Request;

class ProvisionServer extends Controller
{
    public function __invoke(Request $request)
    {
        // Doe hier de complexe logica...
        return 'Server provisioned!';
    }
}

3. Routing

In de route hoef je geen array [Controller::class, 'method'] mee te geven, alleen de classnaam is genoeg.

Bestand: routes/web.php

use App\Http\Controllers\ProvisionServer;

Route::post('/server/provision', ProvisionServer::class);
