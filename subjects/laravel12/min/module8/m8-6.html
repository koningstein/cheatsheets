Redirects & Responses

Een controller methode moet altijd een antwoord (Response) teruggeven aan de browser. Dit kan een view zijn, een redirect, JSON, of een bestand.

Leeruitkomsten

  • Ik kan redirects maken met to_route() en back().
  • Ik kan flash messages meegeven met redirects via ->with().
  • Ik kan JSON responses maken voor API's met response()->json().
  • Ik kan error pagina's tonen met abort().

1. Redirects

Doorverwijzen naar een andere pagina, vaak na het opslaan van een formulier.


use Illuminate\Http\RedirectResponse;

public function store(): RedirectResponse
{
    // Naar een named route
    return to_route('users.index');

    // Terug naar vorige pagina (met input bij errors)
    return back()->withInput();

    // Met een flash message
    return to_route('dashboard')->with('status', 'Profile updated!');
}

2. JSON Responses (API)

Voor API's of AJAX calls stuur je vaak JSON terug.


use Illuminate\Http\JsonResponse;

public function index(): JsonResponse
{
    return response()->json([
        'name' => 'Laravel',
        'status' => 'Active'
    ]);
}

3. Errors (Abort)

Als je een proces wilt stoppen en een error pagina wilt tonen.


if (! $user->isAdmin()) {
    abort(403, 'Toegang geweigerd.');
}

// Zoekt automatisch de view resources/views/errors/404.blade.php
abort(404);
