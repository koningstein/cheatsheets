Order Seeds

Voordat we orders kunnen opslaan, hebben we statussen nodig in de database. We gebruiken een seeder om deze vaste waardes aan te maken.

Leeruitkomsten

  • Ik kan een seeder maken voor statische data (statussen).
  • Ik kan seeders toevoegen aan de DatabaseSeeder.
  • Ik begrijp waarom je ID's soms hardcoded wilt hebben.

Stap 1: State Seeder Maken

Maak een seeder aan:

php artisan make:seeder StateSeeder

Stap 2: Statussen Vullen

In database/seeders/StateSeeder.php voegen we de statussen toe. We gebruiken hier create omdat we zeker willen weten dat deze specifiek bestaan.


use App\Models\State;

public function run(): void
{
    // Vaste statussen aanmaken
    State::create(['name' => 'Pending']); // ID 1
    State::create(['name' => 'Paid']);    // ID 2
    State::create(['name' => 'Shipped']); // ID 3
    State::create(['name' => 'Delivered']); // ID 4
}

Stap 3: DatabaseSeeder Aanpassen

Zorg dat de StateSeeder wordt aangeroepen in de DatabaseSeeder. De volgorde is belangrijk: States moeten bestaan voordat je Orders (of testdata daarvan) zou seeden.


public function run(): void
{
    $this->call([
        RoleAndPermissionSeeder::class,
        UserSeeder::class,
        ProductSeeder::class, // Maakt categorieÃ«n, producten en prijzen
        StateSeeder::class,   // NIEUW: Maakt statussen
    ]);
}

Stap 4: Uitvoeren

Ververs de database om de nieuwe tabellen en data te krijgen.

php artisan migrate:fresh --seed
