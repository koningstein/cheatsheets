Spatie Permissions

Tot nu toe hebben we gezien hoe je zelf logica schrijft in Gates en Policies (bijvoorbeeld $user->is_admin === true). Maar wat als je applicatie groeit? Wat als je Rollen (Roles) wilt zoals 'Admin', 'Editor', 'Moderator', 'User', en deze dynamisch wilt beheren? En wat als die rollen specifieke Permissies (Permissions) hebben zoals 'edit articles' of 'publish posts'?

Het zelf bouwen van zo'n systeem in de database (met tussentabellen) is complex en foutgevoelig. Daarom gebruiken we in de Laravel community bijna standaard de package: spatie/laravel-permission.

Leeruitkomsten

  • Ik begrijp het verschil tussen Roles en Permissions.
  • Ik kan de `spatie/laravel-permission` package gebruiken in mijn User model.
  • Ik kan rollen en permissies aanmaken en toewijzen via een Seeder.
  • Ik kan controleren op permissies in Code en Blade.

Het Concept: Roles vs Permissions

  • Permission: Een specifieke actie die iemand wel/niet mag. Bijvoorbeeld: "artikel verwijderen".
  • Role: Een verzameling van permissies. Bijvoorbeeld: Een "Editor" mag "artikel schrijven" en "artikel bewerken", maar niet "gebruiker verwijderen".

Het beste is om in je code te checken op Permissions (niet op rollen), en gebruikers vervolgens een Rol te geven die deze permissies bevat.

Installatie & Gebruik

Na installatie via Composer voeg je de HasRoles trait toe aan je User model. Hiermee krijgt je user model toegang tot krachtige methodes.

Bestand: app/Models/User.php

use Spatie\Permission\Traits\HasRoles;

class User extends Authenticatable
{
    use HasRoles; // Deze regel activeert de package functionaliteit

    // ...
}

Rollen en Permissies Aanmaken

Dit doe je meestal in een Seeder, zodat je database bij een verse installatie direct goed staat.

Bestand: database/seeders/RolesAndPermissionsSeeder.php

use Spatie\Permission\Models\Role;
use Spatie\Permission\Models\Permission;

// 1. Maak permissies
Permission::create(['name' => 'edit articles']);
Permission::create(['name' => 'delete articles']);
Permission::create(['name' => 'manage users']);

// 2. Maak rollen en wijs permissies toe
$role = Role::create(['name' => 'writer']);
$role->givePermissionTo('edit articles');

$role = Role::create(['name' => 'admin']);
$role->givePermissionTo(Permission::all()); // Admin mag alles

// 3. Geef een gebruiker een rol
$user = User::find(1);
$user->assignRole('admin');

Controleren in Code

Dankzij de package integreert dit naadloos met Laravel's standaard @can en $user->can() methodes!

Voorbeeld Logic

// Check op permissie (Beste manier!)
if ($user->can('edit articles')) {
    // Deze user mag dit, omdat hij de rol 'writer' of 'admin' heeft.
}

// Check op rol (Soms nodig, maar minder flexibel)
if ($user->hasRole('admin')) {
    // ...
}

Middleware in Routes

De package levert ook middleware mee om routes te beschermen op basis van rol of permissie.

Bestand: routes/web.php

// Alleen toegankelijk voor gebruikers met de 'admin' rol
Route::group(['middleware' => ['role:admin']], function () {
    Route::get('/users', [UserController::class, 'index']);
});

// Toegankelijk voor iedereen die 'publish articles' mag (ongeacht de rol)
Route::group(['middleware' => ['permission:publish articles']], function () {
    Route::post('/articles', [ArticleController::class, 'store']);
});
