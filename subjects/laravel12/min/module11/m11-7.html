Helper Functions

Laravel biedt een groot aantal globale 'helper' functies die je werk als developer makkelijker maken. Binnen het thema Security & Auth zijn er specifieke helpers die je dagelijks zult gebruiken om authenticatie, autorisatie en beveiliging te regelen.

Leeruitkomsten

  • Ik ken de belangrijkste security helpers zoals auth() en abort().
  • Ik kan snel de ingelogde gebruiker of diens ID ophalen.
  • Ik weet hoe ik een HTTP-error (zoals 403) kan forceren.

De `auth()` Helper

De meest gebruikte helper is auth(). Hiermee krijg je toegang tot de authenticatie-laag.

Veelgebruikte methodes

// 1. Haal de ingelogde User (Model) op. Geeft null als niemand is ingelogd.
$user = auth()->user();

// 2. Haal alleen het ID van de ingelogde user op (efficiÃ«nter dan hele user).
$id = auth()->id();

// 3. Check of er iemand is ingelogd (geeft true/false).
if (auth()->check()) {
    // ...
}

// 4. Check of er NIEMAND is ingelogd (geeft true/false).
if (auth()->guest()) {
    // ...
}

De `bcrypt()` Helper

Hoewel je in moderne Laravel applicaties vaak de Hash facade gebruikt, bestaat er ook een helper functie voor het hashen van wachtwoorden. Deze gebruikt standaard Bcrypt.


$password = bcrypt('geheim123');
// Is hetzelfde als:
$password = Hash::make('geheim123');

De `abort()` Helper

Soms wil je de uitvoering van je script direct stoppen en een foutmelding tonen, bijvoorbeeld als iemand iets probeert wat niet mag. De abort() functie gooit een HTTP-exception.


// Toon een 403 Forbidden pagina
abort(403);

// Toon een 403 pagina met een aangepast bericht
abort(403, 'Je hebt geen toegang tot deze pagina.');

// Toon een 404 Not Found
abort(404);

Overige Handige Helpers

  • csrf_token(): Geeft de huidige CSRF-token als string terug (handig voor AJAX calls).
  • csrf_field(): Genereert een volledig hidden input veld met het token (gebruik in Blade liever @csrf).
  • request()->user(): Een alternatieve manier om de user op te halen via het Request object.
