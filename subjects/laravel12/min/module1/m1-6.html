Resource Controller - Validatie

We mogen niet zomaar alle invoer accepteren. Om de integriteit van de data te waarborgen, voegen we validatie toe aan het opslaan-proces door middel van een FormRequest.

Leeruitkomsten

  • Ik kan een FormRequest maken.
  • Ik kan validatieregels toepassen (required, min, max, unique).
  • Ik kan validatiefouten tonen in views.
  • Ik kan flash messages gebruiken.

Stap 1: FormRequest Maken

We verplaatsen de validatie naar een eigen class om de controller netjes te houden.

php artisan make:request CategoryStoreRequest

Stap 2: Regels Instellen

In app/Http/Requests/CategoryStoreRequest.php:


public function authorize(): bool
{
    return true; // Zet op TRUE, anders mag niemand het formulier sturen!
}

public function rules(): array
{
    return [
        'name' => 'required|string|max:75|unique:categories',
    ];
}

Stap 3: Controller Aanpassen

We vervangen Request door CategoryStoreRequest. Laravel voert nu automatisch de validatie uit voordat jouw code draait.


// Importeer de class bovenaan!
use App\Http\Requests\CategoryStoreRequest;

public function store(CategoryStoreRequest $request)
{
    // Sla op met gevalideerde data
    Category::create($request->validated());

    // Redirect met flash message
    return to_route('categories.index')
            ->with('status', 'Categorie succesvol aangemaakt!');
}

Stap 4: Fouten Tonen

In create.blade.php tonen we foutmeldingen bij het veld met @error. Ook zorgen we dat de ingevulde tekst blijft staan met old().


<label>Naam:</label>
<input type="text" name="name" value="{{ old('name') }}">

@error('name')
    <p class="text-red-500">{{ $message }}</p>
@enderror

Stap 5: Succesbericht

In index.blade.php controleren we op de sessie variabele 'status'.


@if(session('status'))
    <div class="bg-green-100 p-4 mb-4">
        {{ session('status') }}
    </div>
@endif
