Resource Controller - Edit & Update

Data moet gewijzigd kunnen worden. We maken een Edit-pagina met een vooraf ingevuld formulier en een Update-methode die de wijzigingen verwerkt, inclusief validatie.

Leeruitkomsten

  • Ik kan een pre-filled formulier maken.
  • Ik kan PUT/PATCH method gebruiken.
  • Ik kan update validatie toepassen met unique rule exception.
  • Ik kan bestaande data updaten.

Stap 1: Edit Methode

We halen het record op via Model Binding en tonen het formulier.


public function edit(Category $category)
{
    return view('admin.categories.edit', [
        'category' => $category
    ]);
}

Stap 2: Edit Formulier

Maak edit.blade.php. Drie belangrijke punten:

  1. Actie gaat naar categories.update.
  2. Methode is POST, maar we voegen @method('PUT') toe.
  3. Value gebruikt old() met als tweede parameter de database waarde: old('name', $category->name).

@extends('layouts.layoutadmin')

@section('content')
    <h1>Wijzigen</h1>

    <form action="{{ route('categories.update', $category) }}" method="POST">
        @csrf
        @method('PUT')

        <label>Naam:</label>
        <input type="text" name="name"
               value="{{ old('name', $category->name) }}">

        <button type="submit">Wijzigen</button>
    </form>
@endsection

Stap 3: Update Request

Bij het updaten mag de naam "uniek" zijn, behalve als het de naam van dit record zelf is. We maken een nieuwe request:

php artisan make:request CategoryUpdateRequest

// app/Http/Requests/CategoryUpdateRequest.php
public function rules(): array
{
    // Haal ID op van huidige categorie
    $id = $this->category->id;

    return [
        // "name" moet uniek zijn in tabel "categories", behalve rij met "$id"
        'name' => 'required|max:75|unique:categories,name,' . $id
    ];
}

Stap 4: Update Methode


public function update(CategoryUpdateRequest $request, Category $category)
{
    $category->update($request->validated());

    return to_route('categories.index')
            ->with('status', 'Gewijzigd!');
}
