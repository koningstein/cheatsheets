Soft Deletes

Met Soft Deletes wordt een record niet echt verwijderd, maar krijgt het een datum in de deleted_at kolom. Hierdoor is het "onzichtbaar" voor normale queries, maar kun je het wel herstellen.

Leeruitkomsten

  • Ik kan de softDeletes() kolom toevoegen aan een migratie.
  • Ik kan de SoftDeletes trait gebruiken in een model.
  • Ik begrijp het verschil tussen delete() en echte verwijdering.
  • Ik kan soft deleted items ophalen met withTrashed() en herstellen met restore().

1. Migratie Aanpassen

Voeg de softDeletes() kolom toe.

Bestand: database/migrations/xxxx_create_products_table.php

$table->softDeletes();

2. Model Aanpassen

Gebruik de SoftDeletes trait.

Bestand: app/Models/Product.php

use Illuminate\Database\Eloquent\SoftDeletes;

class Product extends Model
{
    use SoftDeletes;
}

3. Gebruik


// Verwijderen (zet deleted_at)
$product->delete();

// Ophalen inclusief verwijderde items
$allProducts = Product::withTrashed()->get();

// Herstellen
$product->restore();
