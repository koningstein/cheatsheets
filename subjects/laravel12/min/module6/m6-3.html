Model Conventions

Laravel werkt met "Convention over Configuration". Als je je aan de regels houdt, hoef je bijna niets in te stellen.

Leeruitkomsten

  • Ik begrijp de Laravel naamgevingsconventies voor models en tabellen.
  • Ik kan afwijken van standaard conventies wanneer nodig (legacy databases).
  • Ik kan custom primary keys en timestamps configureren in een model.
  • Ik weet wanneer ik $incrementing en $keyType moet aanpassen.

1. Naamgeving

  • Model: Enkelvoud, PascalCase (bijv. Product, UserOrder).
  • Tabel: Meervoud, snake_case (bijv. products, user_orders).
  • Foreign Key: model_id (bijv. product_id).
  • Primary Key: id.

2. Afwijken van de Standaard

Soms werk je met een oude database (legacy) en kloppen de namen niet. Je kunt dit instellen in je Model.

Bestand: app/Models/Product.php

class Product extends Model
{
    // Tabel heet anders
    protected $table = 'my_products_table';

    // Primary key heet anders
    protected $primaryKey = 'product_code';

    // Als je geen integers als ID gebruikt (bijv. UUIDs)
    public $incrementing = false;
    protected $keyType = 'string';

    // Als je geen timestamps (created_at/updated_at) hebt
    public $timestamps = false;
}
