Factories & Faker

Factories genereren nepdata voor je database. Laravel gebruikt de Faker bibliotheek hiervoor.

Leeruitkomsten

  • Ik kan een factory aanmaken en configureren.
  • Ik kan Faker gebruiken voor verschillende soorten testdata (tekst, getallen, datums).
  • Ik kan logische datums genereren met Carbon en dateTimeBetween.
  • Ik kan foreign keys vullen met bestaande data uit de database.

1. Factory Maken

php artisan make:factory ProductFactory

2. Definitie Schrijven

In de definition methode geef je aan hoe de data eruit moet zien.

Bestand: database/factories/ProductFactory.php

public function definition(): array
{
    // Datum logica: Eerst een random startdatum kiezen in het verleden
    $startDate = $this->faker->dateTimeBetween('-1 year', 'now');

    // De einddatum moet NA de startdatum liggen (bijv. 10 dagen later)
    // We gebruiken Carbon::instance() om van het DateTime object
            een Carbon object te maken
    $endDate = Carbon::instance($startDate)->addDays(10);

    return [
        // Datums met logica (Start en Eind)
        'start_date' => $startDate,
        'end_date'   => $endDate,

        // Willekeurige naam (2 woorden)
        'name' => $this->faker->words(2, true),

        // Willekeurige tekst
        'description' => $this->faker->paragraph(),

        // String met specifieke lengte (voor validatie tests: min 5, max 15)
        'short_code' => $this->faker->realTextBetween(5, 15),

        // Getal tussen 1 en 100
        'stock' => $this->faker->numberBetween(1, 100),

        // Decimaal getal (2 cijfers precisie, min 10, max 500)
        'price' => $this->faker->randomFloat(2, 10, 500),

        // Willekeurig element uit een array
        'status' => $this->faker->randomElement(['draft', 'published', 'archived']),

        // Foreign Key vullen met bestaande data
        // Haal alle Users op en kies er willekeurig één (zodat de relatie klopt)
        'user_id' => User::all()->random()->id,
    ];
}
