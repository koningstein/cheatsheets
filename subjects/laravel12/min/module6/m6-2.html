1. De Basis: ForeignId

Gebruik foreignId in combinatie met constrained. Laravel zoekt zelf de tabel (bijv. user_id -> users tabel).

Bestand: database/migrations/xxxx_create_posts_table.php

$table->foreignId('user_id')->constrained();

2. OnDelete Opties

Wat moet er gebeuren als de ouder (bijv. User) wordt verwijderd?

Optie A: Cascade (Alles weg)

Als de User wordt verwijderd, worden zijn Posts ook automatisch verwijderd.


$table->foreignId('user_id')
      ->constrained()
      ->onDelete('cascade');

Optie B: Restrict (Blokkeren)

Je kunt de User NIET verwijderen zolang er nog Posts zijn. Je krijgt een database error.


$table->foreignId('user_id')
      ->constrained()
      ->onDelete('restrict');

Optie C: Set Null (Loskoppelen)

Als de User wordt verwijderd, blijven de Posts bestaan, maar wordt user_id op NULL gezet. (Kolom moet wel nullable() zijn!).


$table->foreignId('user_id')
      ->nullable() // Belangrijk!
      ->constrained()
      ->onDelete('set null');
