Mocking & Spies

Soms wil je testen of een bepaalde class wordt aangeroepen, zonder de echte code uit te voeren. Dit noemen we Mocking. Dit is handig voor services die 'bijeffecten' hebben, zoals het sturen van een SMS via Twilio of het afschrijven van geld via Stripe.

Leeruitkomsten

  • Ik weet wat Mocking is en wanneer ik het moet gebruiken.
  • Ik kan een service class vervangen door een Mock object.
  • Ik kan controleren of methodes op de mock zijn aangeroepen (Spying).

Wanneer Mocken?

Gebruik mocks voor externe afhankelijkheden die:

  • Traag zijn.
  • Geld kosten per aanroep.
  • Niet deterministisch zijn (bijv. een random getal of de huidige tijd).
  • Bijeffecten hebben die je niet wilt tijdens testen (echte emails sturen).

Een Mock Maken

Laravel gebruikt de library Mockery onder water. Je kunt de mock methode gebruiken om een class in de Service Container te vervangen.

Bestand: tests/Feature/PaymentTest.php

use App\Services\PaymentGateway;

test('orders are paid via gateway', function () {
    // 1. Maak de Mock
    // We vertellen Laravel: "Als iemand PaymentGateway nodig heeft,
    // geef dan dit nep-object."
    $this->mock(PaymentGateway::class, function ($mock) {
        // We verwachten dat de 'charge' methode 1x wordt aangeroepen
        // met het bedrag 1000.
        $mock->shouldReceive('charge')
             ->once()
             ->with(1000)
             ->andReturn('transaction_123'); // Nep resultaat
    });

    // 2. Act: Roep de controller aan die de PaymentGateway gebruikt
    $this->post('/checkout', ['amount' => 1000]);
});

Spies (Spionnen)

Een Spy is een variant op een Mock. Bij een Mock definieer je vooraf wat er moet gebeuren (expectations). Bij een Spy voer je eerst de code uit, en controleer je achteraf of de methode is aangeroepen.


test('notification is sent via spy', function () {
    // 1. Maak een Spy
    $spy = $this->spy(NotificationService::class);

    // 2. Act
    $this->post('/register', ['email' => 'henk@test.com']);

    // 3. Assert (achteraf)
    $spy->shouldHaveReceived('sendWelcomeEmail')
        ->once()
        ->with('henk@test.com');
});
