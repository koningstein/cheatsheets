Faking

Naast generieke mocks (via Mockery) biedt Laravel krachtige ingebouwde "Fakes" voor specifieke fa√ßades zoals Mail, Event, Notification, Queue en Storage. Deze zijn nog eenvoudiger in gebruik dan handmatige mocks.

Leeruitkomsten

  • Ik kan `Mail::fake()` gebruiken om te voorkomen dat mails echt verstuurd worden.
  • Ik kan `Event::fake()` gebruiken om events te testen.
  • Ik kan `Storage::fake()` gebruiken voor bestandsuploads.

Mail Faking

Je wilt niet dat je test echte e-mails verstuurt naar klanten. Met Mail::fake() vangt Laravel alle uitgaande mails op.

Bestand: tests/Feature/OrderTest.php

use Illuminate\Support\Facades\Mail;
use App\Mail\OrderShipped;

test('order shipped mail is sent', function () {
    Mail::fake();

    // Act: Order afronden (wat de mail zou moeten triggeren)
    $this->post('/orders/1/ship');

    // Assert: Check of de mail verstuurd is naar de juiste persoon
    Mail::assertSent(OrderShipped::class, function ($mail) {
        return $mail->hasTo('klant@example.com');
    });

    // Of check dat er NIET gemaild is (bijv. bij error)
    // Mail::assertNothingSent();
});

Storage Faking

Bij het uploaden van bestanden wil je niet dat je echte storage/app map vervuild raakt. Storage::fake('avatars') maakt een tijdelijke map aan die na de test automatisch wordt opgeruimd.


use Illuminate\Support\Facades\Storage;
use Illuminate\Http\UploadedFile;

test('avatar upload works', function () {
    Storage::fake('public'); // Simuleer de 'public' disk

    $file = UploadedFile::fake()->image('avatar.jpg');

    $this->post('/profile', [
        'avatar' => $file,
    ]);

    // Assert: Check of het bestand bestaat op de neppe disk
    Storage::disk('public')->assertExists('avatars/' . $file->hashName());
});
