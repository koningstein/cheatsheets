Events & Listeners

Naarmate je applicatie groeit, worden je controllers vaak te groot ("Fat Controllers"). Stel dat er een bestelling wordt geplaatst. Je moet: 1. De order opslaan, 2. Voorraad afboeken, 3. Bevestigingsmail sturen, 4. SMS sturen naar magazijn, 5. Statistieken bijwerken. Als je dit allemaal in de `store` methode propt, wordt het onleesbaar en traag.

Leeruitkomsten

  • Ik begrijp het concept van het Observer Pattern en decoupling.
  • Ik kan events en listeners aanmaken en koppelen.
  • Ik kan events dispatchen en listeners asynchroon laten werken.

Waarom: Decoupling (Ontkoppeling)

Met Events en Listeners splits je dit op. De controller doet één ding: de order opslaan en roepen: "Hé, er is een order geplaatst!" (Het Event). Vervolgens zijn er losse stukjes code (Listeners) die reageren: "Oké, dan stuur ik nu een mail", "Oké, dan boek ik voorraad af". Dit heet het Observer Pattern.

Hoe: Een Event Maken

Maak een event aan: php artisan make:event OrderPlaced. Een event is vaak een simpele data-container (DTO).

Bestand: app/Events/OrderPlaced.php

namespace App\Events;

use App\Models\Order;
use Illuminate\Foundation\Events\Dispatchable;
use Illuminate\Queue\SerializesModels;

class OrderPlaced
{
    use Dispatchable, SerializesModels;

    // Publieke properties zijn toegankelijk voor de Listeners
    public function __construct(public Order $order) {}
}

Hoe: Een Listener Maken

Maak een listener: php artisan make:listener SendOrderConfirmation --event=OrderPlaced. De listener bevat de logica.

Bestand: app/Listeners/SendOrderConfirmation.php

namespace App\Listeners;

use App\Events\OrderPlaced;
use Illuminate\Support\Facades\Mail;

class SendOrderConfirmation
{
    public function handle(OrderPlaced $event): void
    {
        // Logica om te mailen
        // We hebben toegang tot $event->order
        Mail::to($event->order->user)->send(new OrderShipped($event->order));
    }
}

Hoe: Het Event Dispatchen

In je controller hoef je nu alleen nog maar het event af te vuren. Laravel zoekt automatisch de bijbehorende listeners.

Bestand: app/Http/Controllers/OrderController.php

public function store(Request $request)
{
    $order = Order::create($request->all());

    // Roep het event af. Alle gekoppelde listeners worden nu uitgevoerd.
    OrderPlaced::dispatch($order);

    return redirect('/success');
}

Bonus: Je kunt listeners eenvoudig implements ShouldQueue geven. Dan voert Laravel ze op de achtergrond uit (asynchroon) zodat de gebruiker niet hoeft te wachten op het versturen van de e-mail.
