File Storage

Het uploaden en beheren van bestanden (afbeeldingen, PDF's, CSV's) is een veelvoorkomende taak. Laravel maakt dit eenvoudig via de Storage Facade. Het mooie hiervan is dat je code hetzelfde blijft, of je de bestanden nu opslaat op je lokale harde schijf (tijdens development) of op Amazon S3 (in productie).

Leeruitkomsten

  • Ik begrijp de werking van de Storage Facade en disks.
  • Ik kan bestanden valideren en uploaden naar de public disk.
  • Ik kan geüploade bestanden weergeven en verwijderen.

Waarom: De Storage Facade

In plaats van direct met functies als `move_uploaded_file` te werken, gebruik je een abstractie. In config/filesystems.php definieer je je 'disks'.

  • local: Bestanden worden opgeslagen in storage/app. Deze zijn NIET publiekelijk toegankelijk via de browser (veilig voor facturen, ID-kaarten).
  • public: Bestanden worden opgeslagen in storage/app/public. Deze zijn bedoeld om publiekelijk te delen (avatars, blog images).
  • s3: Cloud storage (AWS).

Hoe: Uploaden van Bestanden

Het uploaden gaat het makkelijkst direct via het Request object. De store methode genereert automatisch een unieke bestandsnaam (hash) zodat bestanden elkaar niet overschrijven.

Bestand: app/Http/Controllers/ProfileController.php

public function update(Request $request)
{
    $request->validate([
        'avatar' => 'required|image|max:2048', // max 2MB
    ]);

    if ($request->hasFile('avatar')) {
        // Sla op in de map 'avatars' op de 'public' disk
        // Het resultaat is het pad: "avatars/hashnaam.jpg"
        $path = $request->file('avatar')->store('avatars', 'public');

        // Sla het pad op in de database
        $request->user()->update(['avatar_path' => $path]);
    }

    return back()->with('success', 'Avatar geüpload!');
}

Hoe: Bestanden Weergeven (De Symbolic Link)

Bestanden in storage/app/public zijn niet direct bereikbaar vanuit de browser, omdat de browser alleen in de public/ map mag kijken. Je moet een snelkoppeling (symlink) maken van public/storage naar storage/app/public.

Terminal (Eenmalig uitvoeren!)
php artisan storage:link

Nu kun je in Blade de URL genereren:


<img src="{{ Storage::url($user->avatar_path) }}" alt="Avatar">
<!-- Genereert: /storage/avatars/hashnaam.jpg -->

Bestanden Verwijderen

Vergeet niet oude bestanden op te ruimen als een gebruiker een nieuwe avatar uploadt!


use Illuminate\Support\Facades\Storage;

if ($user->avatar_path) {
    Storage::disk('public')->delete($user->avatar_path);
}
