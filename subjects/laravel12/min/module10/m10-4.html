Database Regels

Sommige validatie hangt af van wat er al in je database staat. Laravel heeft hiervoor krachtige regels.

Leeruitkomsten

  • Ik weet hoe ik kan valideren of een waarde uniek is in de database.
  • Ik weet hoe ik kan valideren of een waarde bestaat in de database (Foreign Keys).
  • Ik kan de uniek-regel toepassen bij updates, waarbij het eigen record wordt genegeerd.

Database Gerelateerde Regels

Unique (Unieke Waarde)

Gebruik dit om duplicaten te voorkomen, bijvoorbeeld bij e-mailadressen of gebruikersnamen.

'email' => 'required|email|unique:users,email'

Syntax: unique:tabelnaam,kolomnaam.

Exists (Bestaande Waarde)

Gebruik dit bij Foreign Keys. Je wilt zeker weten dat de category_id die een gebruiker opstuurt, ook echt bestaat in de categories tabel.

'category_id' => 'required|exists:categories,id'

Dit voorkomt database crashes (Foreign Key Constraint Violations) en geeft een nette foutmelding aan de gebruiker.

Het "Update Probleem" (Ignoreren)

Stel je bewerkt je profiel en slaat het op zonder je e-mailadres te wijzigen. De regel unique:users zal falen, omdat je e-mailadres al in de database staat (namelijk bij jou!).

Je moet Laravel vertellen: "Controleer op uniekheid, maar negeer de regel met MIJN id". Hiervoor gebruik je de Rule class.

Bestand: app/Http/Requests/UpdateUserRequest.php

use Illuminate\Validation\Rule;

public function rules(): array
{
    // We hebben toegang tot de user die wordt geupdate via $this->user (route parameter)
    $userId = $this->route('user')->id;

    return [
        'name'  => 'required|string|max:255',
        'email' => [
            'required',
            'email',
            // Forceer uniekheid in users tabel, maar negeer deze specifieke ID
            Rule::unique('users', 'email')->ignore($userId),
        ],
    ];
}
