Controller Validatie

De snelste manier om inkomende data te valideren is direct in je controller methode. Laravel biedt hiervoor de validate methode op het $request object.

Leeruitkomsten

  • Ik weet hoe ik data direct in de controller kan valideren.
  • Ik ken de veelgebruikte validatieregels in Laravel.
  • Ik begrijp wat er gebeurt als validatie faalt.

Basisgebruik

Je geeft een array mee met veldnamen en de regels waaraan ze moeten voldoen. De regels scheid je met een pipe-teken (|) of je gebruikt een array.

Bestand: app/Http/Controllers/ProjectController.php

public function store(Request $request)
{
    // Valideer de inkomende data
    // Als dit faalt, stopt de code direct en redirect Laravel terug
    $validated = $request->validate([
        'title'       => 'required|min:3|max:255',
        'description' => 'required|string',
        'budget'      => 'nullable|numeric|min:0',
        'email'       => 'required|email',
    ]);

    // Code hieronder wordt alleen uitgevoerd als validatie slaagt.
    // $validated bevat nu alleen de goedgekeurde data (veilig voor Mass Assignment).

    Project::create($validated);

    return redirect()->route('projects.index');
}

Wat gebeurt er bij een fout?

Als een regel niet wordt nageleefd (bijvoorbeeld: de titel is leeg), gooit Laravel automatisch een ValidationException.

  • De gebruiker wordt direct teruggestuurd naar het formulier.
  • Alle foutmeldingen worden in de sessie geplaatst (de $errors variabele).
  • De ingevulde data wordt onthouden (via old input), zodat de gebruiker niet opnieuw hoeft te beginnen.

Veelgebruikte Standaard Regels

  • required: Het veld mag niet leeg zijn.
  • string / numeric / boolean: Het datatype moet kloppen.
  • min:waarde / max:waarde: Voor strings is dit de lengte, voor getallen de waarde.
  • email: Moet een geldig e-mailformaat zijn.
  • nullable: Het veld mag leeg zijn (bijv. optionele opmerkingen).
