Index & Eager Loading

Als we lijsten tonen met data uit meerdere tabellen, kan de applicatie snel traag worden. We leren hoe we het N+1 probleem oplossen met Eager Loading.

Leeruitkomsten

  • Ik kan gerelateerde data (categorie, prijs) tonen in een tabel.
  • Ik begrijp het N+1 probleem.
  • Ik kan Eager Loading (with()) gebruiken om dit op te lossen.

1. Het N+1 Probleem

Als we in de view voor elk product de categorie naam opvragen ($product->category->name), voert Laravel op de achtergrond steeds een nieuwe query uit. Bij 100 producten zijn dat 101 queries! Dit maakt de applicatie onnodig traag.

2. Eager Loading

Met with() halen we de relaties in één keer op. In de Models definiëren we de relaties, bijvoorbeeld category() en een handige latest_price().


// app/Models/Product.php
public function category(): BelongsTo
{
    return $this->belongsTo(Category::class);
}

// Speciale relatie voor de huidige prijs
public function latest_price(): HasOne
{
    return $this->hasOne(Price::class)->orderBy('effdate', 'desc');
}

In de controller gebruiken we with om de data efficiënt op te halen:


// app/Http/Controllers/Admin/ProductController.php
public function index()
{
    // Haal alles in 1x op met relaties
    $products = Product::with('category', 'latest_price')->paginate(15);

    return view('admin.products.index', ['products' => $products]);
}

3. De View

In de Blade view kunnen we nu direct de relaties aanspreken zonder extra database queries te veroorzaken.


@foreach($products as $product)
    <tr>
        <td>{{ $product->name }}</td>

        <!-- Gerelateerde data -->
        <td>{{ $product->category->name }}</td>

        <td>€ {{ $product->latest_price ? $product->latest_price->price : 'N/A' }}</td>

        <td>Edit | Delete</td>
    </tr>
@endforeach
