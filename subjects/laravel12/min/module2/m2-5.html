Middleware bij routes

Niet iedereen mag zomaar overal bij. We gebruiken middleware om routes te beveiligen, zodat alleen ingelogde gebruikers of gebruikers met specifieke rechten toegang krijgen tot bepaalde pagina's.

Leeruitkomsten

  • Ik kan middleware aliases registreren in bootstrap/app.php.
  • Ik kan een custom 403 error page maken.
  • Ik kan redirect after login aanpassen.
  • Ik kan auth middleware op routes toepassen.
  • Ik kan conditionele content tonen (@auth/@guest).

Stap 1: Middleware Aliases Registreren

Om de middleware van Spatie te kunnen gebruiken, moeten we deze registreren in bootstrap/app.php. We voegen hier alle aliases toe.


// bootstrap/app.php

return Application::configure(basePath: dirname(__DIR__))
    ->withMiddleware(function (Middleware $middleware) {
        $middleware->alias([
            'role' => \Spatie\Permission\Middleware\RoleMiddleware::class,
            'permission' => \Spatie\Permission\Middleware\PermissionMiddleware::class,
            'role_or_permission' => \Spatie\Permission\Middleware\RoleOrPermissionMiddleware::class,
        ]);
    })
    // ...

Stap 2: Routes Beveiligen

In routes/web.php kunnen we nu een groep maken die beveiligd is. We willen dat alleen gebruikers met de rol admin of sales bij de admin-routes kunnen.


// routes/web.php

Route::group(['middleware' => ['role:sales|admin']], function () {

    // Deze routes zijn nu beveiligd
    Route::resource('/admin/categories', Admin\CategoryController::class);

});

Stap 3: Custom 403 Error Pagina

Als iemand geen toegang heeft, zien ze een standaard "403 | Forbidden" scherm. We willen dit in onze eigen stijl.

Maak een bestand aan: resources/views/errors/403.blade.php.


@extends('layouts.layoutpublic')

@section('content')
    <div class="bg-red-100 border border-red-400 text-red-700 px-4 py-3 rounded relative" role="alert">
        <strong class="font-bold">Error!</strong>
        <span class="block sm:inline">{{ $exception->getMessage() }}</span>
    </div>
@endsection
