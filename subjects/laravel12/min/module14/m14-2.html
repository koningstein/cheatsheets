Code Style met Laravel Pint

Code schrijf je niet alleen voor computers, maar ook voor je collega's (en je toekomstige zelf). Een inconsistente code style maakt lezen lastig. Laravel Pint is een tool die je code automatisch formatteert volgens de standaarden van Laravel (gebaseerd op PSR-12). Het is standaard aanwezig in elke nieuwe Laravel installatie.

Leeruitkomsten

  • Ik begrijp het belang van uniforme code style (PSR-12).
  • Ik kan Laravel Pint uitvoeren om mijn code te formatteren.
  • Ik kan Pint configureren voor specifieke regels.

Wat doet Pint?

Pint is gebouwd bovenop PHP-CS-Fixer. Het lost automatisch slordigheden op zoals:

  • Verkeerde inspringing (spaces vs tabs).
  • Ontbrekende spaties na `if` of `foreach`.
  • Ongebruikte imports (`use` statements) verwijderen.
  • Array syntax standaardiseren ([] ipv array()).

Alleen testen (Dry Run)

Wil je alleen zien wat er mis is, zonder dat Pint je bestanden aanpast? Gebruik de --test vlag.

./vendor/bin/pint --test

Configuratie

Standaard volgt Pint de 'laravel' preset. Wil je iets aanpassen? Maak een pint.json in je root.


{
    "preset": "laravel",
    "rules": {
        "no_unused_imports": true,
        "array_syntax": {"syntax": "short"}
    }
}
