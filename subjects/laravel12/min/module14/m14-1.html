Static Analysis met Larastan

Testen is één manier om fouten te vinden (door de code daadwerkelijk uit te voeren). Een andere manier is Static Analysis: een tool die je code leest zonder deze uit te voeren en zoekt naar typefouten, onbestaande methodes en mogelijke bugs. Voor Laravel is Larastan (een wrapper om PHPStan) de standaard.

Leeruitkomsten

  • Ik weet wat Static Analysis is en wat het verschil is met Unit Testing.
  • Ik kan Larastan installeren en configureren (`phpstan.neon`).
  • Ik kan fouten oplossen die Larastan rapporteert.

Waarom Larastan?

PHP is van zichzelf niet "type-safe". Je kunt per ongeluk een string doorgeven aan een functie die een integer verwacht. Larastan controleert dit voor je. Het vindt fouten als:

  • Toegang tot properties die niet bestaan op een Model.
  • Methodes aanroepen met verkeerde argumenten.
  • Vergeten return statements.
  • Type mismatches (bijv. array verwachten maar null krijgen).

Installatie & Gebruik

Larastan installeer je als development dependency.

Terminal

composer require --dev larastan/larastan

Maak vervolgens een configuratiebestand phpstan.neon aan in de root van je project:


includes:
    - vendor/larastan/larastan/extension.neon

parameters:
    paths:
        - app/

    # Level 9 is het strengst. Begin op level 5 als je start.
    level: 5

Start de analyse met het volgende commando:

./vendor/bin/phpstan analyse

Voorbeeld Foutmelding

Als je output krijgt als: Access to an undefined property App\Models\User::$fullname, weet je dat je een typefout hebt gemaakt in je code (bijv. $user->fullname terwijl het $user->full_name moet zijn), of dat je property niet gedocumenteerd is via PHPDoc.
